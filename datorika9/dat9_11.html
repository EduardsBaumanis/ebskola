Teorija: Profesionāli rīki

Visual Studio Code (VS Code) ir koda redaktors, nevis teksta apstrādes rīks (kā Word). Tas saprot koda struktūru un palīdz to rakstīt ar krāsu izcelšanu un paplašinājumiem. GitHub ir koda "mākonis", kas glabā visas tavas izmaiņas (versijas). Tas ļauj Tev piekļūt savam darbam no jebkura datora un droši iesniegt mājasdarbus skolotājam.

Svarīgākie jēdzieni:

  • Repository (Repo): Tava projekta mape mākonī.
  • Commit: Izmaiņu "nofotografēšana" un saglabāšana ar komentāru.
  • Push: Saglabāto izmaiņu nosūtīšana no lokālā datora uz GitHub serveri.

1. uzdevums: Identitāte digitālajā vidē

Izveidot personīgo GitHub kontu un uzstādīt VS Code darba vidi.

Izpildes soļi:

  1. Reģistrējies github.com, izmantojot skolas vai personīgo e-pastu (atceries apstiprināt e-pastu!).
  2. Atver VS Code un kreisajā rīkjoslā (Extensions) atrodi un instalē "Live Server" — tas ļaus redzēt koda izmaiņas reālajā laikā.
  3. VS Code apakšējā kreisajā stūrī noklikšķini uz "Accounts" un pieraksties savā GitHub kontā, lai autorizētu ierīci.

2. uzdevums: Repozitorija un mapju struktūra

Sagatavot lokālo darba mapi un savienot to ar GitHub mākoņkrātuvi.

Izpildes soļi:

  1. Izveido datorā mapi Datorika_9.
  2. GitHub vietnē izveido jaunu repozitoriju (New repository) ar nosaukumu datorika-devitais, atzīmē "Public" un "Add a README file".
  3. VS Code izmanto "Clone Repository" funkciju, iekopē sava jaunā GitHub repo adresi un izvēlies savu izveidoto mapi Datorika_9 kā mērķi.

3. uzdevums: Pirmais "Commit" un publikācija

Izveidot pirmo failu un publicēt to tiešsaistē, lai pārbaudītu saiti starp datoru un mākoni.

Izpildes soļi:

  1. VS Code izveido jaunu failu index.html klonētajā mapē.
  2. Ieraksti failā tekstu: <h1>Sveika, pasaule!&s;lt;/h1> un saglabā (Ctrl+S).
  3. Source Control sadaļā (kreisajā pusē) ieraksti ziņojumu "Mans pirmais fails", nospied "Commit" un tad "Sync Changes/Push". Pārbaudi GitHub vietnē, vai fails ir tur!

Biežākās kļūdas (un kā tās labot):

  • Fails neparādās GitHub: Pārliecinies, vai pēc "Commit" nospiedi "Push" vai "Sync Changes".
  • VS Code neatpazīst Git: Restartē programmu pēc Git instalēšanas.
  • E-pasta apstiprināšana: GitHub neļaus veikt "Push", ja e-pasts nav apstiprināts.
