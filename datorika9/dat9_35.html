3.5. tēma: Cikli (Loops) un atkārtošanās loģika

Tavs šīs stundas izaicinājums: Izstrādāt ciklisku algoritmu, lai automātiski apstrādātu datu masīvus un ģenerētu HTML saturu, novēršot koda atkārtošanos.

2.6.1. Pieraksta, lasa un izpilda algoritmus ar cikliskām darbībām; 5.2.1. Problēmrisināšana un optimizācija.

Teorija: Automātiskais dzinējs (For cikls)

Lai liktu datoram atkārtot darbību, mēs izmantojam for ciklu. Tas sastāv no trim svarīgām daļām:

  • Sākums: let i = 0; (Kur mēs sākam skaitīt? Parasti no 0, jo masīvi sākas ar 0).
  • Nosacījums: i < 5; (Cik ilgi mēs atkārtosim? Kamēr i ir mazāks par 5).
  • Solis: i++ (Ko darām pēc katra apļa? Palielinām i par 1).
Cikli ir ideāli piemēroti, lai "izietu cauri" visiem masīva elementiem, izmantojot masivs.length.

1. uzdevums: Skaitīšanas mašīna

Uzrakstīt savu pirmo ciklu, kas automātiski izvada skaitļus konsolē.

Izpildes soļi:

  1. Savā script.js failā izveido struktūru: for (let i = 1; i <= 5; i++) { }.
  2. Starp figūriekavām ieraksti komandu: console.log("Šis ir aplis numur: " + i);.
  3. Palaid kodu un apskaties konsolē – tu redzēsi piecus ziņojumus, kaut gan uzrakstīji tikai vienu koda rindu!

2. uzdevums: Masīva apstrāde ar ciklu

Savienot ciklu ar masīvu, lai izvadītu visus spēlētāja inventāra priekšmetus.

Izpildes soļi:

  1. Izveido inventāra masīvu: let inventars = ["Zobens", "Vairogs", "Zelts", "Atslēga"];.
  2. Izveido for ciklu, kas sākas no 0, bet nosacījumā izmanto masīva garumu: i < inventars.length.
  3. Cikla iekšpusē izvadi katru elementu konsolē: console.log("Priekšmets: " + inventars[i]);. Mainīgais i katrā aplī kļūs par nākamo indeksu (0, tad 1, tad 2...).

3. uzdevums: HTML saraksta ģenerēšana

Izmantot ciklu, lai automātiski izveidotu redzamu sarakstu spēles tīmekļa lapā.

Izpildes soļi:

  1. Sava index.html faila saskarnes daļā pievieno tukšu sarakstu ar ID: <ul id="inventara-saraksts"></ul>.
  2. JS faila cikla iekšpusē pievieno komandu, kas katru priekšmetu ievieto HTML formātā (kā saraksta elementu <li>):
    document.getElementById("inventara-saraksts").innerHTML += "<li>" + inventars[i] + "</li>";.
  3. Ievēro += zīmi! Tā nozīmē "pievieno klāt esošajam", nevis "aizstāj". Sinhronizē darbu ar GitHub.

Biežākās kļūdas (un kā tās labot):

  • Mūžīgais cikls (Infinite Loop): Ja uzrakstīsi i-- (samazināt) vietā i++ (palielināt), nosacījums i < 5 vienmēr būs patiess. Pārlūks uzkārsies un "iesaldēs" lapu. Rūpīgi pārbaudi cikla soli!
  • Komati semikolu vietā: For cikla iekavās trīs daļas ir jāatdala ar semikolu (;), nevis komatu. for (let i = 0, i < 5, i++) izmetīs kļūdu.
  • Kļūda "undefined" pēdējā aplī: Ja izmantosi i <= inventars.length (mazāks vai vienāds), cikls mēģinās nolasīt neeksistējošu elementu. Indeksi beidzas pirms garuma, tāpēc izmanto tikai < (mazāks).
