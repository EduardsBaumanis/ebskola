4.6. tēma: Dzinēja prototipa testēšana un koda analīze

Tavs šīs stundas izaicinājums: Apvienot visas izstrādātās komponentes funkcionējošā prototipā, identificēt un novērst loģiskās kļūdas (bugs) un veikt koda optimizāciju pirms noslēguma darba sākšanas.

2.6.1. Veic programmas atkļūdošanu (debugging); 1.3.4. Novērtē programmas darbības rezultātu un atbilstību mērķim.

Teorija: Kas ir "Bug" un kā to noķert?

Programmēšanā kļūdas jeb "bugs" ir neizbēgamas. To labošanas process (debugging) aizņem līdz pat 50% no izstrādes laika.

  • Sintakses kļūdas: Tavs kods "nesarunājas" ar pārlūku (piem., aizmirsta iekava). To rāda Console sarkanā krāsā.
  • Loģikas kļūdas: Kods izpildās, bet dara nepareizi (piem., punkti tiek atņemti, nevis pieskaitīti).
  • Console ir tavs labākais draugs: Izmanto console.log() katrā aizdomīgā koda vietā, lai redzētu, kādas vērtības tajā brīdī ir mainīgajiem.

1. uzdevums: Pilna cikla integrācija

Apvienot visas iepriekšējās daļas tā, lai spēlei būtu skaidrs sākums, gaita un beigas.

Izpildes soļi:

  1. Izveido funkciju function saktSpeli() { ... }, kas iestata sākuma vērtības (punkti = 0, laiks = 30) un palaiž visus setInterval ciklus.
  2. Piesaisti šo funkciju HTML pogai "Sākt spēli".
  3. Nodrošini, ka pēc spēles beigām (laiks = 0 vai sadursme) visi intervāli tiek apstādināti (clearInterval), lai tie lieki netērētu datora resursus.

2. uzdevums: "Pīļu testēšana" un kļūdu labošana

Veikt koda testēšanu un kļūdu identifikāciju, strādājot pāros (Peer-review).

Izpildes soļi:

  1. Apmainies ar solabiedru ar sava projekta GitHub saiti vai atver blakus esošo datoru.
  2. Mēģini "salauzt" biedra spēli (piemēram, turot vairākus taustiņus, ātri klikšķinot "Sākt").
  3. Ja atrodi kļūdu, nevis vienkārši pasaki to, bet mēģini izskaidrot koda rindu, kas to izraisa. Pieraksti vismaz 2 uzlabojumus savā vai biedra kodā.

3. uzdevums: Koda optimizācija un "Clean Code"

Sakārtot kodu, lai tas būtu salasāms un gatavs vērtēšanai.

Izpildes soļi:

  1. Pievieno komentārus (//) pie visām galvenajām funkcijām un cikliem, paskaidrojot, ko tie dara.
  2. Pārliecinies, ka mainīgo nosaukumi ir saprotami (piem., p vietā izmanto punkti).
  3. Izdēs visus liekos console.log, kas vairs nav vajadzīgi spēlēšanai. Sinhronizē pēdējās izmaiņas ar GitHub.

Pašpārbaudes kontrolsaraksts (Pirms 5. tēmas):

  • ✅ Vai varonis kustas visos 4 virzienos?
  • ✅ Vai punkti un laiks atjaunojas HTML saskarnē (nevis tikai konsolē)?
  • ✅ Vai sadursme ar šķērsli izraisa "Game Over"?
  • ✅ Vai spēli var sākt un apstādināt ar pogām?
  • ✅ Vai kods ir sakārtots un komentēts?

Debugging piemērs (Kā "ārstēt" kodu)

// PROBLĒMA: Spēle kļūst arvien ātrāka, katru reizi nospiežot "Sākt"
let kustibasIntervils;

function saktSpeli() {
    // 1. RISINĀJUMS: Pirms sākt jaunu ciklu, vienmēr nodzēšam iepriekšējo!
    clearInterval(kustibasIntervils); 
    
    atlikusaisLaiks = 30;
    
    // 2. LOGOŠANA: Pārbaudām, vai dati ir ielādējušies pareizi
    console.log("Spēle startēta ar laiku: " + atlikusaisLaiks);
    
    kustibasIntervils = setInterval(spelesLogika, 20);
}

function spelesLogika() {
    // 3. PĀRBAUDE: Ja funkcija nestrādā, ieliec logu šeit, lai redzētu vai cikls vispār iet
    // console.log("Cikls griežas..."); 
    
    kustinatIenaidniekus();
    parbauditSadursmi();
}
