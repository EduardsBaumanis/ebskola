3. tēma: Algoritmi un JavaScript pamati

Tavs mērķis: Izstrādāt un testēt interaktīvu mini-programmu, kas saņem spēlētāja datus, pieņem lēmumus ar sazarojumiem un automātiski ģenerē saturu, izmantojot datu masīvus un ciklus.

2.6.1., 2.6.2., 2.4.7., 5.2.1., 5.2.3.

Darba uzdevuma gaita

1. solis: Datu bāzes un ievades izveide (Pamatlīmenis)

  1. Pieslēdz savam HTML dokumentam ārējo script.js failu.
  2. HTML lapā izveido vietu izvadei: vienu <h2> tagu spēlētāja sveicienam un vienu tukšu <ul> sarakstu ar ID (piemēram, id="inventars").
  3. JS failā izmanto prompt(), lai pajautātu lietotājam viņa varoņa vārdu, un ievieto šo vārdu HTML <h2> elementā (izmantojot innerText).
  4. Izveido masīvu ar vismaz 3 spēles sākuma priekšmetiem (piemēram, "Karte", "Lāpa", "Ūdens").

2. solis: Lēmumu pieņemšana (Vidējais grūtības līmenis)

  1. Izmanto vēl vienu prompt(), lai uzdotu spēlētājam jautājumu (piem., "Vai vēlies doties pa kreisi (1) vai pa labi (2)?"). Pārveido atbildi par skaitli.
  2. Izveido if / else sazarojumu, kas pārbauda ievadīto atbildi.
  3. Ja spēlētājs ievada 1, izvadi alert() ar ziņojumu, ka viņš atrada dārgumus, un pievieno (push()) jaunu priekšmetu (piemēram, "Zelts") tavam masīvam.
  4. Ja spēlētājs ievada kaut ko citu (else), izvadi ziņojumu, ka viņš apmaldījās, un maini vienu priekšmetu masīvā (piemēram, nomaini "Ūdens" pret "Tukša pudele").

3. solis: Algoritma automatizācija ar ciklu (Paaugstināts grūtības līmenis)

  1. Faila beigās (pēc if/else bloka) izveido for ciklu, kas darbosies tik reižu, cik elementu ir tavā masīvā (izmanto .length).
  2. Cikla iekšpusē uzraksti kodu, kas katru masīva elementu pievieno HTML lapas <ul> sarakstam kā jaunu <li> elementu, izmantojot innerHTML +=.
  3. Atver pārlūka "Console" (F12) un pārliecinies, ka kodā nav sarkano kļūdu paziņojumu. Pabeidz darbu ar Commit un Push uz GitHub.

Datu struktūras un Ievade/Izvade (2.6.1., 2.4.7.) Ir mēģināts izveidot mainīgos un masīvu, bet prompt vai datu izvade HTML nedarbojas (sintakses kļūdas). Spēlētāja vārds tiek saņemts un izvadīts. Masīvs ir izveidots, taču tajā netiek veiktas izmaiņas spēles gaitā. Dati tiek veiksmīgi nolasīti. Masīvs tiek dinamiski mainīts (izmantojot push vai pārrakstīšanu) atkarībā no situācijas. Datu struktūra ir nevainojama un atbilst spēles loģikai. Kodā nav kļūdu, tiek pareizi apstrādāti datu tipi (teksts un skaitļi nejaucas).
Algoritmu loģika – Sazarojumi un Cikli (2.6.1., 2.6.2., 5.2.1.) Cikls vai sazarojums ir uzrakstīts kļūdaini, pārlūks nevar izpildīt skriptu. If/else struktūra strādā un reaģē uz ievadi. For cikls nav izveidots vai neprot izvadīt HTML sarakstu. Gan lēmumu loģika, gan for cikls strādā korekti. Cikls automātiski ģenerē sarakstu HTML lapā no masīva datiem. Izveidota sarežģītāka loģika (piem., else if vai loģiskie operatori). Cikls ir perfekti piesaistīts masīva garumam. Kods ir optimizēts un loģiski strukturēts.
