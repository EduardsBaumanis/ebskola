5.6 Noslēguma projekts: Roguelike

Tavs šīs stundas izaicinājums: Pilnvērtīga roguelike spēle ar procedural generation, save sistēmu un meta-progress.

SR 2.4.1. Problēmu dekompozīcija SR 2.4.16. Datu struktūras SR 2.4.17. Failu I/O SR 2.4.19. Algoritmi

Teorija: Spēles apraksts

Roguelike pazemes spēle: spēlētājs iziet cauri nejauši ģenerētiem dungeons, savāc loot, kaujas ar ienaidniekiem, mirstot zaudē viss progress (run), bet meta-progress (kopējie XP, atklātās klases) saglabājas.

Galvenās funkcijas:

  • Procedurāli ģenerēti 10 līmeņu dungeons.
  • 3 spēlētāja klases ar dažādām spējām.
  • 50+ unique items un ienaidnieku tipi.
  • Boss katram 5. līmenim.
  • Save/load starp sesijām (meta-progress).
  • Procedurāla loot ģenerēšana.

1. uzdevums: Dungeon un Player (1.)

Pamata struktūra.

Izpildes soļi:

  1. DungeonGenerator no 5.4 stundas.
  2. Player ar HP, attack, inventory.
  3. Combat sistēma turn-based vai real-time (izvēlies).
  4. 1 līmenis funkcionējoshi.

2. uzdevums: Loot un progresija (2.)

Spēles dziļums.

Izpildes soļi:

  1. ItemResource klases priekš weapons, armor, potions.
  2. Procedurāla loot generation (pēc level + retums).
  3. XP un level up sistēma.
  4. 10 līmeņi ar augošu sarežģītību.

3. uzdevums: Save un meta-progress (3.)

Persistence.

Izpildes soļi:

  1. SaveManager no 5.3 stundas.
  2. Single save slot ar autosave starp līmeņiem.
  3. Pēc death — save tiek dzēsts (roguelike princips), bet meta-progress paliek.
  4. Meta: kopējais kills, deepest level reached, unlocked classes.
  5. GitHub: 10+ commit, README ar gameplay video link.

Papildus uzdevums: Boss un beigas

Spēles kulminācija.

Izpildes soļi:

  1. 10. līmenis — final boss ar 1000 HP.
  2. Multi-phase cīņa.
  3. Pēc uzvaras — credit screen un statistika.

Biežākās kļūdas

  • Save un mirst: Pārbaudi, ka delete save fails notiek tikai pēc death, ne pirms.
  • Performance ar 50+ enemies: Lieto spatial hash un pool no 4. tēmas.
  • Procedural balance: Pārāk daudz/maz loot — pielāgo varbūtības.

Godot ekrānuzņēmumi

Koda piemērs (paplašināts)

class GameSession : public Node {
    GDCLASS(GameSession, Node)
private:
    int current_level = 1;
    Player* player;
    DungeonGenerator gen;

public:
    void start_run() {
        gen.generate(Time::get_singleton()->get_unix_time_from_system());
        spawn_player();
        spawn_enemies();
        spawn_loot();
    }

    void on_player_died() {
        // Save meta-progress
        MetaProgress::instance()->add_run({current_level, player->kills});
        MetaProgress::save();
        // Remove run save
        FileAccess::remove("user://current_run.json");
        get_tree()->change_scene_to_file("res://menu.tscn");
    }

    void on_level_complete() {
        save_run();  // autosave
        current_level++;
        if (current_level > 10) on_game_won();
        else start_next_level();
    }
};
