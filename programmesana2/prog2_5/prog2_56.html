5.6 Noslēguma projekts: Roguelike

Tavs šīs stundas izaicinājums: Pilnvērtīga roguelike spēle ar procedural generation, save sistēmu un meta-progress.

SR 2.4.1. Problēmu dekompozīcija SR 2.4.16. Datu struktūras SR 2.4.17. Failu I/O SR 2.4.19. Algoritmi

Priekšzināšanu robeža: uzdevumos izmanto tikai iepriekš apgūto un šīs lapas teorijas/koda piemēros parādīto. Ja parādās jauna komanda vai rīks, vispirms atrod tās paraugu teorijas blokā.

Teorija: Spēles apraksts

Roguelike pazemes spēle: spēlētājs iziet cauri nejauši ģenerētiem dungeons, savāc loot, kaujas ar ienaidniekiem, mirstot zaudē viss progress (run), bet meta-progress (kopējie XP, atklātās klases) saglabājas.

Galvenās funkcijas:

  • Procedurāli ģenerēti 10 līmeņu dungeons.
  • 3 spēlētāja klases ar dažādām spējām.
  • 50+ unique items un ienaidnieku tipi.
  • Boss katram 5. līmenim.
  • Save/load starp sesijām (meta-progress).
  • Procedurāla loot ģenerēšana.

Teorijas paplašinājums: C++ fokuss

Kritiskais fokuss: šajā stundā nepietiek tikai panākt redzamu efektu editorā. Jāspēj paskaidrot, kura C++ klase glabā stāvokli, kura metode to maina un kā Godot node struktūra šo kodu izmanto.

Pārbaudes pierādījums: C++ kods pārbauda failu kļūdas, validē datus, izmanto versijas lauku un pamato algoritmu sarežģītību.

Konsolidētais praktiskais uzdevums (15+ soļi)

70 min darba sadalījums: 1. uzdevums (~20 min) — izveido minimālo piemēru, kas pārbauda šīs stundas galveno ideju; 2. uzdevums (~25 min) — integrē risinājumu tēmas spēles projektā; 3. uzdevums (~25 min) — notestē robežgadījumus, salabo kļūdas un pieraksti secinājumu README vai komentārā. Tālāk redzamie soļi ir detalizēta karte šiem trim uzdevumiem; ja laiks beidzas, papildu pulēšanu atstāj nākamajai reizei.

Izpildi šo darba plānu, lai 5.6 Noslēguma projekts: Roguelike rezultāts būtu pārbaudāms C++ GDExtension projektā.

Izpildes soļi:

  1. Atver projektu un izveido Git branch datu pastāvības vai algoritmu darbam.
  2. Pieraksti, kādi dati jāsaglabā, jāielādē vai jāģenerē šajā stundā.
  3. Izveido C++ klasi, kas atbild par šo datu plūsmu, piemēram, SaveManager vai Generator.
  4. Definē datu modeli ar skaidriem tipiem: int, float, String, Dictionary vai pielāgotu struct.
  5. Pievieno versijas lauku, ja dati tiks saglabāti failā.
  6. Rakstīšanas metodē pārbaudi, vai fails tiešām ir atvērts rakstīšanai.
  7. Lasīšanas metodē pārbaudi, vai fails eksistē un nav tukšs.
  8. Validē ielādētos laukus pirms to izmantošanas spēles stāvoklī.
  9. Kļūdas gadījumā atgriez drošu noklusējuma vērtību, nevis ļauj spēlei sabrukt.
  10. Ja tēma ir algoritms, pieraksti aptuveno sarežģītību, piemēram, O(n) vai O(n log n).
  11. Pievieno mazu testdatu piemēru, kura rezultātu vari pārbaudīt ar roku.
  12. Pievieno lielāku testu, lai pamanītu veiktspējas vai atmiņas problēmas.
  13. Pievieno UtilityFunctions::print izvadi tikai svarīgiem debug punktiem.
  14. Pārkompilē C++ kodu ar SCons un izlabo kompilācijas kļūdas.
  15. Palaid Godot un testē saglabāšanu, ielādi vai ģenerāciju vismaz divas reizes pēc kārtas.
  16. README pieraksti failu ceļus, datu formātu un zināmos ierobežojumus.
  17. Pārbaudi, ka ģenerētie vai saglabātie faili nav nejauši pievienoti Git, ja tie ir lokāli dati.
  18. Veic commit ar ziņu, kas nosauc saglabāšanas, ielādes vai algoritma uzlabojumu.

Aktivitāte 1: Dungeon un Player (1.)

Pamata struktūra.

Izpildes soļi:

  1. DungeonGenerator no 5.4 stundas.
  2. Player ar HP, attack, inventory.
  3. Combat sistēma turn-based vai real-time (izvēlies).
  4. 1 līmenis funkcionējoshi.

Aktivitāte 2: Loot un progresija (2.)

Spēles dziļums.

Izpildes soļi:

  1. ItemResource klases priekš weapons, armor, potions.
  2. Procedurāla loot generation (pēc level + retums).
  3. XP un level up sistēma.
  4. 10 līmeņi ar augošu sarežģītību.

Aktivitāte 3: Save un meta-progress (3.)

Persistence.

Izpildes soļi:

  1. SaveManager no 5.3 stundas.
  2. Single save slot ar autosave starp līmeņiem.
  3. Pēc death — save tiek dzēsts (roguelike princips), bet meta-progress paliek.
  4. Meta: kopējais kills, deepest level reached, unlocked classes.
  5. GitHub: 10+ commit, README ar gameplay video link.

Papildu izaicinājums: Boss un beigas

Spēles kulminācija.

Izpildes soļi:

  1. 10. līmenis — final boss ar 1000 HP.
  2. Multi-phase cīņa.
  3. Pēc uzvaras — credit screen un statistika.

Biežākās kļūdas

  • Save un mirst: Pārbaudi, ka delete save fails notiek tikai pēc death, ne pirms.
  • Performance ar 50+ enemies: Lieto spatial hash un pool no 4. tēmas.
  • Procedural balance: Pārāk daudz/maz loot — pielāgo varbūtības.

Godot ekrānuzņēmumi

Koda piemērs (paplašināts)

class GameSession : public Node {
    GDCLASS(GameSession, Node)
private:
    int current_level = 1;
    Player* player;
    DungeonGenerator gen;

public:
    void start_run() {
        gen.generate(Time::get_singleton()->get_unix_time_from_system());
        spawn_player();
        spawn_enemies();
        spawn_loot();
    }

    void on_player_died() {
        // Save meta-progress
        MetaProgress::instance()->add_run({current_level, player->kills});
        MetaProgress::save();
        // Remove run save
        FileAccess::remove("user://current_run.json");
        get_tree()->change_scene_to_file("res://menu.tscn");
    }

    void on_level_complete() {
        save_run();  // autosave
        current_level++;
        if (current_level > 10) on_game_won();
        else start_next_level();
    }
};
