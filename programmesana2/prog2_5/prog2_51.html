5.1 Failu I/O Godot vidē

Tavs šīs stundas izaicinājums: Apgūt FileAccess klasi un Godot failu sistēmu.

SR 2.4.17. Failu un ārējās krātuves SR 2.4.4. Iegūst un apstrādā datus

Priekšzināšanu robeža: uzdevumos izmanto tikai iepriekš apgūto un šīs lapas teorijas/koda piemēros parādīto. Ja parādās jauna komanda vai rīks, vispirms atrod tās paraugu teorijas blokā.

Teorija: FileAccess klase

Godot piedāvā FileAccess klasi failu lasīšanai un rakstīšanai. Tā strādā ar res:// (read-only resources) un user:// (lokālā lietotāja krātuve).

#include <godot_cpp/classes/file_access.hpp>

// Rakstīšana
Ref<FileAccess> file = FileAccess::open("user://save.txt", FileAccess::WRITE);
if (file.is_valid()) {
    file->store_string("Hello, file!");
    file->store_line("Otrā rinda");
    file->store_32(42);  // binary
    file->close();
}

// Lasīšana
Ref<FileAccess> rfile = FileAccess::open("user://save.txt", FileAccess::READ);
if (rfile.is_valid()) {
    String content = rfile->get_as_text();
    // vai pa rindām:
    while (!rfile->eof_reached()) {
        String line = rfile->get_line();
    }
    rfile->close();
}

Teorijas paplašinājums: C++ fokuss

Kritiskais fokuss: šajā stundā nepietiek tikai panākt redzamu efektu editorā. Jāspēj paskaidrot, kura C++ klase glabā stāvokli, kura metode to maina un kā Godot node struktūra šo kodu izmanto.

Pārbaudes pierādījums: C++ kods pārbauda failu kļūdas, validē datus, izmanto versijas lauku un pamato algoritmu sarežģītību.

Konsolidētais praktiskais uzdevums (15+ soļi)

70 min darba sadalījums: 1. uzdevums (~20 min) — izveido minimālo piemēru, kas pārbauda šīs stundas galveno ideju; 2. uzdevums (~25 min) — integrē risinājumu tēmas spēles projektā; 3. uzdevums (~25 min) — notestē robežgadījumus, salabo kļūdas un pieraksti secinājumu README vai komentārā. Tālāk redzamie soļi ir detalizēta karte šiem trim uzdevumiem; ja laiks beidzas, papildu pulēšanu atstāj nākamajai reizei.

Izpildi šo darba plānu, lai 5.1 Failu I/O Godot vidē rezultāts būtu pārbaudāms C++ GDExtension projektā.

Izpildes soļi:

  1. Atver projektu un izveido Git branch datu pastāvības vai algoritmu darbam.
  2. Pieraksti, kādi dati jāsaglabā, jāielādē vai jāģenerē šajā stundā.
  3. Izveido C++ klasi, kas atbild par šo datu plūsmu, piemēram, SaveManager vai Generator.
  4. Definē datu modeli ar skaidriem tipiem: int, float, String, Dictionary vai pielāgotu struct.
  5. Pievieno versijas lauku, ja dati tiks saglabāti failā.
  6. Rakstīšanas metodē pārbaudi, vai fails tiešām ir atvērts rakstīšanai.
  7. Lasīšanas metodē pārbaudi, vai fails eksistē un nav tukšs.
  8. Validē ielādētos laukus pirms to izmantošanas spēles stāvoklī.
  9. Kļūdas gadījumā atgriez drošu noklusējuma vērtību, nevis ļauj spēlei sabrukt.
  10. Ja tēma ir algoritms, pieraksti aptuveno sarežģītību, piemēram, O(n) vai O(n log n).
  11. Pievieno mazu testdatu piemēru, kura rezultātu vari pārbaudīt ar roku.
  12. Pievieno lielāku testu, lai pamanītu veiktspējas vai atmiņas problēmas.
  13. Pievieno UtilityFunctions::print izvadi tikai svarīgiem debug punktiem.
  14. Pārkompilē C++ kodu ar SCons un izlabo kompilācijas kļūdas.
  15. Palaid Godot un testē saglabāšanu, ielādi vai ģenerāciju vismaz divas reizes pēc kārtas.
  16. README pieraksti failu ceļus, datu formātu un zināmos ierobežojumus.
  17. Pārbaudi, ka ģenerētie vai saglabātie faili nav nejauši pievienoti Git, ja tie ir lokāli dati.
  18. Veic commit ar ziņu, kas nosauc saglabāšanas, ielādes vai algoritma uzlabojumu.

Aktivitāte 1: Pirmais saglabāšanas fails

Saglabā tekstu user:// mapē.

Izpildes soļi:

  1. Izveido funkciju save_text(String content).
  2. Atver fails ar FileAccess::open("user://test.txt", FileAccess::WRITE).
  3. Pārbaudi file.is_valid().
  4. Saglabā un aizver.
  5. Pārliecinies, ka fails parādās lietotāja datos: Linux ~/.local/share/godot/app_userdata/...

Aktivitāte 2: Lasīšana un parse

Atjauno datus no faila.

Izpildes soļi:

  1. Funkcija load_text() -> String: atver READ, get_as_text(), close.
  2. Pievieno failu pārbaudi: FileAccess::file_exists(...).
  3. Ja nav: atgriež default vērtību.
  4. Testē ar dažādiem failiem.

Aktivitāte 3: Bināro datu rakstīšana

Saglabā ne-tekstuālu datus.

Izpildes soļi:

  1. store_32(int), store_float(float), store_var(Variant).
  2. Lasīšana: get_32(), get_float(), get_var().
  3. Lieto bināro saglabāšanu spēles state vai score.
  4. Salīdzini failu izmēru ar tekstuālo formātu.

Papildu izaicinājums: Encryption

Šifrē saglabājumus.

Izpildes soļi:

  1. FileAccess::open_encrypted_with_pass(...) ar paroli.
  2. Aizsargā pret manipulāciju ar save failu.
  3. Tikai spēlēs, kur tas svarīgi.

Biežākās kļūdas

  • file is null: Permission denied vai path nepareizs.
  • Encoding errors: UTF-8 ir noklusējuma, bet pārbaudi ar speciāliem simboliem.
  • res:// ir read-only: Production buildos to nevar rakstīt — lieto user://.

Koda piemērs (paplašināts)

#include <godot_cpp/classes/file_access.hpp>

void SaveSystem::save_text(String filename, String content) {
    String path = "user://" + filename;
    Ref<FileAccess> file = FileAccess::open(path, FileAccess::WRITE);
    if (file.is_valid()) {
        file->store_string(content);
        file->close();
        UtilityFunctions::print("Saglabāts: ", path);
    } else {
        UtilityFunctions::printerr("Neizdevās atvērt: ", path);
    }
}

String SaveSystem::load_text(String filename) {
    String path = "user://" + filename;
    if (!FileAccess::file_exists(path)) return "";
    Ref<FileAccess> file = FileAccess::open(path, FileAccess::READ);
    if (file.is_valid()) {
        String content = file->get_as_text();
        file->close();
        return content;
    }
    return "";
}
