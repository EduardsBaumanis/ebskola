5.4 Procedural generation

Tavs šīs stundas izaicinājums: Ģenerēt nejaušus, bet sakārtotus līmeņus ar algoritmu.

SR 2.4.19. Algoritmu efektivitāte SR 2.4.16. Datu struktūras

Teorija: Procedural generation pamati

Procedural generation ir tehnika, kas algoritmiski rada saturu (līmeņi, dungeons, pasaules) bez manuālas dizainas. Klasiskais pieteikums — roguelikes.

Pamata pieejas:

  • Random Walk — sāc no centra, ej nejaušā virzienā.
  • BSP (Binary Space Partition) — dali telpu pa pusēm, savieno.
  • Cellular Automata — sākumā nejauši, tad smooth ar reglām.
// Random walk dungeon
class DungeonGenerator {
private:
    static const int WIDTH = 50;
    static const int HEIGHT = 30;
    int grid[WIDTH][HEIGHT];

public:
    void generate(int seed) {
        std::srand(seed);

        // Sāc visu kā sienas
        for (int x = 0; x < WIDTH; x++)
            for (int y = 0; y < HEIGHT; y++)
                grid[x][y] = WALL;

        // Random walk
        int x = WIDTH / 2, y = HEIGHT / 2;
        for (int step = 0; step < 1000; step++) {
            grid[x][y] = FLOOR;
            int dir = std::rand() % 4;
            if (dir == 0 && x > 1) x--;
            else if (dir == 1 && x < WIDTH-2) x++;
            else if (dir == 2 && y > 1) y--;
            else if (dir == 3 && y < HEIGHT-2) y++;
        }
    }
};

Konsolidētais praktiskais uzdevums (15+ soļi)

Izpildi šo darba plānu, lai 5.4 Procedural generation rezultāts būtu pārbaudāms C++ GDExtension projektā.

Izpildes soļi:

  1. Atver projektu un izveido Git branch datu pastāvības vai algoritmu darbam.
  2. Pieraksti, kādi dati jāsaglabā, jāielādē vai jāģenerē šajā stundā.
  3. Izveido C++ klasi, kas atbild par šo datu plūsmu, piemēram, SaveManager vai Generator.
  4. Definē datu modeli ar skaidriem tipiem: int, float, String, Dictionary vai pielāgotu struct.
  5. Pievieno versijas lauku, ja dati tiks saglabāti failā.
  6. Rakstīšanas metodē pārbaudi, vai fails tiešām ir atvērts rakstīšanai.
  7. Lasīšanas metodē pārbaudi, vai fails eksistē un nav tukšs.
  8. Validē ielādētos laukus pirms to izmantošanas spēles stāvoklī.
  9. Kļūdas gadījumā atgriez drošu noklusējuma vērtību, nevis ļauj spēlei sabrukt.
  10. Ja tēma ir algoritms, pieraksti aptuveno sarežģītību, piemēram, O(n) vai O(n log n).
  11. Pievieno mazu testdatu piemēru, kura rezultātu vari pārbaudīt ar roku.
  12. Pievieno lielāku testu, lai pamanītu veiktspējas vai atmiņas problēmas.
  13. Pievieno UtilityFunctions::print izvadi tikai svarīgiem debug punktiem.
  14. Pārkompilē C++ kodu ar SCons un izlabo kompilācijas kļūdas.
  15. Palaid Godot un testē saglabāšanu, ielādi vai ģenerāciju vismaz divas reizes pēc kārtas.
  16. README pieraksti failu ceļus, datu formātu un zināmos ierobežojumus.
  17. Pārbaudi, ka ģenerētie vai saglabātie faili nav nejauši pievienoti Git, ja tie ir lokāli dati.
  18. Veic commit ar ziņu, kas nosauc saglabāšanas, ielādes vai algoritma uzlabojumu.

Aktivitāte 1: Random walk dungeon

Pamata pieeja.

Izpildes soļi:

  1. Implementē DungeonGenerator no koda piemēra.
  2. TileMap node priekš vizualizācijas.
  3. Pēc generate() iziet caur grid un set_cell katram (x, y).
  4. Pievieno spawn point (player) un finish point (eja uz nākamo līmeni).
  5. Ar dažādiem seed iegūsi atšķirīgus dungeons.

Aktivitāte 2: Rooms ar koridoriem

Sarežģītāks ģenerātors.

Izpildes soļi:

  1. Vector<Rect2> rooms — 5-10 nejaušas istabas.
  2. Pārbaudi, ka tās nepārklājas.
  3. Savieno tās ar L-shaped koridoriem (Manhattan path).
  4. Vizuāli ievērojami labāks dungeon.

Aktivitāte 3: Loot un ienaidnieku izvietošana

Saturs dungeonā.

Izpildes soļi:

  1. Pēc dungeon ģenerēšanas — atrod random floor tiles.
  2. Spawn enemies ar varbūtību (10% katram tile).
  3. Spawn chests ar loot.
  4. Boss istabā (lielākajā) — boss enemy.

Papildu izaicinājums: Wave Function Collapse

Advanced algoritms.

Izpildes soļi:

  1. WFC — moderna PCG tehnika.
  2. Iemēģini WFC-Tile bibliotēku.
  3. Rezultāts: dungeon, kas izskatās dabiski.

Biežākās kļūdas

  • Always same dungeon: Aizmirsts seed vai srand izsaukums.
  • Walls disconnected: Pārbaudi flood fill — vai visas istabas savienotas.
  • Slow generation: Lielam grid lieto efficient algorithms — A* vai Voronoi.

Godot ekrānuzņēmumi

Koda piemērs (paplašināts)

class DungeonGenerator {
private:
    static const int WIDTH = 50;
    static const int HEIGHT = 30;
    int grid[50][30];

public:
    void generate(int seed) {
        std::srand(seed);
        // Visi sienas
        for (int x = 0; x < WIDTH; x++)
            for (int y = 0; y < HEIGHT; y++)
                grid[x][y] = 1;  // siena

        // Random walk
        int x = WIDTH / 2, y = HEIGHT / 2;
        for (int step = 0; step < 800; step++) {
            grid[x][y] = 0;  // grīda
            int dir = std::rand() % 4;
            if (dir == 0 && x > 1) x--;
            else if (dir == 1 && x < WIDTH-2) x++;
            else if (dir == 2 && y > 1) y--;
            else if (dir == 3 && y < HEIGHT-2) y++;
        }
    }

    void render_to_tilemap(TileMap* tm) {
        for (int x = 0; x < WIDTH; x++) {
            for (int y = 0; y < HEIGHT; y++) {
                int tile_id = grid[x][y];
                tm->set_cell(0, Vector2i(x, y), tile_id, Vector2i(0, 0));
            }
        }
    }
};
Katrs spēles palaišanas reize ģenerē atšķirīgu dungeon: dažādas istabas, koridori un loot izvietojums.
