5.2 JSON serializācija un Resource sistēma

Tavs šīs stundas izaicinājums: Saglabāt sarežģītus datu struktūras, izmantojot JSON un Godot Resource klases.

SR 2.4.17. Failu SR 2.4.16. Datu struktūras

Teorija: JSON un Resources

JSON (JavaScript Object Notation) ir cilvēkam un mašīnai lasāms datu formāts. Godot piedāvā JSON klasi tā parsēšanai.

#include <godot_cpp/classes/json.hpp>

// Dictionary -> JSON string
Dictionary save_data;
save_data["player_name"] = "Anna";
save_data["level"] = 5;
save_data["inventory"] = Array(); // var saturēt vēl Dictionary

String json_str = JSON::stringify(save_data, "  ");  // 2-space indent

// JSON string -> Dictionary
Ref<JSON> json = memnew(JSON);
Error err = json->parse(json_str);
if (err == OK) {
    Variant data = json->get_data();
    Dictionary loaded = data;
    String name = loaded["player_name"];
}

Resource ir Godot iebūvēta klase resursu (audio, attēli, custom data) saglabāšanai .tres failos.

1. uzdevums: Save JSON sistēma

Spēles state JSON formātā.

Izpildes soļi:

  1. Player klases metode Dictionary to_dict() const: hp, position, level, inventory.
  2. Funkcija save_to_file(): izveido Dict, JSON::stringify, FileAccess WRITE.
  3. Funkcija load_from_file(): FileAccess READ, JSON::parse, atjauno Player state.
  4. Pārbaudi: nospied F5 ar saglabātu state, restart, load — visi dati atjaunoti.

2. uzdevums: Settings sistēma

Lietotāja iestatījumi.

Izpildes soļi:

  1. Ekrāna izšķirtspēja, audio volumes, key bindings.
  2. Saglabā JSON kā user://settings.json.
  3. Ielādē pie spēles starta.
  4. UI panel ar saglabā/atjauno pogām.

3. uzdevums: Custom Resource klase

ItemResource priekš inventāra.

Izpildes soļi:

  1. class ItemResource : public Resource.
  2. Lauki: name, icon (Texture2D), max_stack, value.
  3. Saglabā kā .tres failu: res://items/sword.tres.
  4. Inventārs glabā std::vector<Ref<ItemResource>>.

Papildus uzdevums: Versioning

Save fails atpakaļ-saderīgs.

Izpildes soļi:

  1. Katrā save iekļauj version: 2.
  2. Load: ja vecāka versija — migrē uz jauno.
  3. Saglabā veco save kā backup.

Biežākās kļūdas

  • Crash, parse JSON: Tukšs vai bojāts fails. Pārbaudi err == OK vienmēr.
  • Dictionary keys nav konsistenti: Lieto strukturas vai konstantes (kods centra).
  • Resource fails neielādējas: Path nepareizs vai resurss bojāts.

Godot ekrānuzņēmumi

Koda piemērs (paplašināts)

Dictionary Player::to_dict() const {
    Dictionary d;
    d["name"] = name;
    d["hp"] = hp;
    d["max_hp"] = max_hp;
    d["level"] = level;
    d["xp"] = xp;
    d["position_x"] = get_position().x;
    d["position_y"] = get_position().y;

    Array inv;
    for (const auto& [item, count] : inventory) {
        Dictionary entry;
        entry["item"] = item;
        entry["count"] = count;
        inv.append(entry);
    }
    d["inventory"] = inv;
    return d;
}

void Player::from_dict(const Dictionary& d) {
    name = d["name"];
    hp = d["hp"];
    max_hp = d["max_hp"];
    level = d["level"];
    xp = d["xp"];
    set_position(Vector2(d["position_x"], d["position_y"]));

    inventory.clear();
    Array inv = d["inventory"];
    for (int i = 0; i < inv.size(); i++) {
        Dictionary e = inv[i];
        inventory[e["item"]] = e["count"];
    }
}
