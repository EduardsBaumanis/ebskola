2.4 Sadursmju noteikšana

Tavs šīs stundas izaicinājums: Apgūt CharacterBody2D, Area2D un RigidBody2D atšķirības un to lietojumu.

SR 2.4.14. Vadības struktūras SR 2.4.7. UI

Priekšzināšanu robeža: uzdevumos izmanto tikai iepriekš apgūto un šīs lapas teorijas/koda piemēros parādīto. Ja parādās jauna komanda vai rīks, vispirms atrod tās paraugu teorijas blokā.

Teorija: Sadursmju nodes Godot

Godot piedāvā trīs galvenos sadursmju nodes:

NodeLietojums
StaticBody2DNekustīgi šķēršļi (siena, grīda).
CharacterBody2DSpēlētāji un AI ar pielāgotu kustību (ne fiziku).
RigidBody2DPilna fizika — gravitācija, masa, spēks.
Area2DDetektē ieiešanu/iziešanu zonā (trigger, pickup).

Collision Layers un Masks: nosaka, kas var sadurties ar ko.

// Area2D signāls C++
void Pickup::_ready() {
    connect("body_entered", Callable(this, "on_body_entered"));
}

void Pickup::on_body_entered(Node2D *body) {
    if (body->is_in_group("player")) {
        // savākt pickup
        get_parent()->call("score_up", 1);
        queue_free();  // dzēst sevi
    }
}

Teorijas paplašinājums: C++ fokuss

Kritiskais fokuss: šajā stundā nepietiek tikai panākt redzamu efektu editorā. Jāspēj paskaidrot, kura C++ klase glabā stāvokli, kura metode to maina un kā Godot node struktūra šo kodu izmanto.

Pārbaudes pierādījums: spēlētājs, objekti un sadursmes darbojas ar C++ klasēm, kas lieto _physics_process, Input, CharacterBody2D, Area2D vai move_and_slide.

Konsolidētais praktiskais uzdevums (15+ soļi)

70 min darba sadalījums: 1. uzdevums (~20 min) — izveido minimālo piemēru, kas pārbauda šīs stundas galveno ideju; 2. uzdevums (~25 min) — integrē risinājumu tēmas spēles projektā; 3. uzdevums (~25 min) — notestē robežgadījumus, salabo kļūdas un pieraksti secinājumu README vai komentārā. Tālāk redzamie soļi ir detalizēta karte šiem trim uzdevumiem; ja laiks beidzas, papildu pulēšanu atstāj nākamajai reizei.

Izpildi šo darba plānu, lai 2.4 Sadursmju noteikšana rezultāts būtu pārbaudāms C++ GDExtension projektā.

Izpildes soļi:

  1. Atver esošo Godot projektu un izveido jaunu Git branch šīs stundas darbam.
  2. Pārbaudi, ka spēles loģika tiek rakstīta C++ GDExtension klasē.
  3. Pieraksti, kurš node tips šajā stundā ir galvenais: Node2D, CharacterBody2D, Area2D vai RigidBody2D.
  4. Izveido vai atjaunini atbilstošu .hpp failu ar klases deklarāciju.
  5. Izveido vai atjaunini atbilstošu .cpp failu ar metožu implementāciju.
  6. Pievieno nepieciešamos #include failus no godot_cpp.
  7. Pievieno GDCLASS un tukšu vai vajadzīgu _bind_methods() metodi.
  8. Reģistrē klasi register_types.cpp ar ClassDB::register_class.
  9. Kustībai lieto _physics_process(double delta), nevis tikai manuālu pozīcijas maiņu editorā.
  10. Ja maini pozīciju, reizinājumā iekļauj delta, lai ātrums nebūtu atkarīgs no FPS.
  11. Ja objekts saduras ar vidi, izvēlies move_and_slide() vai move_and_collide() un pamato izvēli.
  12. Iestati CollisionShape2D izmēru tā, lai tas sakristu ar redzamo objektu.
  13. Pārbaudi collision layer/mask, lai objekts reaģē tikai uz vajadzīgajiem slāņiem.
  14. Pievieno īsu UtilityFunctions::print pārbaudi, kas parāda svarīgu stāvokli spēles laikā.
  15. Pārkompilē ar SCons un pārliecinies, ka Godot ielādē jauno bibliotēku bez kļūdām.
  16. Testē vienu normālu gadījumu un vienu robežgadījumu, piemēram, ekrāna malu vai platformas malu.
  17. README pieraksti, kāda fizikas vai sadursmju problēma tika atrisināta.
  18. Veic commit ar skaidru ziņu par šīs stundas mehāniku.

Aktivitāte 1: Platformas un sienas

Izveido statiskus šķēršļus.

Izpildes soļi:

  1. Pievieno StaticBody2D nodes platformām: Floor, LeftWall, RightWall.
  2. Katram CollisionShape2D ar atbilstošu RectangleShape2D.
  3. Krāsa: pievieno ColorRect kā vizuālo elementu.
  4. Spēlētājs CharacterBody2D ar move_and_slide() automātiski apstājas pie tām.

Aktivitāte 2: Pickup ar Area2D

Savācami priekšmeti.

Izpildes soļi:

  1. Pievieno Area2D jaunā scēnā Coin.tscn.
  2. Bērni: Sprite2D (zelta krāsā) un CollisionShape2D.
  3. C++ klase Coin : public Area2D ar signal connection.
  4. on_body_entered(Node2D *body): ja body grupā "player", queue_free() un score++.
  5. Izvieto vairākas Coin instances scēnā.

Aktivitāte 3: RigidBody2D bumbiņa

Fizikas demo ar gravitāciju.

Izpildes soļi:

  1. Pievieno RigidBody2D ar nosaukumu Ball.
  2. Sprite2D, CollisionShape2D bērni.
  3. Inspector → Mass: 1.0; Gravity Scale: 1.0.
  4. Palaiž — bumba krīt un atlec no platformām.
  5. Eksperimentē ar masu un mode (Static, Kinematic, Rigid).

Papildu izaicinājums: Collision Layers

Konfigurē, kas saduras ar ko.

Izpildes soļi:

  1. Project Settings → Layer Names → 2D Physics — pārsauc Layer 1 par 'world', 2 par 'player', 3 par 'pickup'.
  2. Spēlētājs: Collision Layer = player; Mask = world, pickup.
  3. Pickup: Layer = pickup; Mask = player.
  4. Tas atļauj tikai konkrētus saduršanās scenārijus.

Biežākās kļūdas

  • Spēlētājs iziet cauri grīdai: CollisionShape2D nav vai forma 0 izmērā.
  • Area2D signal nestrādā: Aizmirsts connect() vai signal nepareizi nosaukts.
  • RigidBody2D negravitē: Gravity Scale = 0 vai Project Settings → Physics → 2D → Default Gravity = 0.

Godot ekrānuzņēmumi

Koda piemērs (paplašināts)

#include "coin.hpp"
#include <godot_cpp/core/class_db.hpp>
#include <godot_cpp/variant/callable.hpp>

using namespace godot;

void Coin::_bind_methods() {
    ClassDB::bind_method(D_METHOD("on_body_entered", "body"),
                         &Coin::on_body_entered);
}

void Coin::_ready() {
    connect("body_entered", Callable(this, "on_body_entered"));
}

void Coin::on_body_entered(Node2D *body) {
    if (body->is_in_group("player")) {
        // Pavadi ziņu vecākam ar score
        get_tree()->call_group("game", "add_score", 1);
        queue_free();
    }
}
Spēlētājs pārvietojas pa platformām, savācot zelta monētas, kuras pazūd pēc savākšanas.
