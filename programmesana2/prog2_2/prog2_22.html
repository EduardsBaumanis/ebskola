2.2 Input apstrāde un Input Map

Tavs šīs stundas izaicinājums: Konfigurēt Input Map un izveidot WASD/bultiņu vadāmu spēlētāju.

SR 2.4.14. Vadības struktūras SR 2.4.7. UI

Priekšzināšanu robeža: uzdevumos izmanto tikai iepriekš apgūto un šīs lapas teorijas/koda piemēros parādīto. Ja parādās jauna komanda vai rīks, vispirms atrod tās paraugu teorijas blokā.

Teorija: Input pārvaldība Godot

Godot piedāvā divus veidus apstrādāt input:

  • Polling (aptauja) — Input::is_action_pressed("move_right") katrā kadrā pārbauda stāvokli.
  • Events (notikumi) — _input(Ref<InputEvent> event) reaģē uz konkrētu notikumu.

Spēļu kustībai parasti lieto polling (jo notiek katrā kadrā).

#include <godot_cpp/classes/input.hpp>

void Player::_physics_process(double delta) {
    Input *input = Input::get_singleton();
    Vector2 movement = Vector2(0, 0);

    if (input->is_action_pressed("move_right")) movement.x += 1;
    if (input->is_action_pressed("move_left"))  movement.x -= 1;
    if (input->is_action_pressed("move_up"))    movement.y -= 1;
    if (input->is_action_pressed("move_down"))  movement.y += 1;

    movement = movement.normalized();  // diagonāles ātrums = taisns ātrums
    velocity = movement * speed;
    move_and_slide();
}

Input Map definē abstraktas darbības (move_right) un saista tās ar konkrētiem taustiņiem. Project Settings → Input Map.

Teorijas paplašinājums: C++ fokuss

Kritiskais fokuss: šajā stundā nepietiek tikai panākt redzamu efektu editorā. Jāspēj paskaidrot, kura C++ klase glabā stāvokli, kura metode to maina un kā Godot node struktūra šo kodu izmanto.

Pārbaudes pierādījums: spēlētājs, objekti un sadursmes darbojas ar C++ klasēm, kas lieto _physics_process, Input, CharacterBody2D, Area2D vai move_and_slide.

Konsolidētais praktiskais uzdevums (15+ soļi)

70 min darba sadalījums: 1. uzdevums (~20 min) — izveido minimālo piemēru, kas pārbauda šīs stundas galveno ideju; 2. uzdevums (~25 min) — integrē risinājumu tēmas spēles projektā; 3. uzdevums (~25 min) — notestē robežgadījumus, salabo kļūdas un pieraksti secinājumu README vai komentārā. Tālāk redzamie soļi ir detalizēta karte šiem trim uzdevumiem; ja laiks beidzas, papildu pulēšanu atstāj nākamajai reizei.

Izpildi šo darba plānu, lai 2.2 Input apstrāde un Input Map rezultāts būtu pārbaudāms C++ GDExtension projektā.

Izpildes soļi:

  1. Atver esošo Godot projektu un izveido jaunu Git branch šīs stundas darbam.
  2. Pārbaudi, ka spēles loģika tiek rakstīta C++ GDExtension klasē.
  3. Pieraksti, kurš node tips šajā stundā ir galvenais: Node2D, CharacterBody2D, Area2D vai RigidBody2D.
  4. Izveido vai atjaunini atbilstošu .hpp failu ar klases deklarāciju.
  5. Izveido vai atjaunini atbilstošu .cpp failu ar metožu implementāciju.
  6. Pievieno nepieciešamos #include failus no godot_cpp.
  7. Pievieno GDCLASS un tukšu vai vajadzīgu _bind_methods() metodi.
  8. Reģistrē klasi register_types.cpp ar ClassDB::register_class.
  9. Kustībai lieto _physics_process(double delta), nevis tikai manuālu pozīcijas maiņu editorā.
  10. Ja maini pozīciju, reizinājumā iekļauj delta, lai ātrums nebūtu atkarīgs no FPS.
  11. Ja objekts saduras ar vidi, izvēlies move_and_slide() vai move_and_collide() un pamato izvēli.
  12. Iestati CollisionShape2D izmēru tā, lai tas sakristu ar redzamo objektu.
  13. Pārbaudi collision layer/mask, lai objekts reaģē tikai uz vajadzīgajiem slāņiem.
  14. Pievieno īsu UtilityFunctions::print pārbaudi, kas parāda svarīgu stāvokli spēles laikā.
  15. Pārkompilē ar SCons un pārliecinies, ka Godot ielādē jauno bibliotēku bez kļūdām.
  16. Testē vienu normālu gadījumu un vienu robežgadījumu, piemēram, ekrāna malu vai platformas malu.
  17. README pieraksti, kāda fizikas vai sadursmju problēma tika atrisināta.
  18. Veic commit ar skaidru ziņu par šīs stundas mehāniku.

Aktivitāte 1: Input Map konfigurēšana

Iestati spēlētāja darbības.

Izpildes soļi:

  1. Project → Project Settings → Input Map.
  2. Pievieno darbības: move_right, move_left, jump, attack.
  3. Pievieno taustiņus: move_right=D un Right Arrow, move_left=A un Left Arrow, jump=Space, attack=X.
  4. Saglabā un ekspretē uz failu, lai būtu Git-friendly.

Aktivitāte 2: Player klase ar kustību

Izveido pamata spēlētāju.

Izpildes soļi:

  1. Player : public CharacterBody2D. Locekļi: float speed = 300.0f.
  2. _physics_process: lasa input, aprēķina movement vector, normalizē.
  3. Lieto move_and_slide() automātiskai sadursmju apstrādei.
  4. Pievieno scēnai, izveido testa platformas (StaticBody2D ar CollisionShape).
  5. Palaiž, pārvietojies ar WASD/bultiņām.

Aktivitāte 3: Input events

Reaģē uz vienreizējiem notikumiem.

Izpildes soļi:

  1. Pievieno _input(const Ref<InputEvent> &event) override;
  2. Izsauc tikai pie taustiņa nospiešanas: if (event->is_action_pressed("jump"))
  3. Izvada "Jumped!" Output panelī.
  4. Salīdzini polling vs event pieejas.

Papildu izaicinājums: Joystick atbalsts

Pievieno spēļu kontroliera atbalstu.

Izpildes soļi:

  1. Input Map sadaļā pievieno Joypad Axis sasaistes saviem darbībām.
  2. Lieto get_action_strength, lai iegūtu analogo vērtību (0–1).
  3. Testē ar pievienotu kontrolieri.

Biežākās kļūdas

  • Spēlētājs nekustas: Input Map nav saglabāta vai darbība nepareizi nosaukta.
  • Diagonālais ātrums dubults: Aizmirsts .normalized().
  • Spēlētājs iet cauri sienām: Lieto move_and_slide(), ne set_position.
