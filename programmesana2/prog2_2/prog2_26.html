2.6 Noslēguma projekts: Platformas spēle

Tavs šīs stundas izaicinājums: Izveidot pilnvērtīgu 2D platformas spēli ar 3 līmeņiem, savācamiem un finiša bandieri.

SR 2.4.1. Problēmas dekompozīcija SR 2.4.7. UI SR 2.4.14. Vadības struktūras SR 2.4.15. Modulāras programmas

Priekšzināšanu robeža: uzdevumos izmanto tikai iepriekš apgūto un šīs lapas teorijas/koda piemēros parādīto. Ja parādās jauna komanda vai rīks, vispirms atrod tās paraugu teorijas blokā.

Teorija: Spēles apraksts un arhitektūra

Platformas spēle ar šādām funkcijām:

  • 3 līmeņi ar augošu sarežģītību.
  • Spēlētājs ar gravitāciju, jump, variable height un coyote time.
  • Savācamas monētas, dārgakmeņi un atslēgas.
  • Ienaidnieki, kas pārvietojas patrol ceļā.
  • Finiša bandieris katrā līmenī — pārejas uz nākamo.
  • HUD ar HP, score, atlikušo laiku.

Teorijas paplašinājums: C++ fokuss

Kritiskais fokuss: šajā stundā nepietiek tikai panākt redzamu efektu editorā. Jāspēj paskaidrot, kura C++ klase glabā stāvokli, kura metode to maina un kā Godot node struktūra šo kodu izmanto.

Pārbaudes pierādījums: spēlētājs, objekti un sadursmes darbojas ar C++ klasēm, kas lieto _physics_process, Input, CharacterBody2D, Area2D vai move_and_slide.

Konsolidētais praktiskais uzdevums (15+ soļi)

70 min darba sadalījums: 1. uzdevums (~20 min) — izveido minimālo piemēru, kas pārbauda šīs stundas galveno ideju; 2. uzdevums (~25 min) — integrē risinājumu tēmas spēles projektā; 3. uzdevums (~25 min) — notestē robežgadījumus, salabo kļūdas un pieraksti secinājumu README vai komentārā. Tālāk redzamie soļi ir detalizēta karte šiem trim uzdevumiem; ja laiks beidzas, papildu pulēšanu atstāj nākamajai reizei.

Izpildi šo darba plānu, lai 2.6 Noslēguma projekts: Platformas spēle rezultāts būtu pārbaudāms C++ GDExtension projektā.

Izpildes soļi:

  1. Atver esošo Godot projektu un izveido jaunu Git branch šīs stundas darbam.
  2. Pārbaudi, ka spēles loģika tiek rakstīta C++ GDExtension klasē.
  3. Pieraksti, kurš node tips šajā stundā ir galvenais: Node2D, CharacterBody2D, Area2D vai RigidBody2D.
  4. Izveido vai atjaunini atbilstošu .hpp failu ar klases deklarāciju.
  5. Izveido vai atjaunini atbilstošu .cpp failu ar metožu implementāciju.
  6. Pievieno nepieciešamos #include failus no godot_cpp.
  7. Pievieno GDCLASS un tukšu vai vajadzīgu _bind_methods() metodi.
  8. Reģistrē klasi register_types.cpp ar ClassDB::register_class.
  9. Kustībai lieto _physics_process(double delta), nevis tikai manuālu pozīcijas maiņu editorā.
  10. Ja maini pozīciju, reizinājumā iekļauj delta, lai ātrums nebūtu atkarīgs no FPS.
  11. Ja objekts saduras ar vidi, izvēlies move_and_slide() vai move_and_collide() un pamato izvēli.
  12. Iestati CollisionShape2D izmēru tā, lai tas sakristu ar redzamo objektu.
  13. Pārbaudi collision layer/mask, lai objekts reaģē tikai uz vajadzīgajiem slāņiem.
  14. Pievieno īsu UtilityFunctions::print pārbaudi, kas parāda svarīgu stāvokli spēles laikā.
  15. Pārkompilē ar SCons un pārliecinies, ka Godot ielādē jauno bibliotēku bez kļūdām.
  16. Testē vienu normālu gadījumu un vienu robežgadījumu, piemēram, ekrāna malu vai platformas malu.
  17. README pieraksti, kāda fizikas vai sadursmju problēma tika atrisināta.
  18. Veic commit ar skaidru ziņu par šīs stundas mehāniku.

Aktivitāte 1: Player un Level (1. līmenis)

Pamata struktūra.

Izpildes soļi:

  1. Pārkopē Player klasi no 2.5 stundas.
  2. Izveido 3 Level scēnas: Level1.tscn, Level2.tscn, Level3.tscn.
  3. Katrā: TileMap (vai StaticBody2D platformas), spēlētāja sākuma punkts (Marker2D), finiša bandieris (Area2D).
  4. Ielādē Level1 un pārliecinies, ka spēlētājs var izstaigāt.

Aktivitāte 2: Pickups un punkti (2. līmenis)

Savācamie objekti.

Izpildes soļi:

  1. Izveido Coin, Gem, Key scēnas (no 2.4 stundas pieredzes).
  2. Game klase glabā int score un bool has_key.
  3. HUD: pievieno Label nodes, atjaunini caur get_tree()->call_group("hud", "refresh", score).
  4. Finiša bandieris atveras tikai, ja has_key==true.
  5. Pārliecinies, ka score saglabājas, pārejot uz nākamo līmeni.

Aktivitāte 3: Ienaidnieki un līmeņu pāreja (3. līmenis)

Sarežģītība un struktūra.

Izpildes soļi:

  1. Enemy klase: patrol starp diviem punktiem (Marker2D references).
  2. Sadursme ar Player: spēlētājs zaudē HP. Ja HP==0 — restart.
  3. Game klase: load_next_level() pārslēdz scēnu ar get_tree()->change_scene_to_file().
  4. Pēc 3. līmeņa: parādi "Game complete!" un kopējos punktus.
  5. GitHub: 6+ commit, README ar control schema un scene struktūru.

Papildu izaicinājums: Bonus: Power-ups un saglabāšana

Pievieno spēles pastāvību.

Izpildes soļi:

  1. Mushroom power-up: spēlētājs kļūst lielāks (Scale 1.5x), HP+50.
  2. Saglabā augstāko score JSON failā (5. tēmā apgūsim).
  3. Pievieno checkpoint sistēmu līmeņos.

Biežākās kļūdas

  • Scēnas pāreja crash: Pārliecinies, ka mērķa scēnas ceļš ir pareizs.
  • Score reset starp līmeņiem: Game klase jābūt Autoload (Project → Autoload).
  • Patrol AI iet ārpus zonām: Pārbaudi Marker2D pozīcijas līmenī.

Godot ekrānuzņēmumi

Koda piemērs (paplašināts)

class Game : public Node {
    GDCLASS(Game, Node)
private:
    int score = 0;
    int current_level = 1;
public:
    void load_next_level() {
        current_level++;
        if (current_level > 3) {
            UtilityFunctions::print("Game complete! Score: ", score);
            get_tree()->quit();
            return;
        }
        String scene_path = "res://levels/Level" + String::num(current_level) + ".tscn";
        get_tree()->change_scene_to_file(scene_path);
    }
    void add_score(int amount) { score += amount; }
};
