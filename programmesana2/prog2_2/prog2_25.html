2.5 Gravitācija un platformas mehānika

Tavs šīs stundas izaicinājums: Implementēt platformas spēles tipisko jump+gravitāciju mehāniku.

SR 2.4.14. Vadības struktūras SR 2.4.13. Datu tipi

Pirms sāc: izmanto iepriekš apgūto un šīs lapas teorijas/koda piemērus. Ja vajadzīga jauna komanda vai rīks, vispirms atrodi tās paraugu teorijas sadaļā.

Teorija: Gravitācija un jump mehānika

Klasiska 2D platformas mehānika ietver:

  • Gravitāciju — y velocity pieaug katrā kadrā.
  • Jump — viena reize y velocity tiek piešķirta negatīva vērtība.
  • Ground check — pārbaude, vai spēlētājs ir uz zemes.
  • Coyote time — neliela tolerance, ja spēlētājs nupat nokrita no malas.
class Player : public CharacterBody2D {
private:
    const float SPEED = 300.0f;
    const float JUMP_FORCE = -550.0f;  // negatīvs = uz augšu
    const float GRAVITY = 1500.0f;
    Vector2 velocity_internal;

public:
    void _physics_process(double delta) override {
        // Gravitācija
        if (!is_on_floor()) {
            velocity_internal.y += GRAVITY * delta;
        }

        // Jump
        Input *input = Input::get_singleton();
        if (input->is_action_just_pressed("jump") && is_on_floor()) {
            velocity_internal.y = JUMP_FORCE;
        }

        // Horizontālā kustība
        float dir = input->get_axis("move_left", "move_right");
        velocity_internal.x = dir * SPEED;

        set_velocity(velocity_internal);
        move_and_slide();
        velocity_internal = get_velocity();
    }
};

Praktiskie uzdevumi

1. uzdevums — Izpēti pamatus

Atpazīsti šīs stundas galveno ideju un sasaisti to ar gala projektu Platformas spēle.

  1. Izlasi stundas izaicinājumu un ar vienu teikumu pieraksti, ko šodien mācīsies par tēmas projektu Platformas spēle.
  2. Atrodi teorijas sadaļā vienu piemēru, kas izmanto Godot 4 un C++ GDExtension, un pieraksti trīs svarīgākās darbības.
  3. Pieraksti, kurā gala projekta daļā šī prasme būs vajadzīga: sākuma ekrānā, spēles laukumā, datos, vadībā, rezultātā vai publicēšanā.
  4. Izveido nelielu izmēģinājuma fragmentu .hpp, .cpp vai Godot scēnas failā, nevis uzreiz lielu funkciju.

  6. Nomaini piemēra mainīgos uz klases situācijām, piemēram, klases_punkti, zvana_taimeris, pazudusais_markieris vai kafijas_pauze; mazliet humora drīkst, bet kodam jāpaliek skaidram.
  7. Pievieno vienu komentāru, kas paskaidro, kāpēc šī darbība nepieciešama gala spēles projektam.
  8. Pārkompilē C++ un palaid projektu Godot editorā, lai redzētu, vai minimālais piemērs darbojas.
  9. Nomaini vienu ievades vērtību vai tekstu un pārbaudi, vai rezultāts mainās paredzami.
  10. Pieraksti vienu secinājumu: kas šajā stundā ir jaunais jēdziens un kā tas palīdzēs gala spēlei.

  6. Pārbaudi, ka risinājums netraucē iepriekšējām stundām izveidoto funkcionalitāti.
  7. Pievieno vienu lietotājam saprotamu tekstu vai izvadi, kas parāda, ka jaunā daļa strādā.
  8. Pārkompilē C++ un palaid projektu Godot editorā.
  9. Salabo pirmo kļūdu, kuru pamani, pirms pievieno nākamo uzlabojumu.
  10. Veic Git commit ar īsu tehnisku ziņu, kurā minēts, kā šī prasme tika pielietota gala projektā.

Biežākās kļūdas

  • Spēlētājs lec dubultjump: Aizmirsts is_on_floor() pārbaude.
  • Gravitācija pārāk ātra/lēna: Pielāgo GRAVITY konstanti (1000–2500 ir parastais).
  • Spēlētājs uzkaras ar sienu: Pievieno horizontālu velocity slāpēšanu, kad pie sienas.

void Player::_physics_process(double delta) {
    Input *input = Input::get_singleton();

    // Gravitācija
    if (!is_on_floor()) {
        velocity_internal.y += GRAVITY * delta;
    }

    // Jump ar variable height
    if (input->is_action_just_pressed("jump") && is_on_floor()) {
        velocity_internal.y = JUMP_FORCE;
    }
    if (input->is_action_just_released("jump") && velocity_internal.y < 0) {
        velocity_internal.y *= 0.5f;
    }

    // Horizontāla
    float dir = input->get_axis("move_left", "move_right");
    velocity_internal.x = dir * SPEED;

    set_velocity(velocity_internal);
    move_and_slide();
    velocity_internal = get_velocity();
}
