2.5 Gravitācija un platformas mehānika

Tavs šīs stundas izaicinājums: Implementēt platformas spēles tipisko jump+gravitāciju mehāniku.

SR 2.4.14. Vadības struktūras SR 2.4.13. Datu tipi

Priekšzināšanu robeža: uzdevumos izmanto tikai iepriekš apgūto un šīs lapas teorijas/koda piemēros parādīto. Ja parādās jauna komanda vai rīks, vispirms atrod tās paraugu teorijas blokā.

Teorija: Gravitācija un jump mehānika

Klasiska 2D platformas mehānika ietver:

  • Gravitāciju — y velocity pieaug katrā kadrā.
  • Jump — viena reize y velocity tiek piešķirta negatīva vērtība.
  • Ground check — pārbaude, vai spēlētājs ir uz zemes.
  • Coyote time — neliela tolerance, ja spēlētājs nupat nokrita no malas.
class Player : public CharacterBody2D {
private:
    const float SPEED = 300.0f;
    const float JUMP_FORCE = -550.0f;  // negatīvs = uz augšu
    const float GRAVITY = 1500.0f;
    Vector2 velocity_internal;

public:
    void _physics_process(double delta) override {
        // Gravitācija
        if (!is_on_floor()) {
            velocity_internal.y += GRAVITY * delta;
        }

        // Jump
        Input *input = Input::get_singleton();
        if (input->is_action_just_pressed("jump") && is_on_floor()) {
            velocity_internal.y = JUMP_FORCE;
        }

        // Horizontālā kustība
        float dir = input->get_axis("move_left", "move_right");
        velocity_internal.x = dir * SPEED;

        set_velocity(velocity_internal);
        move_and_slide();
        velocity_internal = get_velocity();
    }
};

Aktivitāte 1: Pamata jump+gravitācija

Implementē klasisko mehānika.

Izpildes soļi:

  1. Player klasē pievieno gravitāciju, jump force konstantes.
  2. _physics_process: pievieno y velocity gravitāciju, ja ne uz grīdas.
  3. is_action_just_pressed("jump") + is_on_floor() = lec.
  4. Saglabā velocity starp kadriem (velocity_internal instances mainīgajā).
  5. Testē uz vienkārša līmeņa ar 3 platformām.

Aktivitāte 2: Variable jump height

Lec augstāk, ja taustiņš tur garāk.

Izpildes soļi:

  1. Pievieno is_action_just_released("jump") pārbaudi.
  2. Ja atlaiž jump taustiņu un velocity.y < 0: velocity.y *= 0.5f;
  3. Tas dod kontroli pār lec augstumu — īsa nospiešana = mazs lec, ilga = liels.
  4. Iegūsts atsaucīgāks spēles juties.

Aktivitāte 3: Coyote time un jump buffer

Lietotāja draudzīgi uzlabojumi.

Izpildes soļi:

  1. Pievieno float coyote_timer = 0, float COYOTE_TIME = 0.1f.
  2. Ja was on floor pagājušajā kadrā un nav tagad — sāc skaitīt coyote_timer.
  3. Ja jump tiek nospiests un (is_on_floor || coyote_timer > 0): atļauj lec.
  4. Tāpat jump_buffer: ja jump nospiests īsi pirms grīdas — atceries un lec, kad nokļūst.
  5. Šie uzlabojumi padara platformeri profesionāli labu.

Papildu izaicinājums: Wall jump

Lec no vertikālas sienas.

Izpildes soļi:

  1. Pievieno is_on_wall() pārbaudi.
  2. Ja lec, atrodoties pie sienas: velocity.y = JUMP_FORCE; velocity.x = -wall_normal * 200.
  3. Iegūsi Celeste-style mehāniku.

Biežākās kļūdas

  • Spēlētājs lec dubultjump: Aizmirsts is_on_floor() pārbaude.
  • Gravitācija pārāk ātra/lēna: Pielāgo GRAVITY konstanti (1000–2500 ir parastais).
  • Spēlētājs uzkaras ar sienu: Pievieno horizontālu velocity slāpēšanu, kad pie sienas.

Godot ekrānuzņēmumi

Koda piemērs (paplašināts)

void Player::_physics_process(double delta) {
    Input *input = Input::get_singleton();

    // Gravitācija
    if (!is_on_floor()) {
        velocity_internal.y += GRAVITY * delta;
    }

    // Jump ar variable height
    if (input->is_action_just_pressed("jump") && is_on_floor()) {
        velocity_internal.y = JUMP_FORCE;
    }
    if (input->is_action_just_released("jump") && velocity_internal.y < 0) {
        velocity_internal.y *= 0.5f;
    }

    // Horizontāla
    float dir = input->get_axis("move_left", "move_right");
    velocity_internal.x = dir * SPEED;

    set_velocity(velocity_internal);
    move_and_slide();
    velocity_internal = get_velocity();
}
