4.4 Pathfinding ar A*

Tavs šīs stundas izaicinājums: Saprast un implementēt pathfinding (ceļa meklēšana) algoritmu spēles AI.

SR 2.4.19. Algoritmu efektivitāte SR 2.4.16. Datu struktūras

Priekšzināšanu robeža: uzdevumos izmanto tikai iepriekš apgūto un šīs lapas teorijas/koda piemēros parādīto. Ja parādās jauna komanda vai rīks, vispirms atrod tās paraugu teorijas blokā.

Teorija: A* algoritms un NavigationAgent2D

Reālistiska AI prasa pathfinding — meklēt īsāko ceļu starp diviem punktiem, izvairoties no šķēršļiem.

A* algoritms izmanto:

  • g(n) — faktiskais ceļa garums no sākuma līdz n.
  • h(n) — heuristika (taisns attālums līdz mērķim).
  • f(n) = g(n) + h(n) — kopējais novērtējums.

Godot 4 piedāvā NavigationAgent2D kas to dara automātiski.

#include <godot_cpp/classes/navigation_agent2d.hpp>

class Enemy : public CharacterBody2D {
    NavigationAgent2D *nav_agent;
public:
    void _ready() override {
        nav_agent = get_node<NavigationAgent2D>("NavigationAgent2D");
        nav_agent->set_target_position(player_pos);
    }

    void _physics_process(double delta) override {
        if (nav_agent->is_navigation_finished()) return;

        Vector2 next_pos = nav_agent->get_next_path_position();
        Vector2 dir = (next_pos - get_position()).normalized();
        velocity = dir * speed;
        move_and_slide();
    }
};

Konsolidētais praktiskais uzdevums (15+ soļi)

70 min darba sadalījums: 1. uzdevums (~20 min) — izveido minimālo piemēru, kas pārbauda šīs stundas galveno ideju; 2. uzdevums (~25 min) — integrē risinājumu tēmas spēles projektā; 3. uzdevums (~25 min) — notestē robežgadījumus, salabo kļūdas un pieraksti secinājumu README vai komentārā.   6. Pievieno publiskas metodes datu pievienošanai, nolasīšanai un drošai dzēšanai.

Aktivitāte 1: NavigationRegion2D iestatīšana

Sagatavo nav mesh.

Izpildes soļi:

  1. Pievieno NavigationRegion2D līmeņa scēnai.
  2. Pievieno NavigationPolygon resursu.
  3. Tilbeing Bake NavigationPolygon — Godot ģenerē navigējamo zonu.
  4. Pārliecinies, ka šķēršļi (StaticBody2D) tiek izslēgti no nav mesh.

Aktivitāte 2: NavigationAgent2D Enemy

Pievieno pathfinding ienaidniekam.

Izpildes soļi:

  1. Pievieno NavigationAgent2D kā Enemy bērnu.
  2. Inspector: max_speed, path_desired_distance, target_desired_distance.
  3. Enemy klases _ready: nav_agent = get_node<...>(...);
  4. _physics_process: nav_agent->set_target_position(player_pos); izmanto next_path_position.

Aktivitāte 3: Vairāki ienaidnieki nemudžojas

Avoidance konfigurācija.

Izpildes soļi:

  1. NavigationAgent2D: avoidance_enabled = true.
  2. Pievieno NavigationObstacle2D ap katru ienaidnieku.
  3. Tagad ienaidnieki automātiski izvairās viens no otra.
  4. Eksperimentē ar dažādiem max_speed un radius.

Papildu izaicinājums: Custom A* implementation

Manuāli ieprogrammē A*.

Izpildes soļi:

  1. Godot AStar2D klase pieejama C++.
  2. Izveido grid no scenes, marķē šķēršļus.
  3. Implementē find_path metodi.
  4. Salīdzini ātrumu ar NavigationAgent2D.

Biežākās kļūdas

  • NavigationAgent neatrod ceļu: NavigationPolygon nav baked vai nepareiza forma.
  • Ienaidnieki iestrēgst stūros: Path radius pārāk mazs vai šķēršļi pārāk tuvu.
  • Slow performance: Pārāk daudz NavigationAgent — pārbaudi update freq.

Godot ekrānuzņēmumi

Koda piemērs (paplašināts)

#include <godot_cpp/classes/navigation_agent2d.hpp>
#include <godot_cpp/classes/character_body2d.hpp>

void Enemy::_ready() {
    nav_agent = get_node<NavigationAgent2D>("NavigationAgent2D");
    nav_agent->set_avoidance_enabled(true);
    update_target();
}

void Enemy::_physics_process(double delta) {
    if (nav_agent->is_navigation_finished()) return;

    Vector2 current = get_position();
    Vector2 next = nav_agent->get_next_path_position();
    Vector2 direction = (next - current).normalized();

    velocity = direction * speed;
    move_and_slide();
}

void Enemy::update_target() {
    if (player) nav_agent->set_target_position(player->get_position());
}
