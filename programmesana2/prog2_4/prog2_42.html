4.2 Map un Dictionary

Tavs šīs stundas izaicinājums: Apgūt asociatīvās datu struktūras inventāram un konfigurācijai.

SR 2.4.16. Saliktas datu struktūras SR 2.4.4. Iegūst un apstrādā datus

Priekšzināšanu robeža: uzdevumos izmanto tikai iepriekš apgūto un šīs lapas teorijas/koda piemēros parādīto. Ja parādās jauna komanda vai rīks, vispirms atrod tās paraugu teorijas blokā.

Teorija: std::map un Godot Dictionary

Asociatīvā datu struktūra glabā atslēga → vērtība pārus.

// std::map (sakārtots, log n)
#include <map>
std::map<String, int> inventory;
inventory["Health Potion"] = 5;
inventory["Mana Potion"] = 3;

if (inventory.count("Health Potion") > 0) {
    int amount = inventory["Health Potion"];
}

// std::unordered_map (hash, O(1))
#include <unordered_map>
std::unordered_map<int, Enemy*> enemy_by_id;

// Godot Dictionary
Dictionary inv;
inv["coins"] = 100;
inv["arrows"] = 25;
if (inv.has("coins")) print(inv["coins"]);

Konsolidētais praktiskais uzdevums (15+ soļi)

70 min darba sadalījums: 1. uzdevums (~20 min) — izveido minimālo piemēru, kas pārbauda šīs stundas galveno ideju; 2. uzdevums (~25 min) — integrē risinājumu tēmas spēles projektā; 3. uzdevums (~25 min) — notestē robežgadījumus, salabo kļūdas un pieraksti secinājumu README vai komentārā. Tālāk redzamie soļi ir detalizēta karte šiem trim uzdevumiem; ja laiks beidzas, papildu pulēšanu atstāj nākamajai reizei.

Izpildi šo darba plānu, lai 4.2 Map un Dictionary rezultāts būtu pārbaudāms C++ GDExtension projektā.

Izpildes soļi:

  1. Atver projektu un izveido Git branch datu struktūru vai AI darbam.
  2. Pieraksti, kāds datu apjoms vai AI uzvedība šajā stundā jāapstrādā.
  3. Izvēlies piemērotu C++ struktūru: std::vector, std::map, std::unordered_map vai std::set.
  4. Pamato izvēli komentārā vai README, minot meklēšanu, kārtošanu, dzēšanu vai iterāciju.
  5. Izveido C++ klasi vai sistēmas komponenti, kas glabā šo struktūru privātā laukā.
  6. Pievieno publiskas metodes datu pievienošanai, nolasīšanai un drošai dzēšanai.
  7. Ja ir AI, definē skaidrus stāvokļus ar enum class.
  8. Nodali stāvokļa maiņas loģiku no kustības vai renderēšanas koda.
  9. Pievieno robežgadījumu pārbaudi tukšam sarakstam, nezināmam ID vai nederīgam mērķim.

  11. Pievieno debug izvadi, kas rāda aktīvo stāvokli, objektu skaitu vai izvēlēto ceļu.
  12. Pārkompilē C++ kodu un pārbaudi, ka Godot ielādē jauno bibliotēku.
  13. Testē ar mazu datu apjomu, kur rezultātu var pārbaudīt manuāli.
  14. Testē ar lielāku datu apjomu, lai pamanītu veiktspējas vai loģikas problēmas.

  17. Sakārto kodu tā, lai citiem objektiem nav tiešas piekļuves iekšējam konteineram.

Papildu izaicinājums: Set datu struktūra

Unikālu vērtību kolekcija.

Izpildes soļi:

  1. std::set<int> visited_levels — apmeklētie līmeņi.
  2. Insert, find, count operācijas.
  3. Lieto, lai pārliecinātos, ka kaut kas notiek tikai vienreiz.

Biežākās kļūdas

  • Map operator[] uz neeksistējošu key: Izveido jaunu ar default. Lieto find() pārbaudei.
  • String key sajaucies: C++ String ir Godot Variant — atšķiras no std::string.
  • unordered_map ar custom key: Vajag operator== un std::hash specializāciju.

Godot ekrānuzņēmumi

Koda piemērs (paplašināts)

#include <map>

class Player {
private:
    std::map<String, int> inventory;
public:
    void add_item(String name, int amount = 1) {
        inventory[name] += amount;
        UtilityFunctions::print("+", amount, " ", name);
    }

    bool use_item(String name) {
        auto it = inventory.find(name);
        if (it == inventory.end() || it->second <= 0) {
            UtilityFunctions::print("Nav ", name);
            return false;
        }
        it->second--;
        return true;
    }

    void print_inventory() {
        for (const auto& [name, count] : inventory) {
            UtilityFunctions::print(name, ": ", count);
        }
    }
};
