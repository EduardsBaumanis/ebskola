4.3 State machines AI uzvedībai

Tavs šīs stundas izaicinājums: Implementēt AI ar stāvokļu mašīnu (state machine) reālistiskai uzvedībai.

SR 2.4.18. OOP SR 2.4.19. Algoritmu efektivitāte

Priekšzināšanu robeža: uzdevumos izmanto tikai iepriekš apgūto un šīs lapas teorijas/koda piemēros parādīto. Ja parādās jauna komanda vai rīks, vispirms atrod tās paraugu teorijas blokā.

Teorija: State machine konceptu

State machine (stāvokļu mašīna) ir AI dizaina paraugs, kur objekts vienlaikus ir vienā no diskrētiem stāvokļiem un pāriet starp tiem pēc noteikumiem.

enum class EnemyState {
    IDLE,       // gaida
    PATROL,     // staigā
    CHASE,      // dzenas
    ATTACK,     // uzbrūk
    DEAD
};

class Enemy : public CharacterBody2D {
private:
    EnemyState state = EnemyState::IDLE;
    Vector2 patrol_target;
    Player* player = nullptr;

public:
    void _physics_process(double delta) override {
        switch (state) {
            case EnemyState::IDLE:
                if (player_visible()) state = EnemyState::CHASE;
                else if (idle_too_long()) state = EnemyState::PATROL;
                break;
            case EnemyState::PATROL:
                move_to_patrol_target();
                if (player_visible()) state = EnemyState::CHASE;
                break;
            case EnemyState::CHASE:
                move_toward(player->get_position());
                if (in_attack_range()) state = EnemyState::ATTACK;
                else if (!player_visible()) state = EnemyState::IDLE;
                break;
            case EnemyState::ATTACK:
                attack_player();
                if (!in_attack_range()) state = EnemyState::CHASE;
                break;
        }
    }
};

Konsolidētais praktiskais uzdevums (15+ soļi)

70 min darba sadalījums: 1. uzdevums (~20 min) — izveido minimālo piemēru, kas pārbauda šīs stundas galveno ideju; 2. uzdevums (~25 min) — integrē risinājumu tēmas spēles projektā; 3. uzdevums (~25 min) — notestē robežgadījumus, salabo kļūdas un pieraksti secinājumu README vai komentārā. Tālāk redzamie soļi ir detalizēta karte šiem trim uzdevumiem; ja laiks beidzas, papildu pulēšanu atstāj nākamajai reizei.

Izpildi šo darba plānu, lai 4.3 State machines AI uzvedībai rezultāts būtu pārbaudāms C++ GDExtension projektā.

Izpildes soļi:

  1. Atver projektu un izveido Git branch datu struktūru vai AI darbam.
  2. Pieraksti, kāds datu apjoms vai AI uzvedība šajā stundā jāapstrādā.
  3. Izvēlies piemērotu C++ struktūru: std::vector, std::map, std::unordered_map vai std::set.
  4. Pamato izvēli komentārā vai README, minot meklēšanu, kārtošanu, dzēšanu vai iterāciju.
  5. Izveido C++ klasi vai sistēmas komponenti, kas glabā šo struktūru privātā laukā.
  6. Pievieno publiskas metodes datu pievienošanai, nolasīšanai un drošai dzēšanai.
  7. Ja ir AI, definē skaidrus stāvokļus ar enum class.
  8. Nodali stāvokļa maiņas loģiku no kustības vai renderēšanas koda.
  9. Pievieno robežgadījumu pārbaudi tukšam sarakstam, nezināmam ID vai nederīgam mērķim.
  10. Izvairies no dzēšanas no vector vienkārša range-for cikla laikā.
  11. Pievieno debug izvadi, kas rāda aktīvo stāvokli, objektu skaitu vai izvēlēto ceļu.
  12. Pārkompilē C++ kodu un pārbaudi, ka Godot ielādē jauno bibliotēku.
  13. Testē ar mazu datu apjomu, kur rezultātu var pārbaudīt manuāli.
  14. Testē ar lielāku datu apjomu, lai pamanītu veiktspējas vai loģikas problēmas.
  15. Ja iespējams, salīdzini divas struktūras vai divas AI pieejas un pieraksti secinājumu.
  16. Atjaunini README ar datu struktūras izvēli un sarežģītības piezīmēm.
  17. Sakārto kodu tā, lai citiem objektiem nav tiešas piekļuves iekšējam konteineram.
  18. Veic commit ar ziņu, kas nosauc izmantoto datu struktūru vai AI mehāniku.

Aktivitāte 1: Idle un Patrol stāvokļi

Pamata uzvedība bez spēlētāja.

Izpildes soļi:

  1. Enemy klasē: enum class EnemyState, mainīgais state.
  2. Pievieno divus Marker2D nodes scēnai — patrol punkti A un B.
  3. PATROL stāvoklī: kustās uz mērķi; nokļuvis — pāriet uz otru.
  4. IDLE stāvoklī: stāv uz vietas, pēc 3 sec sāk patrol.

Aktivitāte 2: Detekcija un CHASE

Reaģē uz spēlētāju.

Izpildes soļi:

  1. Pievieno Area2D ar lielu Circle Shape — "vision" zona.
  2. Connect uz body_entered → ja player → state = CHASE; uz body_exited → state = IDLE.
  3. CHASE stāvoklī: pārvietojas uz player.position ar speed 150.
  4. Ja distance < 50 → state = ATTACK.

Aktivitāte 3: ATTACK un DEAD

Komplets cikls.

Izpildes soļi:

  1. ATTACK: ik pa 1 sec dod 10 bojājumu spēlētājam.
  2. Animation: pievieno AnimationPlayer un attack animāciju.
  3. Ja Enemy HP <= 0 → state = DEAD, queue_free() pēc 0.5 sec animācijas.
  4. Pievieno UI debug label, kas rāda pašreizējo stāvokli.

Papildu izaicinājums: Behavior Tree

Sarežģītāka AI struktūra.

Izpildes soļi:

  1. Behavior tree = state machine evolūcija ar prioritātēm.
  2. Iemēģini implementēt vienkāršu BT ar Sequence un Selector nodes.
  3. Aprakstīts dziļāk Game AI Pro grāmatā.

Biežākās kļūdas

  • Stāvoklis paliek IDLE: Pārbaudi, vai pārejas nosacījumi tiek izpildīti.
  • Crash, kad player == nullptr: Lieto null check pirms piekļūšanas.
  • Animation neattēlojas: AnimationPlayer ne pareizi pievienots vai animation name nepareizs.

Godot ekrānuzņēmumi

Koda piemērs (paplašināts)

enum class EnemyState { IDLE, PATROL, CHASE, ATTACK, DEAD };

void Enemy::_physics_process(double delta) {
    state_timer += delta;

    switch (state) {
        case EnemyState::IDLE:
            if (player_in_vision) change_state(EnemyState::CHASE);
            else if (state_timer > 3.0) change_state(EnemyState::PATROL);
            break;

        case EnemyState::PATROL: {
            Vector2 dir = (patrol_target - get_position()).normalized();
            velocity = dir * 100.0f;
            move_and_slide();
            if (get_position().distance_to(patrol_target) < 10) {
                swap_patrol_target();
            }
            if (player_in_vision) change_state(EnemyState::CHASE);
        } break;

        case EnemyState::CHASE: {
            Vector2 dir = (player_pos - get_position()).normalized();
            velocity = dir * 200.0f;
            move_and_slide();
            if (get_position().distance_to(player_pos) < 50) {
                change_state(EnemyState::ATTACK);
            }
        } break;

        case EnemyState::ATTACK:
            if (state_timer >= 1.0) {
                player->take_damage(10);
                state_timer = 0;
            }
            break;
    }
}

void Enemy::change_state(EnemyState new_state) {
    state = new_state;
    state_timer = 0;
}
