3.4 Custom GDExtension klases editor redzamas

Tavs šīs stundas izaicinājums: Padarīt savu C++ klasi pilnvērtīgi izmantojamu Godot editor (ar Inspector īpašībām).

SR 2.4.7. UI SR 2.4.18. OOP

Priekšzināšanu robeža: uzdevumos izmanto tikai iepriekš apgūto un šīs lapas teorijas/koda piemēros parādīto. Ja parādās jauna komanda vai rīks, vispirms atrod tās paraugu teorijas blokā.

Teorija: GDCLASS un property exposing

Lai C++ klase parādītos Godot editor un tās laukus varētu rediģēt Inspector, jāizmanto GDCLASS makro un jāreģistrē properties.

class Enemy : public CharacterBody2D {
    GDCLASS(Enemy, CharacterBody2D)

private:
    int hp = 50;
    float speed = 100.0f;
    String enemy_name = "Goblins";

protected:
    static void _bind_methods() {
        ClassDB::bind_method(D_METHOD("get_hp"), &Enemy::get_hp);
        ClassDB::bind_method(D_METHOD("set_hp", "v"), &Enemy::set_hp);
        // Padari property redzamu Inspector
        ADD_PROPERTY(PropertyInfo(Variant::INT, "hp"), "set_hp", "get_hp");

        ClassDB::bind_method(D_METHOD("get_speed"), &Enemy::get_speed);
        ClassDB::bind_method(D_METHOD("set_speed", "v"), &Enemy::set_speed);
        ADD_PROPERTY(PropertyInfo(Variant::FLOAT, "speed"), "set_speed", "get_speed");
    }

public:
    int get_hp() const { return hp; }
    void set_hp(int v) { hp = v; }
    float get_speed() const { return speed; }
    void set_speed(float v) { speed = v; }
};

Teorijas paplašinājums: C++ fokuss

Kritiskais fokuss: šajā stundā nepietiek tikai panākt redzamu efektu editorā. Jāspēj paskaidrot, kura C++ klase glabā stāvokli, kura metode to maina un kā Godot node struktūra šo kodu izmanto.

Pārbaudes pierādījums: katrai būtiskai spēles lomai ir C++ klase ar header/implementation dalījumu, konstruktoriem, metodēm, signāliem vai mantošanu.

#include <godot_cpp/variant/string.hpp>

Konsolidētais praktiskais uzdevums (15+ soļi)

70 min darba sadalījums: 1. uzdevums (~20 min) — izveido minimālo piemēru, kas pārbauda šīs stundas galveno ideju; 2. uzdevums (~25 min) — integrē risinājumu tēmas spēles projektā; 3. uzdevums (~25 min) — notestē robežgadījumus, salabo kļūdas un pieraksti secinājumu README vai komentārā. Tālāk redzamie soļi ir detalizēta karte šiem trim uzdevumiem; ja laiks beidzas, papildu pulēšanu atstāj nākamajai reizei.

Izpildi šo darba plānu, lai 3.4 Custom GDExtension klases editor redzamas rezultāts būtu pārbaudāms C++ GDExtension projektā.

Izpildes soļi:

  1. Atver projektu un izveido Git branch šīs stundas OOP darbam.
  2. Apraksti vienā teikumā, kādu spēles lomu modelē jaunā C++ klase.
  3. Izveido .hpp failu ar klases deklarāciju un privātajiem datiem.
  4. Izveido .cpp failu ar konstruktoru un metožu implementāciju.
  5. Pārliecinies, ka katrai klasei ir viena skaidra atbildība.
  6. Ja klase ir Godot node, pievieno GDCLASS makro un _bind_methods().
  7. Ja klase ir parasta C++ palīgklase, atstāj to neatkarīgu no Scene Tree.
  8. Pievieno konstruktoru, kas objektu atstāj derīgā sākumstāvoklī.
  9. Pievieno vismaz vienu metodi, kas maina objekta stāvokli kontrolētā veidā.
  10. Pievieno vismaz vienu const metodi, kas tikai nolasa stāvokli.
  11. Ja lieto mantošanu, pārbaudi, ka bāzes klase satur kopīgo uzvedību, nevis konkrētas detaļas.
  12. Ja lieto signālus, deklarē tos _bind_methods() un pieslēdz ar Callable.
  13. Reģistrē Godot redzamās klases register_types.cpp failā.
  14. Pievieno īsu C++ izvadi vai UI pārbaudi, kas pierāda, ka metode tiek izsaukta.
  15. Pārkompilē projektu un izlabo visas kompilatora kļūdas pirms Godot testēšanas.
  16. Godot editorā pārbaudi, vai pielāgotā klase vai tās īpašības parādās pareizajā vietā.
  17. README pieraksti, kāpēc izvēlējies kompozīciju, mantošanu vai signālus.
  18. Veic commit ar ziņu, kas nosauc izveidoto klasi vai OOP principu.

Aktivitāte 1: Inspector redzami lauki

Padari Hero atribūtus rediģējamus.

Izpildes soļi:

  1. Hero klasei pievieno getter/setter visiem laukiem (name, hp, attack, defense).
  2. _bind_methods: ADD_PROPERTY katram.
  3. Pārkompilē, atver Godot, atlasi Hero node.
  4. Inspector tagad rāda visus laukus — vari mainīt vērtības bez koda.

Aktivitāte 2: Property hints

Padari saskarni jaukāku.

Izpildes soļi:

  1. PROPERTY_HINT_RANGE ar "0,200" priekš HP.
  2. PROPERTY_HINT_ENUM priekš varoņa class: "Knight,Mage,Archer".
  3. Tagad lietotājs nevar ievadīt nederīgas vērtības.

Aktivitāte 3: Export grupās

Organizē Inspector laukus.

Izpildes soļi:

  1. Pievieno: ADD_GROUP("Stats", ""); pirms HP property.
  2. Pievieno: ADD_GROUP("Combat", ""); pirms attack property.
  3. Lauki tagad ir loģiski sakārtoti grupās Inspector.

Papildu izaicinājums: Custom resource types

Izveido pielāgotu Resource klasi.

Izpildes soļi:

  1. HeroData : public Resource ar export properties.
  2. Saglabā kā .tres fails: 'knight_data.tres'.
  3. Vairāki Hero nodes var atsaukties uz vienu HeroData.

Biežākās kļūdas

  • Inspector lauks neparādās: Aizmirsts ADD_PROPERTY pēc bind_method.
  • Set funkcija nestrādā: Property name jāatbilst getter/setter pattern.
  • Export hint kļūda: Pārbaudi sintaksi ar Godot dokumentāciju.

Godot ekrānuzņēmumi

Koda piemērs (paplašināts)

class Enemy : public CharacterBody2D {
    GDCLASS(Enemy, CharacterBody2D)
private:
    int hp = 50;
    float speed = 100.0f;
public:
    int get_hp() const { return hp; }
    void set_hp(int v) { hp = std::max(0, v); }
    float get_speed() const { return speed; }
    void set_speed(float v) { speed = std::max(0.0f, v); }

protected:
    static void _bind_methods() {
        ClassDB::bind_method(D_METHOD("get_hp"), &Enemy::get_hp);
        ClassDB::bind_method(D_METHOD("set_hp", "v"), &Enemy::set_hp);
        ADD_PROPERTY(PropertyInfo(Variant::INT, "hp",
            PROPERTY_HINT_RANGE, "0,500"), "set_hp", "get_hp");

        ClassDB::bind_method(D_METHOD("get_speed"), &Enemy::get_speed);
        ClassDB::bind_method(D_METHOD("set_speed", "v"), &Enemy::set_speed);
        ADD_PROPERTY(PropertyInfo(Variant::FLOAT, "speed",
            PROPERTY_HINT_RANGE, "0,500,0.1"), "set_speed", "get_speed");
    }
};
Inspector rāda Hero ar Stats grupu, kurā ir HP slider 0-200, ATK skaitlis, DEF skaitlis.
