3.5 Statistikas sistēma un struct

Tavs šīs stundas izaicinājums: Izmantot struct datu grupēšanai un kompozīciju (composition) klašu organizēšanai.

SR 2.4.16. Saliktas datu struktūras SR 2.4.18. OOP

Priekšzināšanu robeža: uzdevumos izmanto tikai iepriekš apgūto un šīs lapas teorijas/koda piemēros parādīto. Ja parādās jauna komanda vai rīks, vispirms atrod tās paraugu teorijas blokā.

Teorija: struct un kompozīcija

struct ir vienkāršāka klase — visi lauki ir public pēc noklusējuma. Lietojama datu grupēšanai.

struct Stats {
    int hp = 100;
    int max_hp = 100;
    int attack = 10;
    int defense = 5;
    int speed = 5;

    void heal(int amount) {
        hp = std::min(max_hp, hp + amount);
    }
};

class Hero {
private:
    Stats stats;        // kompozīcija!
    String name;
public:
    Hero(String n) : name(n) {}

    Stats& get_stats() { return stats; }  // atgriež references
};

Kompozīcija (klase satur citu klasi) bieži ir labāka par mantošanu — fleksiblāka un mazāk saistīta.

Teorijas paplašinājums: C++ fokuss

Kritiskais fokuss: šajā stundā nepietiek tikai panākt redzamu efektu editorā. Jāspēj paskaidrot, kura C++ klase glabā stāvokli, kura metode to maina un kā Godot node struktūra šo kodu izmanto.

Pārbaudes pierādījums: katrai būtiskai spēles lomai ir C++ klase ar header/implementation dalījumu, konstruktoriem, metodēm, signāliem vai mantošanu.

Konsolidētais praktiskais uzdevums (15+ soļi)

70 min darba sadalījums: 1. uzdevums (~20 min) — izveido minimālo piemēru, kas pārbauda šīs stundas galveno ideju; 2. uzdevums (~25 min) — integrē risinājumu tēmas spēles projektā; 3. uzdevums (~25 min) — notestē robežgadījumus, salabo kļūdas un pieraksti secinājumu README vai komentārā. Tālāk redzamie soļi ir detalizēta karte šiem trim uzdevumiem; ja laiks beidzas, papildu pulēšanu atstāj nākamajai reizei.

Izpildi šo darba plānu, lai 3.5 Statistikas sistēma un struct rezultāts būtu pārbaudāms C++ GDExtension projektā.

Izpildes soļi:

  1. Atver projektu un izveido Git branch šīs stundas OOP darbam.
  2. Apraksti vienā teikumā, kādu spēles lomu modelē jaunā C++ klase.
  3. Izveido .hpp failu ar klases deklarāciju un privātajiem datiem.
  4. Izveido .cpp failu ar konstruktoru un metožu implementāciju.
  5. Pārliecinies, ka katrai klasei ir viena skaidra atbildība.
  6. Ja klase ir Godot node, pievieno GDCLASS makro un _bind_methods().
  7. Ja klase ir parasta C++ palīgklase, atstāj to neatkarīgu no Scene Tree.
  8. Pievieno konstruktoru, kas objektu atstāj derīgā sākumstāvoklī.
  9. Pievieno vismaz vienu metodi, kas maina objekta stāvokli kontrolētā veidā.
  10. Pievieno vismaz vienu const metodi, kas tikai nolasa stāvokli.
  11. Ja lieto mantošanu, pārbaudi, ka bāzes klase satur kopīgo uzvedību, nevis konkrētas detaļas.
  12. Ja lieto signālus, deklarē tos _bind_methods() un pieslēdz ar Callable.
  13. Reģistrē Godot redzamās klases register_types.cpp failā.
  14. Pievieno īsu C++ izvadi vai UI pārbaudi, kas pierāda, ka metode tiek izsaukta.
  15. Pārkompilē projektu un izlabo visas kompilatora kļūdas pirms Godot testēšanas.
  16. Godot editorā pārbaudi, vai pielāgotā klase vai tās īpašības parādās pareizajā vietā.
  17. README pieraksti, kāpēc izvēlējies kompozīciju, mantošanu vai signālus.
  18. Veic commit ar ziņu, kas nosauc izveidoto klasi vai OOP principu.

Aktivitāte 1: Stats struct

Izveido statistikas datu tipu.

Izpildes soļi:

  1. stats.hpp: definē Stats struct ar laukiem hp, max_hp, attack, defense, speed.
  2. Pievieno metodes: heal(int), damage(int), buff(int amount, int duration).
  3. Lieto initializer list konstruktorā.
  4. Hero klase tagad satur Stats stats.

Aktivitāte 2: Buff/Debuff sistēma

Pagaidu modifikatori.

Izpildes soļi:

  1. Izveido struct Buff { String name; int amount; float duration; };
  2. Stats glabā std::vector<Buff> active_buffs.
  3. Apply un tick funkcijas: katrā kadrā mazina duration, noņem expired.
  4. Piemērs: 'strength_potion' +5 attack uz 30 sec.

Aktivitāte 3: Equipment sistēma

Ekipējums modificē Stats.

Izpildes soļi:

  1. struct Equipment { String name; int hp_bonus; int atk_bonus; ... };
  2. Hero glabā Equipment weapon un Equipment armor.
  3. Total stats = base stats + equipment bonuses.
  4. Pievieno equip/unequip metodes.

Papildu izaicinājums: Save/Load Stats

Serializē uz JSON.

Izpildes soļi:

  1. Stats metode Dictionary to_dict() const.
  2. Stats metode void from_dict(const Dictionary& d).
  3. Tas ļaus 5. tēmā vienkārši saglabāt visu varoni.

Biežākās kļūdas

  • Stats default values neuzstādās: C++ struct lauki bez init = nedef vērtība. Vienmēr inicializē.
  • Vector iteration crash: Mainot vektoru cikla laikā, neizvairies no UB.
  • Slow buff lookup: Lielam vector lieto std::map vai std::unordered_map.

Godot ekrānuzņēmumi

Koda piemērs (paplašināts)

struct Stats {
    int hp = 100, max_hp = 100;
    int attack = 10, defense = 5, speed = 5;
    std::vector<Buff> buffs;

    int get_total_attack() const {
        int total = attack;
        for (const auto& b : buffs) {
            if (b.target == "attack") total += b.amount;
        }
        return total;
    }

    void take_damage(int amount) {
        int actual = std::max(1, amount - defense);
        hp = std::max(0, hp - actual);
    }

    void update(float delta) {
        for (auto& b : buffs) b.duration -= delta;
        buffs.erase(std::remove_if(buffs.begin(), buffs.end(),
            [](const Buff& b) { return b.duration <= 0; }),
            buffs.end());
    }
};
Hero get_total_attack atgriež 15 (10 base + 5 buff), pēc 30 sec — atkal 10.
