3.3 Signāli un objektu komunikācija

Tavs šīs stundas izaicinājums: Apgūt Godot signal sistēmu objektu komunikācijai.

SR 2.4.15. Modulāras programmas SR 2.4.18. OOP

Teorija: Godot signāli (Signals)

Godot signāli ir veids, kā objekti var sazināties bez tieša savienojuma. Viens objekts emit (raida) signālu, citi connect (pievienojas), lai uz to reaģētu.

// Klases definīcijā ar signal
class Hero : public Node {
    GDCLASS(Hero, Node)
protected:
    static void _bind_methods() {
        ClassDB::bind_method(...);
        // Reģistrē signal
        ADD_SIGNAL(MethodInfo("died", PropertyInfo(Variant::STRING, "name")));
        ADD_SIGNAL(MethodInfo("damaged", PropertyInfo(Variant::INT, "amount")));
    }

public:
    void take_damage(int amount) {
        hp -= amount;
        emit_signal("damaged", amount);
        if (hp <= 0) emit_signal("died", name);
    }
};

Citur kodā:

hero->connect("died", Callable(this, "on_hero_died"));

Teorijas paplašinājums: C++ fokuss

Kritiskais fokuss: šajā stundā nepietiek tikai panākt redzamu efektu editorā. Jāspēj paskaidrot, kura C++ klase glabā stāvokli, kura metode to maina un kā Godot node struktūra šo kodu izmanto.

Pārbaudes pierādījums: katrai būtiskai spēles lomai ir C++ klase ar header/implementation dalījumu, konstruktoriem, metodēm, signāliem vai mantošanu.

Konsolidētais praktiskais uzdevums (15+ soļi)

Izpildi šo darba plānu, lai 3.3 Signāli un objektu komunikācija rezultāts būtu pārbaudāms C++ GDExtension projektā.

Izpildes soļi:

  1. Atver projektu un izveido Git branch šīs stundas OOP darbam.
  2. Apraksti vienā teikumā, kādu spēles lomu modelē jaunā C++ klase.
  3. Izveido .hpp failu ar klases deklarāciju un privātajiem datiem.
  4. Izveido .cpp failu ar konstruktoru un metožu implementāciju.
  5. Pārliecinies, ka katrai klasei ir viena skaidra atbildība.
  6. Ja klase ir Godot node, pievieno GDCLASS makro un _bind_methods().
  7. Ja klase ir parasta C++ palīgklase, atstāj to neatkarīgu no Scene Tree.
  8. Pievieno konstruktoru, kas objektu atstāj derīgā sākumstāvoklī.
  9. Pievieno vismaz vienu metodi, kas maina objekta stāvokli kontrolētā veidā.
  10. Pievieno vismaz vienu const metodi, kas tikai nolasa stāvokli.
  11. Ja lieto mantošanu, pārbaudi, ka bāzes klase satur kopīgo uzvedību, nevis konkrētas detaļas.
  12. Ja lieto signālus, deklarē tos _bind_methods() un pieslēdz ar Callable.
  13. Reģistrē Godot redzamās klases register_types.cpp failā.
  14. Pievieno īsu C++ izvadi vai UI pārbaudi, kas pierāda, ka metode tiek izsaukta.
  15. Pārkompilē projektu un izlabo visas kompilatora kļūdas pirms Godot testēšanas.
  16. Godot editorā pārbaudi, vai pielāgotā klase vai tās īpašības parādās pareizajā vietā.
  17. README pieraksti, kāpēc izvēlējies kompozīciju, mantošanu vai signālus.
  18. Veic commit ar ziņu, kas nosauc izveidoto klasi vai OOP principu.

Aktivitāte 1: Damage signāls

Pievieno bojājuma signāl Hero klasei.

Izpildes soļi:

  1. Hero klasē _bind_methods pievieno: ADD_SIGNAL(MethodInfo("damaged", PropertyInfo(Variant::INT, "amount")));
  2. take_damage beigās: emit_signal("damaged", amount);
  3. Game klasē connect: hero->connect("damaged", Callable(this, "on_damage"));
  4. on_damage(int amount): izvada "Hero saņēma {amount} bojājumu!"

Aktivitāte 2: Death signāls un game over

Reaģē uz varoņa nāvi.

Izpildes soļi:

  1. Pievieno died signal Hero klasei.
  2. Emit, kad hp <= 0.
  3. Game klase to noķer un parāda Game Over.
  4. Visiem varoņiem tas pats konekts — testē visus scenārijus.

Aktivitāte 3: UI atjaunošana ar signāliem

HUD atjauninās automātiski.

Izpildes soļi:

  1. Izveido HUD scēnu ar Label nodes priekš katra varoņa HP.
  2. HUD klase connect uz Hero damaged signal.
  3. on_damage: atjaunina Label tekstu.
  4. Tagad UI ir decoupled no spēles loģikas — tīrāka arhitektūra.

Papildu izaicinājums: Custom Callable lambdas

Modernā Godot signal sintakse.

Izpildes soļi:

  1. Iemēģini lambda kā callback: Callable::bind(...)
  2. Vai single-time signal: connect(... CONNECT_ONE_SHOT)
  3. Disconnect, kad vajag: hero->disconnect("damaged", ...)

Biežākās kļūdas

  • emit_signal nav efekta: Signal nav reģistrēts ar ADD_SIGNAL.
  • Callable error: Metode klasei nav reģistrēta ar bind_method.
  • Signal saņemts vairākkārt: Connect izsaukts vairākas reizes — pārbaudi is_connected.

Godot ekrānuzņēmumi

Koda piemērs (paplašināts)

#include <godot_cpp/core/method_bind.hpp>

void Hero::_bind_methods() {
    ClassDB::bind_method(D_METHOD("take_damage", "amount"),
                         &Hero::take_damage);
    ADD_SIGNAL(MethodInfo("damaged",
        PropertyInfo(Variant::INT, "amount")));
    ADD_SIGNAL(MethodInfo("died",
        PropertyInfo(Variant::STRING, "name")));
}

void Hero::take_damage(int amount) {
    hp = std::max(0, hp - amount);
    emit_signal("damaged", amount);
    if (hp == 0) {
        emit_signal("died", name);
    }
}

// Game klasē
void Game::_ready() {
    Hero* artur = ...;
    artur->connect("damaged", Callable(this, "on_damage"));
    artur->connect("died", Callable(this, "on_death"));
}
