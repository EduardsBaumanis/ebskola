3.1 C++ klases pamati

Tavs šīs stundas izaicinājums: Saprast OOP fundamentālo konceptu — klases un objekti — un izveidot pirmo varoņa klasi.

SR 2.4.18. OOP pamati SR 2.4.15. Modulāras programmas

Priekšzināšanu robeža: uzdevumos izmanto tikai iepriekš apgūto un šīs lapas teorijas/koda piemēros parādīto. Ja parādās jauna komanda vai rīks, vispirms atrod tās paraugu teorijas blokā.

Teorija: C++ klases struktūra

Klase ir veidne objektu radīšanai. Tā satur datus (atribūtus) un funkcijas (metodes).

class Hero {
private:           // tikai klases iekšienē pieejams
    int hp;
    int max_hp;

public:            // pieejams ārpusē
    String name;
    int level = 1;

    // Konstruktors
    Hero(String n, int h) {
        name = n;
        hp = max_hp = h;
    }

    // Metodes
    void take_damage(int amount) {
        hp -= amount;
        if (hp < 0) hp = 0;
    }

    bool is_alive() const {
        return hp > 0;
    }

    int get_hp() const { return hp; }
};

public/private/protected kontrolē encapsulation (iekapsulāciju).

Teorijas paplašinājums: C++ fokuss

Kritiskais fokuss: šajā stundā nepietiek tikai panākt redzamu efektu editorā. Jāspēj paskaidrot, kura C++ klase glabā stāvokli, kura metode to maina un kā Godot node struktūra šo kodu izmanto.

Pārbaudes pierādījums: katrai būtiskai spēles lomai ir C++ klase ar header/implementation dalījumu, konstruktoriem, metodēm, signāliem vai mantošanu.

Konsolidētais praktiskais uzdevums (15+ soļi)

70 min darba sadalījums: 1. uzdevums (~20 min) — izveido minimālo piemēru, kas pārbauda šīs stundas galveno ideju; 2. uzdevums (~25 min) — integrē risinājumu tēmas spēles projektā; 3. uzdevums (~25 min) — notestē robežgadījumus, salabo kļūdas un pieraksti secinājumu README vai komentārā. Tālāk redzamie soļi ir detalizēta karte šiem trim uzdevumiem; ja laiks beidzas, papildu pulēšanu atstāj nākamajai reizei.

Izpildi šo darba plānu, lai 3.1 C++ klases pamati rezultāts būtu pārbaudāms C++ GDExtension projektā.

Izpildes soļi:

  1. Atver projektu un izveido Git branch šīs stundas OOP darbam.
  2. Apraksti vienā teikumā, kādu spēles lomu modelē jaunā C++ klase.
  3. Izveido .hpp failu ar klases deklarāciju un privātajiem datiem.
  4. Izveido .cpp failu ar konstruktoru un metožu implementāciju.
  5. Pārliecinies, ka katrai klasei ir viena skaidra atbildība.
  6. Ja klase ir Godot node, pievieno GDCLASS makro un _bind_methods().
  7. Ja klase ir parasta C++ palīgklase, atstāj to neatkarīgu no Scene Tree.
  8. Pievieno konstruktoru, kas objektu atstāj derīgā sākumstāvoklī.
  9. Pievieno vismaz vienu metodi, kas maina objekta stāvokli kontrolētā veidā.
  10. Pievieno vismaz vienu const metodi, kas tikai nolasa stāvokli.
  11. Ja lieto mantošanu, pārbaudi, ka bāzes klase satur kopīgo uzvedību, nevis konkrētas detaļas.
  12. Ja lieto signālus, deklarē tos _bind_methods() un pieslēdz ar Callable.
  13. Reģistrē Godot redzamās klases register_types.cpp failā.
  14. Pievieno īsu C++ izvadi vai UI pārbaudi, kas pierāda, ka metode tiek izsaukta.
  15. Pārkompilē projektu un izlabo visas kompilatora kļūdas pirms Godot testēšanas.
  16. Godot editorā pārbaudi, vai pielāgotā klase vai tās īpašības parādās pareizajā vietā.
  17. README pieraksti, kāpēc izvēlējies kompozīciju, mantošanu vai signālus.
  18. Veic commit ar ziņu, kas nosauc izveidoto klasi vai OOP principu.

Aktivitāte 1: Hero klase

Izveido pamata varoņa klasi.

Izpildes soļi:

  1. hero.hpp: definē Hero klasi ar String name, int hp, int attack, int defense.
  2. Konstruktors: Hero(String n, int h, int a, int d).
  3. Metodes: take_damage(int), is_alive(), print_status().
  4. hero.cpp: implementē metodes.

Aktivitāte 2: Hero objekta lietošana

Izveido un izmanto Hero objektus.

Izpildes soļi:

  1. Test scēnā Game klasē izveido: Hero artur("Artūrs", 100, 15, 5);
  2. Izsauc artur.print_status(); artur.take_damage(20);
  3. Izvada rezultātus Output panelī.
  4. Izveido vēl 2-3 Hero objektus ar dažādiem parametriem.

Aktivitāte 3: const un mutability

Apgūsti const lietojumu C++.

Izpildes soļi:

  1. Padari getter metodes const: int get_hp() const;
  2. Mēģini const objektu modificēt — kompilators dod kļūdu.
  3. Pievieno const Hero& parametri funkcijām, kas nemaina objektu.
  4. Iemēģini static method: static Hero create_random();

Papildu izaicinājums: Operator overloading

Pielāgo operatorus.

Izpildes soļi:

  1. Pievieno bool operator==(const Hero& other) const; — salīdzina pēc nosaukuma.
  2. Pievieno operator<<, lai varētu lietot ar std::cout.
  3. Šī ir advanced C++ tehnika.

Biežākās kļūdas

  • Cannot access private member: Mēģini piekļūt private laukam ārpus klases — lieto getter.
  • Multiple definition error: Implementācija .hpp failā — pārvieto uz .cpp.
  • Crash, lietojot const objektu: Metode jāmarķē kā const, lai tiktu izsaukta uz const objekta.

Godot ekrānuzņēmumi

Koda piemērs (paplašināts)

#include "hero.hpp"
#include <godot_cpp/variant/utility_functions.hpp>

using namespace godot;

Hero::Hero(String n, int h, int a, int d)
    : name(n), hp(h), max_hp(h), attack(a), defense(d) {}

void Hero::take_damage(int amount) {
    int actual = std::max(0, amount - defense);
    hp = std::max(0, hp - actual);
    UtilityFunctions::print(name, " saņem ", actual, " bojājumu. HP: ", hp);
}

bool Hero::is_alive() const { return hp > 0; }

void Hero::print_status() const {
    UtilityFunctions::print(name, " HP: ", hp, "/", max_hp,
                            " ATK: ", attack, " DEF: ", defense);
}
Output: 'Artūrs HP: 100/100 ATK: 15 DEF: 5' un pēc bojājuma 'HP: 80'.
