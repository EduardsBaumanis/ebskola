3.2 Mantošana C++ un Godot

Tavs šīs stundas izaicinājums: Izveidot specializētas varoņu klases, izmantojot inheritance (mantošanu).

SR 2.4.18. OOP SR 2.4.15. Modulāras programmas

Teorija: Inheritance un virtuālas funkcijas

Mantošana ļauj bērna klasei pārmantot vecāka atribūtus un metodes, pievienojot vai pārrakstot.

class Hero { /* base klase */ };

class Knight : public Hero {
public:
    int armor;
    Knight(String n) : Hero(n, 150, 18, 12) {  // izsauc vecāku konstruktoru
        armor = 10;
    }
    // Pārraksta (override) vecāka metodi
    void take_damage(int amount) override {
        amount -= armor;  // bruņas papildus aizsargā
        Hero::take_damage(amount);
    }
};

class Mage : public Hero {
public:
    int mana;
    Mage(String n) : Hero(n, 80, 25, 3), mana(100) {}

    void cast_spell(Hero& target) {
        if (mana >= 20) {
            mana -= 20;
            target.take_damage(40);
        }
    }
};

virtual atslēgvārds ļauj izsaukt bērna metodi caur vecāka pointer (polimorfisms).

Teorijas paplašinājums: C++ fokuss

Kritiskais fokuss: šajā stundā nepietiek tikai panākt redzamu efektu editorā. Jāspēj paskaidrot, kura C++ klase glabā stāvokli, kura metode to maina un kā Godot node struktūra šo kodu izmanto.

Konsolidētais praktiskais uzdevums (15+ soļi)

Izpildi šo darba plānu, lai 3.2 Mantošana C++ un Godot rezultāts būtu pārbaudāms C++ GDExtension projektā.

Izpildes soļi:

  1. Atver projektu un izveido Git branch šīs stundas OOP darbam.
  2. Apraksti vienā teikumā, kādu spēles lomu modelē jaunā C++ klase.
  3. Izveido .hpp failu ar klases deklarāciju un privātajiem datiem.
  4. Izveido .cpp failu ar konstruktoru un metožu implementāciju.
  5. Pārliecinies, ka katrai klasei ir viena skaidra atbildība.
  6. Ja klase ir Godot node, pievieno GDCLASS makro un _bind_methods().
  7. Ja klase ir parasta C++ palīgklase, atstāj to neatkarīgu no Scene Tree.
  8. Pievieno konstruktoru, kas objektu atstāj derīgā sākumstāvoklī.
  9. Pievieno vismaz vienu metodi, kas maina objekta stāvokli kontrolētā veidā.
  10. Pievieno vismaz vienu const metodi, kas tikai nolasa stāvokli.
  11. Ja lieto mantošanu, pārbaudi, ka bāzes klase satur kopīgo uzvedību, nevis konkrētas detaļas.
  12. Ja lieto signālus, deklarē tos _bind_methods() un pieslēdz ar Callable.
  13. Reģistrē Godot redzamās klases register_types.cpp failā.
  14. Pievieno īsu C++ izvadi vai UI pārbaudi, kas pierāda, ka metode tiek izsaukta.
  15. Pārkompilē projektu un izlabo visas kompilatora kļūdas pirms Godot testēšanas.
  16. Godot editorā pārbaudi, vai pielāgotā klase vai tās īpašības parādās pareizajā vietā.
  17. README pieraksti, kāpēc izvēlējies kompozīciju, mantošanu vai signālus.
  18. Veic commit ar ziņu, kas nosauc izveidoto klasi vai OOP principu.

Aktivitāte 1: Knight un Mage klases

Izveido divas specializētas Hero apakšklases.

Izpildes soļi:

  1. knight.hpp: class Knight : public Hero ar int armor.
  2. Konstruktors: Knight(String n) : Hero(n, 150, 18, 12), armor(10) {}
  3. Pārraksta take_damage: amortē 5 punkti pirms vecāka metodes izsaukuma.
  4. Atkārto ar Mage: HP=80, ATK=25, mana=100, metode cast_spell.

Aktivitāte 2: Virtuālas funkcijas

Polimorfisma demonstrācija.

Izpildes soļi:

  1. Hero klasē padari take_damage un attack_target par virtual.
  2. Izveido masīvu: std::vector<Hero*> party, pievieno Knight un Mage.
  3. For ciklā izsauc h->take_damage(20) — izsaucas pareizās klases versijas.
  4. Tas ir polimorfisms darbībā.

Aktivitāte 3: Trešais varonis: Archer

Izveido vēl vienu specializāciju.

Izpildes soļi:

  1. Archer : public Hero ar int agility un int arrows.
  2. shoot() metode: prasa arrows > 0, dara 2x bojājumu.
  3. dodge_chance: pārraksta take_damage — 30% iespēja izvairīties pavisam.
  4. Pievieno party masīvam un testē.

Papildu izaicinājums: Diamond inheritance

Vairāku vecāku problēma.

Izpildes soļi:

  1. Iemēģini izveidot klasi Paladin : public Knight, public Mage.
  2. Tas izraisa diamond problēmu — Hero atribūti dublētos.
  3. Risinājums: virtual mantošana vai izvairies.

Biežākās kļūdas

  • Cannot call base constructor: Trūkst initializer list ar : Hero(...).
  • 'override' was not declared: C++11+ vajadzīgs (Godot prasa C++17).
  • Slicing problem: Pieliecot Hero objektu, ne Hero*, polimorfisms nedarbojas.

Godot ekrānuzņēmumi

Koda piemērs (paplašināts)

class Knight : public Hero {
private:
    int armor;
public:
    Knight(String n) : Hero(n, 150, 18, 12), armor(10) {}

    void take_damage(int amount) override {
        amount = std::max(1, amount - armor);
        Hero::take_damage(amount);
    }
};

class Mage : public Hero {
private:
    int mana;
public:
    Mage(String n) : Hero(n, 80, 25, 3), mana(100) {}

    void cast_spell(Hero* target) {
        if (mana >= 20) {
            mana -= 20;
            UtilityFunctions::print(name, " met burvojumu pret ", target->get_name());
            target->take_damage(40);
        }
    }
};
