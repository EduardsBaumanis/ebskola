1.6 Noslēguma projekts: Pong

Tavs šīs stundas izaicinājums: Apvienot tēmas prasmes — Godot Scene Tree, C++ klases — pilnvērtīgā Pong arkādes spēlē.

SR 2.4.1. Problēmas dekompozīcija SR 2.4.7. UI SR 2.4.15. Modulāras programmas

Teorija: Spēles apraksts un arhitektūra

Pong ir klasiska 1972. gada arkādes spēle. Divi spēlētāji vada vertikālas paddles un atsit bumbu pretiniekam. Pirmais ar 5 punktiem uzvar.

C++ klases:

  • Paddle.cpp — spēlētāja vadītā raķete (W/S vai Augšup/Lejup).
  • Ball.cpp — bumba ar konstantu ātrumu, atlec.
  • Game.cpp — punktu uzskaite, restart.
class Game : public Node {
    GDCLASS(Game, Node)
private:
    int score_left = 0;
    int score_right = 0;
    Ball *ball = nullptr;
public:
    void _ready() override;
    void on_score(bool left_player);
    void reset_ball();
    void update_score_labels();
};

1. uzdevums: Paddle klase

Spēlētāja vadāmā raķete.

Izpildes soļi:

  1. paddle.hpp/paddle.cpp: Paddle : public CharacterBody2D.
  2. Locekļi: float speed = 400.0f, String input_up, String input_down.
  3. _physics_process(double delta): lasa Input, atjaunina velocity.y, izsauc move_and_slide().
  4. Reģistrē klasi, pievieno paddle nodes. PaddleLeft input_up="p1_up" utt.
  5. Project Settings → Input Map: p1_up=W, p1_down=S, p2_up=Up, p2_down=Down.

2. uzdevums: Ball klase

Bumba ar fiziku.

Izpildes soļi:

  1. Ball : public CharacterBody2D ar float speed = 500.0f.
  2. _ready(): velocity = Vector2(speed, randf_range(-200, 200)).
  3. _physics_process: auto collision = move_and_collide(velocity * delta);
  4. Sadursmes gadījumā: velocity = velocity.bounce(collision->get_normal());
  5. Paātrināšanās pēc atsiteņa: velocity *= 1.05f;

3. uzdevums: Game un punkti

Punktu sistēma.

Izpildes soļi:

  1. Game klase glabā score_left, score_right.
  2. Bumba _physics_process: ja x < 0 — labā punkts; ja x > 1280 — kreisā.
  3. Game::on_score(bool left): palielina punktus, atjaunina Label.
  4. Pie 5 punktiem: "Player X wins!" + restart poga.
  5. GitHub: 4+ commit, README.md ar instrukcijām.

Papildus uzdevums: AI pretinieks

Pievieno datora pretinieku.

Izpildes soļi:

  1. AIPaddle : public Paddle apakšklase.
  2. _physics_process: paddle.y → tuvināt ball.y ar maksimālo speed.
  3. Reaģē tikai, kad bumba kustās uz to: velocity.x norāda virzienu.
  4. Game opcija: bool single_player.

Biežākās kļūdas

  • Bumba iziet cauri paddle: Speed pārāk augsts; lieto move_and_collide nevis set_position.
  • Punkti netiek atjaunināti: Pārliecinies par get_node<Label>("ScoreLeft") references.
  • Build neizmainās: Izdzēs .sconsign.dblite un kompilē no jauna.

Godot ekrānuzņēmumi

Koda piemērs (paplašināts)

void Ball::_physics_process(double delta) {
    auto collision = move_and_collide(velocity * delta);
    if (collision.is_valid()) {
        velocity = velocity.bounce(collision->get_normal());
        velocity *= 1.05f;
    }
    Vector2 pos = get_position();
    if (pos.x < 0) get_parent()->call("on_score", false);
    else if (pos.x > 1280) get_parent()->call("on_score", true);
}

void Game::on_score(bool left_player) {
    if (left_player) score_left++;
    else score_right++;
    update_score_labels();
    reset_ball();
}
