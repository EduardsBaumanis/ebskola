1.6 Noslēguma projekts: Pong

Tavs šīs stundas izaicinājums: Apvienot tēmas prasmes — Godot Scene Tree, C++ klases — pilnvērtīgā Pong arkādes spēlē.

SR 2.4.1. Problēmas dekompozīcija SR 2.4.7. UI SR 2.4.15. Modulāras programmas

Teorija: Spēles apraksts un arhitektūra

Pong ir klasiska 1972. gada arkādes spēle. Divi spēlētāji vada vertikālas paddles un atsit bumbu pretiniekam. Pirmais ar 5 punktiem uzvar.

Šis projekts ir labs 1. tēmas noslēgumam, jo tajā ir maz objektu, bet tie pārbauda gandrīz visu pamatu: Scene Tree, atkārtoti lietojamas scēnas, Input Map, C++ klases, sadursmes, UI atjaunināšanu un spēles stāvokli. Pong nav izvēlēts tāpēc, ka tas ir vizuāli sarežģīts; tas ir izvēlēts tāpēc, ka kļūdas tajā ir viegli pamanāmas.

Spēles loģika tiek rakstīta C++. Godot editorā drīkst salikt scēnas un iestatīt resursus, bet paddle kustība, bumbas atlēkšana, punktu pieskaitīšana un uzvaras pārbaude jārealizē GDExtension klasēs.

C++ klases:

  • Paddle — spēlētāja vadītā raķete ar ievades darbībām input_up un input_down.
  • Ball — bumba ar ātrumu, sadursmju apstrādi un reset mehāniku.
  • Game — punktu uzskaite, UI atjaunināšana, uzvaras noteikšana un raunda restartēšana.
class Game : public Node {
    GDCLASS(Game, Node)
private:
    int score_left = 0;
    int score_right = 0;
    Ball *ball = nullptr;
public:
    void _ready() override;
    void on_score(bool left_player);
    void reset_ball();
    void update_score_labels();
};

Noslēguma projekta uzdevums: Pong ar C++

Apvieno tēmas prasmes vienā spēlē, kur uzvedību vada C++ klases.

Izpildes soļi:

  1. Pārbaudi, ka projekts kompilējas pēc 1.3 un 1.4 stundas sagataves.
  2. Project Settings → Input Map pievieno p1_up, p1_down, p2_up un p2_down.
  3. Piesaisti p1_up taustiņam W un p1_down taustiņam S.
  4. Piesaisti p2_up taustiņam Up Arrow un p2_down taustiņam Down Arrow.
  5. Izveido paddle.hpp un paddle.cpp ar klasi Paddle : public CharacterBody2D.
  6. Paddle klasē pievieno float speed = 400.0f, String input_up un String input_down.
  7. _bind_methods() piesaisti setter/getter metodes, lai input darbības varētu mainīt Godot Inspector panelī.
  8. _physics_process(double delta) nolasi Input::get_singleton() un aprēķini vertikālo virzienu.
  9. Iestati paddle velocity ar set_velocity(Vector2(0, direction * speed)) un izsauc move_and_slide().
  10. Reģistrē Paddle klasi register_types.cpp failā un pārkompilē.
  11. Godot editorā nomaini PaddleLeft un PaddleRight tipu vai pievieno tiem C++ klasi.
  12. PaddleLeft iestati input_up = "p1_up" un input_down = "p1_down".
  13. PaddleRight iestati input_up = "p2_up" un input_down = "p2_down".
  14. Izveido ball.hpp un ball.cpp ar klasi Ball : public CharacterBody2D.
  15. Ball klasē pievieno Vector2 velocity, float base_speed = 500.0f un reset_ball() metodi.
  16. _ready() metodē izsauc reset_ball(), lai bumba sākas centrā.
  17. _physics_process izmanto move_and_collide(velocity * delta).
  18. Sadursmes gadījumā aprēķini velocity = velocity.bounce(collision->get_normal()).
  19. Pēc katras paddle sadursmes nedaudz palielini ātrumu, piemēram, velocity *= 1.05f.
  20. Izveido game.hpp un game.cpp ar klasi Game : public Node.
  21. Game klasē glabā score_left, score_right, winning_score un norādes uz Label objektiem.
  22. Game::_ready() atrodi ScoreLeft, ScoreRight un Ball mezglus ar get_node.
  23. Ja bumbas x pozīcija ir mazāka par 0, pieskaiti punktu labajam spēlētājam.
  24. Ja bumbas x pozīcija ir lielāka par 1280, pieskaiti punktu kreisajam spēlētājam.
  25. Pēc punkta atjaunini abus Label tekstus un izsauc bumbas reset_ball().
  26. Pie winning_score punktiem apturi bumbu un izdrukā uzvarētāju Output panelī vai parādi Label.
  27. Pievieno restart darbību, piemēram, taustiņu R, kas atiestata punktus un bumbu.
  28. Veic vismaz 4 jēgpilnus commit: setup, paddle, ball, scoring/restart.
  29. README failā apraksti vadību, build komandu, klases un zināmās problēmas.
  30. Pēdējā pārbaudē palaid spēli no tīra Godot starta un pārliecinies, ka tā nav atkarīga no editorā atvērtas pagaidu scēnas.

Papildu izaicinājumi

Pievieno vienu uzlabojumu, kas neizjauc pamata spēli.

  • Izveido AIPaddle : public Paddle, kas seko bumbai tikai tad, kad bumba kustas tā virzienā.
  • Pievieno skaņas efektu paddle sadursmei un punktam, bet skaņas palaišanu izsauc no C++.
  • Pievieno pauzes režīmu ar bool paused, kas aptur bumbas un paddle kustību.

Biežākās kļūdas

  • Bumba iziet cauri paddle: Speed pārāk augsts; lieto move_and_collide nevis set_position.
  • Punkti netiek atjaunināti: Pārliecinies par get_node<Label>("ScoreLeft") references.
  • Build neizmainās: Izdzēs .sconsign.dblite un kompilē no jauna.

Godot ekrānuzņēmumi

Koda piemērs (paplašināts)

void Ball::reset_ball() {
    set_position(Vector2(640, 360));
    double y_speed = UtilityFunctions::randf_range(-180.0, 180.0);
    velocity = Vector2(base_speed, y_speed);
}

void Ball::_physics_process(double delta) {
    auto collision = move_and_collide(velocity * delta);
    if (collision.is_valid()) {
        velocity = velocity.bounce(collision->get_normal());
        velocity *= 1.05f;
    }
}

void Game::on_score(bool left_player) {
    if (left_player) score_left++;
    else score_right++;
    update_score_labels();
    ball->reset_ball();
}

void Game::update_score_labels() {
    score_left_label->set_text(String::num_int64(score_left));
    score_right_label->set_text(String::num_int64(score_right));
}
