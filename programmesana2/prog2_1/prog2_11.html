1.1 Godot instalācija un projekta sagatavošana

Tavs šīs stundas izaicinājums: Instalēt Godot 4, izveidot pirmo projektu un saprast Godot editor (redaktora) galvenās daļas.

SR 2.4.12. Programmēšanas vide SR 3.1.2. Darba vides riski

Teorija: Godot Engine un editor (redaktors)

Godot 4 Engine (Godot dzinējs) ir bezmaksas, atvērtā koda spēļu dzinējs ar visu nepieciešamo 2D un 3D spēļu izstrādei.

Editor (redaktora) galvenās daļas:

  • Scene Tree (Skatu koks) — kreisajā augšā; spēles objektu hierarhija.
  • FileSystem (Failu sistēma) — kreisajā apakšā; projekta faili.
  • Viewport (Skatlogs) — centrā; vizuālais redaktors.
  • Inspector (Inspektors) — labajā; aktīvā Node īpašības.
  • Output / Debugger (Atkļūdotājs) — apakšā; izvades konsole.
# Godot 4 instalācija
# 1. Lejupielādē no godotengine.org/download
# 2. Izpako ZIP
# 3. Palaiž godot.exe (Windows) vai ./godot (Linux/Mac)
# 4. Project Manager → New Project

1. uzdevums: Godot instalācija

Lejupielādē un instalē Godot 4 stable versiju.

Izpildes soļi:

  1. Atver godotengine.org/download un lejupielādē Godot 4 Standard versiju.
  2. Izpako ZIP arhīvu mapē C:\Tools\Godot (Windows) vai ~/Tools/Godot (Linux/Mac).
  3. Palaiž godot.exe — atvēras Project Manager (Projektu pārvaldnieks).
  4. Spied New Project → nosaukums Pong, mape ~/Projects/Pong, Renderer: Forward+.
  5. Spied Create & Edit.

2. uzdevums: Editor iepazīšana

Izpēti galvenās editor daļas un saglabā tukšo scēnu.

Izpildes soļi:

  1. Apskati Scene Tree (Skatu koks) un FileSystem (Failu sistēma) panes kreisajā pusē.
  2. Centrā ir Viewport (Skatlogs) — pamēģini ar peles ritentiņu pietuvot/attālināt.
  3. Augšā: 2D, 3D un Script (Skripts) workspace pogas.
  4. Editor → Editor Settings → Interface → Editor → Editor Language: en.
  5. Saglabā tukšu scēnu: Scene → Save Scene As → main.tscn.

3. uzdevums: Versiju kontrole ar Git

Sagatavo projektu Git pārvaldībai.

Izpildes soļi:

  1. Terminālī projekta mapē: git init.
  2. Izveido .gitignore:
    .godot/
    .import/
    *.tmp
    builds/
  3. Pievieno failus: git add . un commit: git commit -m "Pirmais Godot projekts"
  4. Izveido GitHub repo un pievieno remote: git remote add origin URL; git push -u origin main

Papildus uzdevums: Editor paplašinājumi

Instalē noderīgus uzlabojumus.

Izpildes soļi:

  1. AssetLib cilnē meklē noderīgus papildinājumus.
  2. Layout (Izkārtojums) → Save → pielāgots tavām vajadzībām.
  3. Help → Search Help (Ctrl+F1) — Godot dokumentācija pieejama tieši editorā.

Biežākās kļūdas

  • Editor neatveras: Pārbaudi, vai sistēma atbalsta Godot 4 (Windows 10+, macOS 10.15+, Linux).
  • Lēna darbība: Forward+ prasa atjauninātus grafikas draiverus. Vecākiem datoriem mēģini Compatibility.
  • Saglabājot crashojas: Mapes ceļš nedrīkst saturēt latīņu burtus.
