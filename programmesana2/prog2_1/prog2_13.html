1.3 C++ un GDExtension sagatavošana

Tavs šīs stundas izaicinājums: Sagatavot C++ izstrādes vidi un kompilatorus, lai varētu rakstīt C++ kodu Godot.

SR 2.4.12. Programmēšanas vide SR 2.4.2. Argumentēti izvēlas rīkus

Teorija: GDExtension un C++ build sistēma

GDExtension ir Godot 4 mehānisms, kas ļauj rakstīt C++ kodu, kas darbojas spēlē. C++ kods tiek kompilēts par koplietojamu bibliotēku (.so Linux, .dll Windows), ko Godot ielādē izpildes laikā.

Kāpēc C++?

  • Veiktspēja: 5–50× ātrāks par GDScript intensīvos algoritmos.
  • Industrijas standarts: Lielākās spēles tiek rakstītas C/C++.
  • Kontrole: Tieša piekļuve atmiņai un datu tipiem.

Nepieciešamie rīki: Compiler (kompilators), SCons, godot-cpp.

# Atkarību instalēšana
# Linux (Debian/Ubuntu):
sudo apt install build-essential scons git python3

# Windows: Visual Studio Community ar "Desktop development with C++"
pip install scons

# macOS:
brew install scons

1. uzdevums: Build rīku instalācija

Sagatavo sistēmu C++ kompilācijai.

Izpildes soļi:

  1. Pārbaudi C++ kompilatoru: Linux/Mac g++ --version; Windows cl /? Developer CMD.
  2. Instalē Python 3 (ja nav).
  3. pip install scons — SCons build sistēma.
  4. Pārbaudi: scons --version — vajadzētu rādīt 4.0+.
  5. Linux: sudo apt install build-essential.

2. uzdevums: godot-cpp klonēšana

Lejupielādē oficiālo C++ binding.

Izpildes soļi:

  1. Projekta saknē izveido apakšmapi: mkdir gdext && cd gdext.
  2. Klonē kā submodule: git submodule add -b 4.2 https://github.com/godotengine/godot-cpp.git
  3. git submodule update --init
  4. Pārliecinies, ka mapē gdext/godot-cpp ir saturs.
  5. Commit: git commit -am "Pievienots godot-cpp"

3. uzdevums: SConstruct un kompilācija

Konfigurē build skriptu un kompilē.

Izpildes soļi:

  1. Failā gdext/SConstruct ievieto SCons konfigurāciju (skat. koda piemēru).
  2. Failā gdext/src/register_types.cpp izveido empty boilerplate (paplašināsim 1.4 stundā).
  3. Mapē gdext/: scons platform=linuxbsd (vai windows/macos).
  4. Tipiskā kompilācijas laiks pirmajā reizē: 2–10 minūtes.

Papildus uzdevums: VS Code konfigurācija

Iestati VS Code C++ izstrādei.

Izpildes soļi:

  1. code . projektā.
  2. Instalē extensions: C/C++ (Microsoft) un Godot Tools.
  3. .vscode/c_cpp_properties.json — pievieno include path: gdext/godot-cpp/include.

Biežākās kļūdas

  • scons: command not found: SCons nav uzstādīts. pip install --user scons.
  • Linker errors par godot-cpp: Vispirms jākompilē godot-cpp pati. cd godot-cpp && scons.
  • Visual Studio nav atrasts (Windows): Palaiž SCons no Developer Command Prompt.

Godot ekrānuzņēmumi

Koda piemērs (paplašināts)

# gdext/SConstruct
import os

env = SConscript("godot-cpp/SConstruct")

env.Append(CPPPATH=["src/"])
sources = Glob("src/*.cpp")

if env["platform"] == "macos":
    library = env.SharedLibrary(
        "../bin/libpong_ext.{}.{}.framework/libpong_ext.{}.{}".format(
            env["platform"], env["target"], env["platform"], env["target"]),
        source=sources,
    )
else:
    library = env.SharedLibrary(
        "../bin/libpong_ext{}{}".format(env["suffix"], env["SHLIBSUFFIX"]),
        source=sources,
    )

Default(library)
Pirmā veiksmīga SCons kompilācija ar 'scons: done building targets.' ziņojumu.
