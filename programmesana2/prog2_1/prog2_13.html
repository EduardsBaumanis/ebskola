1.3 C++ un GDExtension sagatavošana

Tavs šīs stundas izaicinājums: Sagatavot C++ izstrādes vidi un kompilatorus, lai varētu rakstīt C++ kodu Godot.

SR 2.4.12. Programmēšanas vide SR 2.4.2. Argumentēti izvēlas rīkus

Teorija: GDExtension un C++ build sistēma

GDExtension ir Godot 4 mehānisms, kas ļauj rakstīt C++ kodu, kas darbojas spēlē. C++ kods tiek kompilēts par koplietojamu bibliotēku (.so Linux, .dll Windows, .dylib macOS), ko Godot ielādē izpildes laikā.

Šajā kursā spēles skripti ir C++ klases. SConstruct ir build konfigurācija, termināļa komandas ir tikai rīku palaišana, bet spēles uzvedību rakstām .hpp un .cpp failos.

Kāpēc C++?

  • Veiktspēja: piemērots intensīvai fizikai, datu apstrādei un AI algoritmiem.
  • Industrijas standarts: C++ sintakse, kompilācija un moduļu dalījums ir bieži sastopami spēļu izstrādē.
  • Kontrole: skaidri tipi, galvenes, implementācijas faili un precīza bibliotēku sasaiste.

Nepieciešamie rīki: C++ kompilators, Python, SCons, Git un godot-cpp bindings. godot-cpp versijai jāatbilst Godot 4 versijai, ko lieto projektā.

# Atkarību instalēšana
# Linux (Debian/Ubuntu):
sudo apt install build-essential scons git python3

# Windows: Visual Studio Community ar "Desktop development with C++"
pip install scons

# macOS:
brew install scons

Praktiskais uzdevums: C++ GDExtension sagatavošana

Sagatavo projektu tā, lai Godot varētu ielādēt kompilētu C++ bibliotēku.

Izpildes soļi:

  1. Pārbaudi C++ kompilatoru: Linux/macOS terminālī g++ --version, Windows Developer Command Prompt vidē cl /?.
  2. Ja kompilators nav pieejams, instalē atbilstošu rīku: build-essential, Xcode Command Line Tools vai Visual Studio C++ workload.
  3. Pārbaudi Python 3 ar python3 --version vai python --version.
  4. Instalē SCons ar pip install --user scons vai sistēmas pakotņu pārvaldnieku.
  5. Pārbaudi SCons ar scons --version.
  6. Projekta saknē izveido mapi gdext, ja tā vēl nepastāv.
  7. Mapē gdext izveido apakšmapes src un bin.
  8. Pievieno godot-cpp kā Git submodule, izvēloties branch, kas sakrīt ar tavas Godot 4 versijas līniju.
  9. Palaid git submodule update --init --recursive, lai lejupielādētu bindings saturu.
  10. Izveido gdext/SConstruct build konfigurāciju, kas kompilē visus src/*.cpp failus.
  11. Izveido gdext/src/register_types.hpp ar inicializācijas funkciju deklarācijām.
  12. Izveido gdext/src/register_types.cpp ar initialize_pong_module un uninitialize_pong_module.
  13. Pagaidām reģistrēšanas funkcijās neatstāj spēles loģiku; tajās tikai pieslēdz klases Godot sistēmai.
  14. Projekta saknē izveido pong_ext.gdextension failu ar ceļu uz kompilēto bibliotēku.
  15. No gdext mapes palaid scons platform=linuxbsd, scons platform=windows vai scons platform=macos.
  16. Pārbaudi, ka mapē bin parādās koplietojamā bibliotēka.
  17. Atver Godot projektu un pārliecinies, ka Output panelī nav GDExtension ielādes kļūdu.
  18. Veic commit ar ziņu Sagatavota C++ GDExtension vide.
  19. README failā pieraksti precīzu komandu, ar kuru tev izdevās kompilēt projektu.

Papildu izaicinājumi

Iestati ērtāku C++ rediģēšanu.

  • Atver projektu VS Code ar code ..
  • Instalē C++ paplašinājumu un pievieno include ceļus uz gdext/godot-cpp/include un ģenerētajiem bindings failiem.
  • Pievieno VS Code task, kas palaiž to pašu scons komandu, ko ierakstīji README.

Biežākās kļūdas

  • scons: command not found: SCons nav uzstādīts. pip install --user scons.
  • Linker errors par godot-cpp: Vispirms jākompilē godot-cpp pati. cd godot-cpp && scons.
  • Visual Studio nav atrasts (Windows): Palaiž SCons no Developer Command Prompt.

Godot ekrānuzņēmumi

Koda piemērs (paplašināts)

// gdext/src/register_types.cpp
#include "register_types.hpp"
#include "hello.hpp"

#include <gdextension_interface.h>
#include <godot_cpp/core/class_db.hpp>
#include <godot_cpp/core/defs.hpp>
#include <godot_cpp/godot.hpp>

using namespace godot;

void initialize_pong_module(ModuleInitializationLevel level) {
    if (level != MODULE_INITIALIZATION_LEVEL_SCENE) {
        return;
    }

    ClassDB::register_class<Hello>();
}

void uninitialize_pong_module(ModuleInitializationLevel level) {
    if (level != MODULE_INITIALIZATION_LEVEL_SCENE) {
        return;
    }
}

extern "C" {
GDExtensionBool GDE_EXPORT pong_library_init(
    GDExtensionInterfaceGetProcAddress get_proc_address,
    GDExtensionClassLibraryPtr library,
    GDExtensionInitialization *initialization) {
    GDExtensionBinding::InitObject init_obj(get_proc_address, library, initialization);
    init_obj.register_initializer(initialize_pong_module);
    init_obj.register_terminator(uninitialize_pong_module);
    init_obj.set_minimum_library_initialization_level(MODULE_INITIALIZATION_LEVEL_SCENE);
    return init_obj.init();
}
}
