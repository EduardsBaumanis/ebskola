1.4 Pirmais C++ skripts: mainīgie un izvade

Tavs šīs stundas izaicinājums: Uzrakstīt pirmo strādājošu C++ klasi un attiecināt to uz Godot Node.

SR 2.4.13. Mainīgie un datu tipi SR 2.4.15. Modulāras programmas

Teorija: C++ klases struktūra GDExtension ietvaros

Godot C++ klase tiek definēta divos failos: .hpp (header) un .cpp (implementation). Klasei jāmanto no Godot Node un jāizmanto GDCLASS makro.

// hello.hpp
#pragma once
#include <godot_cpp/classes/node.hpp>

namespace godot {

class Hello : public Node {
    GDCLASS(Hello, Node)

private:
    int counter = 0;

protected:
    static void _bind_methods();

public:
    Hello();
    ~Hello();
    void _ready() override;
    void say_hello();
};

}  // namespace godot

Pamata C++ datu tipi:

TipsApraksts
intVesels skaitlis
floatDecimālskaitlis 32-bit
booltrue/false
StringGodot virkne
Vector22D vektors (x, y)

1. uzdevums: Header fails

Izveido pirmo C++ klases header failu.

Izpildes soļi:

  1. gdext/src/hello.hpp — iekopē klases definīciju no teorijas.
  2. Saglabā un pārbaudi, ka VS Code neuzrāda kļūdas.
  3. Pievieno komentārus pie katras rindas, lai paskaidrotu nozīmi.

2. uzdevums: Implementation fails

Implementē klases metodes.

Izpildes soļi:

  1. hello.cpp tajā pašā mapē.
  2. Pievieno: #include "hello.hpp", #include <godot_cpp/core/class_db.hpp>, #include <godot_cpp/variant/utility_functions.hpp>
  3. Implementē konstruktoru, destruktoru, _bind_methods un _ready (skat. apakšējo piemēru).
  4. Kompilē: scons platform=linuxbsd.

3. uzdevums: Reģistrācija un testēšana

Reģistrē klasi un izmēģini.

Izpildes soļi:

  1. register_types.cpp: pievieno ClassDB::register_class<Hello>();
  2. Failā pong_ext.gdextension projekta saknē — bibliotēkas ceļa konfigurācija.
  3. Pārkompilē, atver Godot — Hello node parādīsies Add Child Node sarakstā.
  4. Pievieno scēnai, palaiž (F5), pārbaudi Output panelī "Hello, Godot!".

Papildus uzdevums: Mainīgo eksperiments

Eksperimentē ar dažādiem datu tipiem.

Izpildes soļi:

  1. Pievieno locekļus: String player_name = "Anna";, float score = 0.0f;, Vector2 position;.
  2. _ready(): UtilityFunctions::print("Spēlētājs: ", player_name);
  3. Pārkompilē un pārbaudi izvadi.

Biežākās kļūdas

  • error: 'GDCLASS' was not declared: Trūkst include statement.
  • Klase neparādās Godot: Aizmirsts register_class.
  • Crash, palaižot: .gdextension fails nepareizi konfigurēts.

Godot ekrānuzņēmumi

Koda piemērs (paplašināts)

// hello.cpp
#include "hello.hpp"
#include <godot_cpp/core/class_db.hpp>
#include <godot_cpp/variant/utility_functions.hpp>

using namespace godot;

void Hello::_bind_methods() {
    ClassDB::bind_method(D_METHOD("say_hello"), &Hello::say_hello);
}

Hello::Hello() { counter = 0; }
Hello::~Hello() {}

void Hello::_ready() {
    UtilityFunctions::print("Hello, Godot! No C++.");
    say_hello();
}

void Hello::say_hello() {
    counter++;
    UtilityFunctions::print("Sveicieni Nr. ", counter);
}
Godot Output panelis ar 'Hello, Godot!' un 'Sveicieni Nr. 1' tekstu.
