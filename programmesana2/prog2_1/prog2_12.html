1.2 Scene Tree un Node sistēma

Tavs šīs stundas izaicinājums: Saprast, kā Godot organizē objektus Node hierarhijā un izveidot pirmo strādājošu scēnu.

SR 2.4.13. Datu tipi SR 2.4.15. Modulāras programmas

Teorija: Node un Scene konceptu

Godot fundamentālais princips: viss ir Node (Mezgls). Spēles laukums, varonis, poga, skaņa — viss ir Node, savienots hierarhiskā kokā jeb Scene (Skatu).

Galvenie Node tipi 2D spēlēs:

  • Node2D — bāzes klase 2D objektiem ar pozīciju, rotāciju, mērogu.
  • Sprite2D — rāda 2D attēlu (texture).
  • CollisionShape2D — sadursmes forma.
  • CharacterBody2D — kustīgs objekts ar fiziku (varonis).
  • Area2D — detektē objektus, kas šķērso zonu.
  • Camera2D — kameras skats.
// Pong scēnas struktūra:
// Main (Node2D)
// ├── Background (ColorRect)
// ├── PaddleLeft (CharacterBody2D)
// │   ├── Sprite2D
// │   └── CollisionShape2D
// ├── PaddleRight (CharacterBody2D)
// └── Ball (CharacterBody2D)
//     ├── Sprite2D
//     └── CollisionShape2D

Bērnu nodes pārmanto vecāku transformāciju.

1. uzdevums: Pong scēnas izveide

Izveido Pong spēles laukuma struktūru.

Izpildes soļi:

  1. Atver main.tscn. Spied + (Add Child Node) → Node2D, pārsauc par Main.
  2. Pievieno ColorRect kā bērnu (fons). Inspector → Color: melna, Size: 1280×720.
  3. Pievieno CharacterBody2D, pārsauc par PaddleLeft.
  4. PaddleLeft bērniem pievieno Sprite2D un CollisionShape2D.
  5. Sprite2D Texture → New PlaceholderTexture2D → Size: 20×100. CollisionShape2D Shape → New RectangleShape2D → Size: 20×100.

2. uzdevums: Bumba un pretinieks

Pievieno bumbu un labo paddle.

Izpildes soļi:

  1. Right-click PaddleLeft → Duplicate. Pārsauc par PaddleRight. Position X: 1240.
  2. Pievieno jaunu CharacterBody2D kā Main bērnu, pārsauc par Ball.
  3. Ball bērniem pievieno Sprite2D (PlaceholderTexture 20×20) un CollisionShape2D (RectangleShape 20×20).
  4. Pārvieto Ball uz centru: Position 640, 360.
  5. Spied F5, lai palaistu — laukums būs redzams.

3. uzdevums: Atkārtoti lietojamas scēnas

Saglabā Ball un Paddle kā atsevišķas scēnas.

Izpildes soļi:

  1. Right-click Ball → Save Branch as Scene → Ball.tscn.
  2. Atkārto ar PaddleLeft → Paddle.tscn.
  3. Izdzēš PaddleRight, izveido jaunu instanci no Paddle.tscn (Instantiate Child Scene poga).
  4. Pārliecinies, ka spēle joprojām palaižas korekti.

Papildus uzdevums: Groups un Tags

Iepazīsties ar grupu sistēmu.

Izpildes soļi:

  1. Atlasi Ball Node Tree.
  2. Inspector blakus → Node tab → Groups → Add Group → balls.
  3. Atkārto Paddle nodes ar grupu paddles.

Biežākās kļūdas

  • Bērns nepārvietojas ar vecāku: Pārliecinies, ka Sprite2D un CollisionShape2D ir tieši zem CharacterBody2D.
  • CollisionShape2D rāda dzeltenu trīsstūri: Trūkst Shape resursa.
  • Sprite nav redzams: Texture lauks tukšs.

[node name="Background" type="ColorRect" parent="."]
size = Vector2(1280, 720)
color = Color(0, 0, 0, 1)

[node name="PaddleLeft" parent="." instance=ExtResource("Paddle.tscn")]
position = Vector2(40, 360)

[node name="PaddleRight" parent="." instance=ExtResource("Paddle.tscn")]
position = Vector2(1240, 360)

[node name="Ball" parent="." instance=ExtResource("Ball.tscn")]
position = Vector2(640, 360)
