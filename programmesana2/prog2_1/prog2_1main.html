1. Tēma — Ievads Godot un C++

Par šo tēmu

Pirmā tēma iepazīstina ar Godot 4 Engine (Godot dzinēju) un C++ programmēšanu caur GDExtension. Apgūsi spēļu dzinēja darba vidi (editor), Scene Tree (Skatu koks) konceptu, C++ build procesu un klases reģistrāciju Godot vidē. Tēma noslēdzas ar klasiskās arkādes spēles Pong izveidi.

Šajā tēmā spēles loģika tiek rakstīta C++. Godot editorā var redzēt vārdu Script, taču šajā kursā tas nozīmē GDExtension C++ klasi.

Kritiskā analīze

1. tēmas galvenais uzdevums ir pārvērst Godot no "klikšķināma redaktora" par programmējamu sistēmu. Ja skolēns tikai saliek objektus ekrānā, tēma paliek virspusēja; ja skolēns saprot, kā Scene Tree, C++ klase, kompilācija un Godot resursi sadarbojas, tad turpmākās tēmas kļūst daudz drošākas.

  • Stiprā puse: Pong ir mazs projekts ar skaidru rezultātu, tāpēc tajā var fokusēties uz C++ klases struktūru, mezglu hierarhiju, ievadi, kustību un punktu uzskaiti.
  • Riska punkts: GDExtension prasa precīzu versiju un build rīku saderību. Tāpēc katrā stundā jābūt pārbaudes soļiem, ne tikai "uzraksti kodu".
  • Pedagoģiska izvēle: teorijai jāsaista editorā redzamais ar C++ API. Skolēnam jāredz, ka Node2D, CharacterBody2D, Label un Input nav abstrakti vārdi, bet tipi, ar kuriem strādā C++ kodā.
  • Vērtēšanas kritērijs: projekta beigās jābūt palaižamai Pong spēlei, kur spēles uzvedību nosaka C++ klases, nevis tikai manuāli salikti editor iestatījumi.

Tēmas projekts: Pong

Tēmas noslēgumā Tu izveidosi Arcade žanra spēli — Pong, izmantojot visu, ko šajā tēmā iemācījies. Spēli izstrādāsi Godot 4 dzinējā ar C++ koda loģiku caur GDExtension.

Minimālais pierādījums: pielāgotas C++ klases parādās Godot node sarakstā, projekts kompilējas ar SCons, bumba un paddle kustas ar C++ kodu, punkti atjaunojas UI elementos, un README apraksta build/palaišanas soļus.
