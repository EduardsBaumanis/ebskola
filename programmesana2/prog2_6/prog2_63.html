6.3 Audio sistēma

Tavs šīs stundas izaicinājums: Pievienot mūziku, skaņas efektus un audio buses spēlei.

SR 2.4.7. UI SR 2.4.15. Modulāras programmas

Priekšzināšanu robeža: uzdevumos izmanto tikai iepriekš apgūto un šīs lapas teorijas/koda piemēros parādīto. Ja parādās jauna komanda vai rīks, vispirms atrod tās paraugu teorijas blokā.

Teorija: Godot audio

Godot audio sistēma izmanto AudioStreamPlayer nodes ar audio bus arhitektūru.

#include <godot_cpp/classes/audio_stream_player.hpp>
#include <godot_cpp/classes/resource_loader.hpp>

// Atskaņot SFX
AudioStreamPlayer* sfx = get_node<AudioStreamPlayer>("SFX");
sfx->set_stream(load_audio("res://sounds/jump.ogg"));
sfx->play();

// Mūzika ar fade-in
void play_music_fade_in(String path) {
    music_player->set_stream(load_audio(path));
    music_player->set_volume_db(-30);
    music_player->play();
    Tween* t = create_tween();
    t->tween_method("set_volume_db", -30, -10, 2.0);  // fade up
}

Audio buses: Master → Music, SFX, Voice. Lietotājs var pielāgot katru atsevišķi.

Teorijas paplašinājums: C++ fokuss

Kritiskais fokuss: šajā stundā nepietiek tikai panākt redzamu efektu editorā. Jāspēj paskaidrot, kura C++ klase glabā stāvokli, kura metode to maina un kā Godot node struktūra šo kodu izmanto.

Pārbaudes pierādījums: UI signāli, audio darbības, debug izvade un publicēšanas sagatave ir sasaistīta ar C++ spēles stāvokli.

Konsolidētais praktiskais uzdevums (15+ soļi)

70 min darba sadalījums: 1. uzdevums (~20 min) — izveido minimālo piemēru, kas pārbauda šīs stundas galveno ideju; 2. uzdevums (~25 min) — integrē risinājumu tēmas spēles projektā; 3. uzdevums (~25 min) — notestē robežgadījumus, salabo kļūdas un pieraksti secinājumu README vai komentārā. Tālāk redzamie soļi ir detalizēta karte šiem trim uzdevumiem; ja laiks beidzas, papildu pulēšanu atstāj nākamajai reizei.

Izpildi šo darba plānu, lai 6.3 Audio sistēma rezultāts būtu pārbaudāms C++ GDExtension projektā.

Izpildes soļi:

  1. Atver projektu un izveido Git branch UI, audio, debug vai publicēšanas darbam.
  2. Pieraksti lietotāja ceļu: ko spēlētājs redz, nospiež, dzird vai lejupielādē.
  3. Pārbaudi, ka galvenā spēles stāvokļa loģika joprojām ir C++ klasēs.
  4. Izveido vai atjaunini C++ klasi, kas savieno UI/audio/publicēšanas stāvokli ar spēli.
  5. Ja lieto UI pogas, pieslēdz signālus ar Callable vai reģistrētu C++ metodi.
  6. Ja lieto audio, nosauc busus un skaņu tipus tā, lai tos var saprast pēc nosaukuma.
  7. Ja lieto animācijas, nodali vizuālo efektu no spēles noteikumu aprēķina.
  8. Pievieno debug izvadi tikai vietās, kur tā palīdz atrast stāvokļa kļūdu.
  9. Pārbaudi kļūdu scenāriju: trūkst audio, nav save faila, poga nav pieslēgta vai export fails nav izveidots.
  10. Pārkompilē C++ kodu ar SCons pirms testēšanas Godot vidē.
  11. Palaid spēli no sākuma ekrāna līdz galvenajai darbībai, neizlaižot UI ceļu.
  12. Pārbaudi skaļumu, pauzi vai iestatījumus vismaz divas reizes vienā sesijā.
  13. Ja šī ir eksporta stunda, izveido tīru build mapi un eksportē projektu no jauna.
  14. Pārbaudi, ka eksportā iekļauta GDExtension bibliotēka vajadzīgajai platformai.
  15. Pārbaudi gala rezultātu ārpus Godot editora, piemēram, palaižot eksportēto failu.
  16. README pieraksti lietotāja vadību, zināmos ierobežojumus un build/publish soļus.
  17. Noņem lieku debug izvadi, kas spēlētājam nav vajadzīga.
  18. Veic commit ar ziņu, kas nosauc UI, audio, debug vai publicēšanas izmaiņu.

Aktivitāte 1: Audio bus iestatīšana

Arhitektūra.

Izpildes soļi:

  1. Project → Audio Bus.
  2. Pievieno 3 sub-buses: Music, SFX, Voice (visi route uz Master).
  3. AudioStreamPlayer Inspector → Bus: izvēlies attiecīgo.
  4. Slider settings UI — saistīts ar AudioServer::set_bus_volume_db().

Aktivitāte 2: SFX sound effects

Spēles skaņas.

Izpildes soļi:

  1. AudioStreamPlayer pool — līdzīgi bullet pool no 4. tēmas.
  2. Funkcija play_sfx(String path): paņem free player, set_stream, play.
  3. Skaņas: jump, hit, coin pickup, button click, level complete.
  4. Bezmaksas skaņas: opengameart.org vai freesound.org.

Aktivitāte 3: Music sistēma

Background mūzika.

Izpildes soļi:

  1. Music AudioStreamPlayer ar attribute autoplay.
  2. Crossfade starp tracks: ielādē jauno trak, vienlaikus fade-in jauno + fade-out veco.
  3. Different music katram līmenim.
  4. Pause menu: lower music volume.

Papildu izaicinājums: 3D positional audio

Skaņa pielāgojas pozīcijai.

Izpildes soļi:

  1. AudioStreamPlayer2D node — automātiski pielāgo volume un panning.
  2. Eksplozija tālumā skan klusāk un no attiecīgās puses.
  3. Setup max_distance, attenuation.

Biežākās kļūdas

  • Skaņa nestrādā: stream nav ielādēts vai bus mute.
  • Lag music start: OGG fails liels — preload ResourceLoader.
  • SFX cuts off: Lieto pool, lai daudzi vienlaikus.

Godot ekrānuzņēmumi

Koda piemērs (paplašināts)

class AudioManager : public Node {
    GDCLASS(AudioManager, Node)
private:
    std::vector<AudioStreamPlayer*> sfx_pool;
    AudioStreamPlayer* music_player;
    static const int POOL_SIZE = 8;

public:
    void _ready() override {
        for (int i = 0; i < POOL_SIZE; i++) {
            AudioStreamPlayer* p = memnew(AudioStreamPlayer);
            p->set_bus("SFX");
            add_child(p);
            sfx_pool.push_back(p);
        }
        music_player = memnew(AudioStreamPlayer);
        music_player->set_bus("Music");
        add_child(music_player);
    }

    void play_sfx(String path) {
        for (auto p : sfx_pool) {
            if (!p->is_playing()) {
                Ref<AudioStream> stream = ResourceLoader::get_singleton()->load(path);
                p->set_stream(stream);
                p->play();
                return;
            }
        }
    }

    void play_music(String path) {
        Ref<AudioStream> stream = ResourceLoader::get_singleton()->load(path);
        music_player->set_stream(stream);
        music_player->play();
    }
};
