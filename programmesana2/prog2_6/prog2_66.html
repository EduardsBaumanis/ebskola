6.6 Noslēguma projekts: Puzzle spēle

Tavs šīs stundas izaicinājums: Izveidot un publicēt pilnvērtīgu puzzle spēli ar visu apgūto.

SR 2.4.1. Problēmu dekompozīcija SR 2.4.7. UI SR 2.4.6. Plāno SR 3.1.4. Licences

Teorija: Spēles apraksts

Puzzle spēle: spēlētājs risina sēriju problēmu, kā Sokoban, Tetris vai krāsu match-3.

Izvēlies vienu no opcijām (vai izveido savu):

  1. Sokoban — bīdi kastes uz mērķa vietām.
  2. Match-3 — savieno 3+ vienādas krāsas.
  3. Tetris klons — krītošie bloki.
  4. Sliding puzzle — 4×4 attēla mozaīka.

Prasības:

  • Sākuma menu, settings, augstākie rezultāti.
  • Vismaz 10 līmeņi (Sokoban) vai endless mode (Match-3, Tetris).
  • Audio: mūzika + 5+ SFX.
  • Animācijas/Tween efekti.
  • Save sistēma (highscore + progress).
  • Eksportēta uz Web un publicēta GitHub Pages.

1. uzdevums: Spēles loģika (1.)

Pamata mehānika.

Izpildes soļi:

  1. Izvēlies spēles žanru un implementē core mehāniku C++.
  2. Sokoban: 2D grid, kastes pārvietošana, mērķu pārbaude.
  3. Match-3: grid, swap mechanic, match detection algoritms.
  4. 5 sākuma līmeņi vai endless mode.

2. uzdevums: UI un audio (2.)

Polish.

Izpildes soļi:

  1. Sākuma menu (no 6.1).
  2. Audio sistēma (no 6.3).
  3. Animācijas: ievietošana ar Tween, particle effects pie match.
  4. HUD ar score, time, moves remaining.
  5. Pause un game over screens.

3. uzdevums: Save un publicēšana (3.)

Galaprodukcija.

Izpildes soļi:

  1. Highscore JSON save (no 5. tēmas).
  2. 10 līmeņi vai 5 dažādas grūtības levels.
  3. Eksports uz Web (no 6.5).
  4. Publicēšana GitHub Pages.
  5. README ar play link, screenshots, controls.
  6. GitHub: 12+ commits, semantic versioning (v1.0.0).

Papildus uzdevums: Multiplayer

Iemēģini multiplayer.

Izpildes soļi:

  1. Lokāls 2-player split screen.
  2. Vai online ar Godot's High-Level Multiplayer API.
  3. Leaderboards no kopējās DB.

Biežākās kļūdas

  • Web build slow: Optimize assets, mazāk particles, lower texture resolution.
  • Save fails neielādējas web vidē: Web localStorage atšķirās — pārbaudi, ka use_localstorage iestatīts.
  • Mobile controls don't work: Pievieno touch input handlers.

Koda piemērs (paplašināts)

// Match-3 spēles core
class Match3Board : public Node2D {
    GDCLASS(Match3Board, Node2D)
private:
    static const int W = 8, H = 8;
    int grid[W][H];

public:
    void initialize() {
        for (int x = 0; x < W; x++)
            for (int y = 0; y < H; y++)
                grid[x][y] = std::rand() % 5;  // 5 colors
    }

    std::vector<Vector2i> find_matches() {
        std::vector<Vector2i> matches;
        for (int x = 0; x < W; x++) {
            for (int y = 0; y < H - 2; y++) {
                if (grid[x][y] == grid[x][y+1] && grid[x][y+1] == grid[x][y+2]) {
                    matches.push_back({x, y});
                    matches.push_back({x, y+1});
                    matches.push_back({x, y+2});
                }
            }
        }
        // similarly for horizontal
        return matches;
    }

    void clear_and_drop(const std::vector<Vector2i>& matches) {
        // mark cleared, gravity, refill
    }
};
