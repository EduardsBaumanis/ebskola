6.5 Eksports un publicēšana

Tavs šīs stundas izaicinājums: Eksportēt spēli uz vairākām platformām un publicēt internetā.

SR 2.4.6. Plāno izstrādes procesu SR 3.1.4. IP tiesības

Teorija: Export presets

Godot atbalsta eksportu uz: Windows, Linux, macOS, Web (HTML5), Android, iOS un konsolēm.

# Web eksporta soļi
# 1. Project → Export → Add → Web
# 2. Lejupielādē export templates: Editor → Manage Export Templates
# 3. Custom HTML — par default labi
# 4. Export Project → save as index.html mapē web_build/
# 5. web_build/ satur: index.html, .pck, .wasm, .js

# Lokāli testē:
python3 -m http.server 8000
# Atver http://localhost:8000

GitHub Pages publicēšana: ielādē web_build saturu uz gh-pages branch un GitHub automātiski hosto.

Teorijas paplašinājums: C++ fokuss

Kritiskais fokuss: šajā stundā nepietiek tikai panākt redzamu efektu editorā. Jāspēj paskaidrot, kura C++ klase glabā stāvokli, kura metode to maina un kā Godot node struktūra šo kodu izmanto.

Pārbaudes pierādījums: UI signāli, audio darbības, debug izvade un publicēšanas sagatave ir sasaistīta ar C++ spēles stāvokli.

#include <godot_cpp/variant/string.hpp>

using namespace godot;

struct PolishCheckpoint {

Konsolidētais praktiskais uzdevums (15+ soļi)

Izpildi šo darba plānu, lai 6.5 Eksports un publicēšana rezultāts būtu pārbaudāms C++ GDExtension projektā.

Izpildes soļi:

  1. Atver projektu un izveido Git branch UI, audio, debug vai publicēšanas darbam.
  2. Pieraksti lietotāja ceļu: ko spēlētājs redz, nospiež, dzird vai lejupielādē.
  3. Pārbaudi, ka galvenā spēles stāvokļa loģika joprojām ir C++ klasēs.
  4. Izveido vai atjaunini C++ klasi, kas savieno UI/audio/publicēšanas stāvokli ar spēli.
  5. Ja lieto UI pogas, pieslēdz signālus ar Callable vai reģistrētu C++ metodi.
  6. Ja lieto audio, nosauc busus un skaņu tipus tā, lai tos var saprast pēc nosaukuma.
  7. Ja lieto animācijas, nodali vizuālo efektu no spēles noteikumu aprēķina.
  8. Pievieno debug izvadi tikai vietās, kur tā palīdz atrast stāvokļa kļūdu.
  9. Pārbaudi kļūdu scenāriju: trūkst audio, nav save faila, poga nav pieslēgta vai export fails nav izveidots.
  10. Pārkompilē C++ kodu ar SCons pirms testēšanas Godot vidē.
  11. Palaid spēli no sākuma ekrāna līdz galvenajai darbībai, neizlaižot UI ceļu.
  12. Pārbaudi skaļumu, pauzi vai iestatījumus vismaz divas reizes vienā sesijā.
  13. Ja šī ir eksporta stunda, izveido tīru build mapi un eksportē projektu no jauna.
  14. Pārbaudi, ka eksportā iekļauta GDExtension bibliotēka vajadzīgajai platformai.
  15. Pārbaudi gala rezultātu ārpus Godot editora, piemēram, palaižot eksportēto failu.
  16. README pieraksti lietotāja vadību, zināmos ierobežojumus un build/publish soļus.
  17. Noņem lieku debug izvadi, kas spēlētājam nav vajadzīga.
  18. Veic commit ar ziņu, kas nosauc UI, audio, debug vai publicēšanas izmaiņu.

Aktivitāte 1: Windows export

Sagatavo Windows .exe.

Izpildes soļi:

  1. Project → Export → Add → Windows Desktop.
  2. Iestati Application name, Icon (.ico).
  3. Export Project → save as my_game.exe.
  4. Iemēģini palaist citā datorā — vajadzētu darboties bez Godot.
  5. Pievieno installer (Inno Setup vai NSIS) advanced lietotājiem.

Aktivitāte 2: Web export

HTML5 publicēšana.

Izpildes soļi:

  1. Editor → Manage Export Templates → Download.
  2. Project → Export → Add → Web.
  3. Iestati canvas size, threading.
  4. Export → web_build/index.html.
  5. Lokāli testē: cd web_build && python3 -m http.server 8000

Aktivitāte 3: GitHub Pages publicēšana

Bezmaksas hostings.

Izpildes soļi:

  1. Repo Settings → Pages → Source: gh-pages branch.
  2. Push web_build saturu uz gh-pages branch.
  3. Pēc dažām minūtēm — pieejams https://username.github.io/repo.
  4. Pievieno saiti README.
  5. Dalies ar draugiem!

Papildu izaicinājums: Steam publicēšana

Komerciālu spēli pārdod.

Izpildes soļi:

  1. Steamworks integration ar GodotSteam plugin.
  2. Achievement, leaderboards, cloud saves.
  3. Komerciāls modelis vai bezmaksas.

Biežākās kļūdas

  • Web build crashes browser: Threading issue — disable threading export setting.
  • EXE not running: Antivirus block — sign executable.
  • Large export size: Optimize textures (PNG → WebP), audio (compress).

Godot ekrānuzņēmumi

Koda piemērs (paplašināts)

#include <godot_cpp/classes/os.hpp>
#include <godot_cpp/variant/string.hpp>
#include <godot_cpp/variant/utility_functions.hpp>

using namespace godot;

struct ExportChecklist {
    bool has_export_template = false;
    bool has_cpp_library_for_target = false;
    bool has_index_html = false;
    bool tested_outside_editor = false;

    bool is_ready_to_publish() const {
        return has_export_template
            && has_cpp_library_for_target
            && has_index_html
            && tested_outside_editor;
    }
};

void ExportReporter::print_status(const ExportChecklist &checklist) {
    UtilityFunctions::print("Platform: ",
        OS::get_singleton()->get_name());
    UtilityFunctions::print("Ready to publish: ",
        checklist.is_ready_to_publish());
}
