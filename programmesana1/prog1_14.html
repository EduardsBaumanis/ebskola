1. tēma: Ievads un darba vide. 4. stunda: GitHub un drošs SSH savienojums.

Tavs šīs stundas izaicinājums: Tu iemācīsies atpazīt atšķirību starp darbu savā datorā un mākonī, izveidot šifrētu SSH drošības atslēgu un sinhronizēt savu kodu ar GitHub serveri.

SR 2.4.9. (Git attālināti), 2.3.1. (Datu drošība tīklā)

1. uzdevums (Vienkāršs): GitHub krātuves izveide

Sagatavot vietu savam projektam internetā, lai tam varētu piekļūt no jebkura datora.

Izpildes soļi:

  1. Atver pārlūkprogrammu un dodies uz github.com. Pieraksties savā kontā.
  2. Augšējā labajā stūrī spied uz + ikonas un izvēlies New repository.
  3. Laukā Repository name ieraksti: mana_spele.
  4. Izvēlies Private (tā Tavu kodu redzēsi tikai Tu).
  5. Nespied nevienu citu izvēles rūtiņu un spied pogu Create repository.
  6. Atstāj atvērto lapu — mums tūlīt vajadzēs tajā redzamo adresi.

2. uzdevums (Viduvējs): SSH drošības atslēgas iestatīšana

Izveidot unikālu digitālo atslēgu, lai GitHub serveris atpazītu Tavu datoru bez paroles ievadīšanas.

Izpildes soļi:

  1. Atver VS Code termināli un ieraksti: ssh-keygen -t ed25519 -C "tavs_epasts@piemers.lv" (izmanto savu e-pastu).
  2. Spied Enter trīs reizes (nepievienojot paroli), līdz redzi simbolu zīmējumu.
  3. Terminālī ieraksti komandu: cat ~/.ssh/id_ed25519.pub.
  4. Iezīmē un nokopē parādīto teksta rindu (tā sākas ar ssh-ed25519...).
  5. GitHub vietnē dodies uz Settings -> SSH and GPG keys -> New SSH key.
  6. Ieraksti nosaukumu "Skolas dators", ielīmē nokopēto tekstu un spied Add SSH key.

3. uzdevums (Padziļināts): Koda nosūtīšana uz mākoni (Push)

Savienot savu lokālo mapi ar GitHub un pirmo reizi nosūtīt saglabāto vēsturi uz serveri.

Izpildes soļi:

  1. GitHub lapā, kur izveidoji krātuvi, izvēlies SSH cilni un nokopē adresi (piemēram, git@github.com:vards/mana_spele.git).
  2. VS Code terminālī ieraksti: git remote add origin [šeit ielīmē nokopēto adresi].
  3. Ieraksti komandu: git branch -M main.
  4. Nosūti kodu: git push -u origin main.
  5. Ja terminālis jautā par savienojuma turpināšanu, ieraksti yes un spied Enter.
  6. Atjaunini GitHub lapu pārlūkā — tur tagad jābūt redzamiem Taviem failiem.

Jaunās vielas kopsavilkums

  • GitHub: Mākoņpakalpojums, kas glabā Tavu kodu un tā versiju vēsturi internetā.
  • Remote (Attālinātā krātuve): Projekta kopija, kas atrodas uz servera. Galveno adresi parasti sauc par origin.
  • SSH atslēgas: Drošības metode, kas izmanto divas atslēgas (privāto un publisko), lai šifrētu saziņu starp datoru un serveri.
  • Push (Grūdiens): Komanda, kas nosūta Tavas lokālās izmaiņas no datora uz GitHub.

Biežākās kļūdas (un kā tās labot):

  • Kļūda: "Permission denied (publickey)": Tu, iespējams, ielīmēji nepareizo atslēgu. Pārliecinies, ka GitHub iestatījumos pievienoji publisko atslēgu (failu ar galotni .pub), nevis privāto.
  • Kļūda: "remote origin already exists": Tu jau esi mēģinājis pievienot adresi. Lieto komandu git remote remove origin un tad mēģini 3. uzdevumu vēlreiz.
  • Kļūda: "Everything up-to-date", bet GitHub mapē nekā nav: Tas nozīmē, ka Tu esi aizmirsis git add un git commit pirms nosūtīšanas. Git nav ko sūtīt, ja nekas nav saglabāts vēsturē.
  • Kļūda: SSH adrese pret HTTPS: 3. uzdevumā pārliecinies, ka nokopēji adresi, kas sākas ar git@github.com..., nevis https://.... SSH atslēgas darbojas tikai ar pirmo variantu.
  • Terminālis "iesalst" pie jautājuma: Kad parādās jautājums par "Authenticity of host", nebaidies — dators vienkārši prasa apstiprinājumu. Ieraksti yes un spied Enter.

Atceries: SSH atslēgu iestatīšana parasti ir jādara tikai vienreiz uz katra jauna datora. Turpmākajās stundās Tu varēsi nosūtīt darbu ar vienu vienkāršu komandu: git push.
