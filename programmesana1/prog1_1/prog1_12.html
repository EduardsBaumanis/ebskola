Teorija: Koda glabāšana mākonī

Programmēšanā ir svarīgi nodrošināt, lai kods nepazustu un būtu pieejams neatkarīgi no konkrētā datora.

  • Cloud hosting (Mākoņvides uzturēšana): GitHub ir platforma, kas glabā tavu kodu drošībā internetā un ļauj pie tā strādāt komandā.
  • Repository (Krātuve): Tava projekta "mape" GitHub vidē, kas satur visus failus un to izmaiņu vēsturi.
  • Version control (Versiju kontrole): Sistēma (Git), kas izseko katrai izmaiņai kodā. GitHub Desktop ir vizuāla programma, kas palīdz lietot šo sistēmu, neizmantojot sarežģītas komandrindas komandas.

1. uzdevums: GitHub konta izveide

Reģistrējies platformā, kas kalpos kā tavs digitālais portfolio visā kursa laikā.

Izpildes soļi:

  1. Tīmekļa pārlūkprogrammā atver github.com.
  2. Nospied Sign up un ievadi prasīto informāciju (e-pastu, drošu paroli un unikālu lietotājvārdu).
  3. Atver savu e-pastu un apstiprini reģistrāciju, noklikšķinot uz atsūtītās saites vai ievadot kodu.
  4. Pieslēdzies savam jaunajam kontam un dodies uz profila iestatījumiem, lai pievienotu atpazīstamu profila attēlu.

2. uzdevums: GitHub Desktop uzstādīšana

Uzstādi datorā programmu, kas ļaus ērti sinhronizēt lokālos failus ar mākoņkrātuvi.

Izpildes soļi:

  1. Atver Microsoft Store lietotni savā datorā (vai lejupielādē no desktop.github.com).
  2. Meklētājā ieraksti GitHub Desktop un veic programmas uzstādīšanu.
  3. Atver programmu un sākuma ekrānā izvēlies Sign in to GitHub.com.
  4. Pārlūkprogrammā apstiprini autorizāciju (Authorize desktop).
  5. Programmas iestatījumos pārliecinies, ka ir norādīts tavs vārds un piesaistītais e-pasts.

3. uzdevums: Pirmās krātuves izveide mākonī

Izveido jaunu publisku Repository (Krātuvi), kas kalpos kā pamatmape visiem turpmākajiem skolas darbiem.

Izpildes soļi:

  1. GitHub tīmekļa vietnē (pārlūkprogrammā) augšējā labajā stūrī nospied + un izvēlies New repository.
  2. Laukā Repository name precīzi ieraksti: programmesana-pamatkurss.
  3. Pārliecinies, ka krātuve ir iestatīta kā Public (Publiska).
  4. Noteikti atzīmē izvēles rūtiņu Add a README file, lai krātuve nebūtu tukša, un nospied Create repository.
  5. Iesūtīt uzdevuma github saiti e-klasē.

Papildus uzdevums: Profila personalizēšana (Secret README)

Izveido īpašu krātuvi, kas darbosies kā tava GitHub profila vizītkarte (titullapa).

Izpildes soļi:

  1. GitHub platformā izveido vēl vienu jaunu Repository (Krātuvi).
  2. Kā nosaukumu ieraksti tieši tādu pašu vārdu, kāds ir tavs GitHub lietotājvārds. Parādīsies ziņojums "You found a secret!".
  3. Atzīmē Add a README file un izveido krātuvi.
  4. Atver šo jauno krātuvi, nospied uz zīmuļa ikonas (Edit) un uzraksti īsu tekstu par sevi un savu mērķi šajā programmēšanas kursā.
  5. Saglabā izmaiņas, nospiežot Commit changes.

Biežākās kļūdas (un kā tās labot):

  • Neapstiprināts e-pasts: GitHub ierobežos tavas iespējas (neļaus izveidot krātuves), ja nebūsi apstiprinājis savu e-pastu. Pārbaudi "Spam" vai "Surogātpasts" mapi, ja neredzi vēstuli.
  • Neprofesionāls lietotājvārds: Izvairies no iesaukām, kuras būtu neērti rādīt nākotnes darba devējam vai skolotājam. Izmanto sava vārda un uzvārda variācijas.
  • Autorizācijas kļūda Desktop lietotnē: Ja programmai neizdodas savienoties ar tavu kontu, dodies uz File -> Options -> Accounts, izraksties (Sign out) un mēģini autorizācijas procesu no jauna.
