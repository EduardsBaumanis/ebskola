4.5 Spēles plānošana ar shēmām

Tavs šīs stundas izaicinājums: Analizēt "Skaitļu minētāja" spēles gaitu, izstrādāt vizuālu algoritma shēmu un sasaistīt to ar konkrētām funkcijām pirms koda rakstīšanas.

Teorija: Algoritma shēmu simboli

Pirms koda rakstīšanas algoritmu vizualizē shēmā. Katrs simbols Python kodā nozīmē konkrētu konstrukciju.

Cikli shēmā tiek attēloti kā bulta atpakaļ uz iepriekšējo soli ar lēmuma rombu, kas pārtrauc vai turpina ciklu — tas atbilst Python while ciklam.

1. uzdevums: Mini-shēma vienam mēģinājumam

Uzzīmē shēmu vienam spēles mēģinājumam, izmantojot pareizos simbolus un trīs iznākumus.

Izpildes soļi:

  1. Atver bezmaksas rīku draw.io vai zīmē uz papīra.
  2. Uzzīmē sākumu (Ovāls): Sākt mēģinājumu.
  3. Pievieno Paralelogramu: Lūgt skaitli no spēlētāja.
  4. Pievieno Rombu: Vai skaitlis = patiesais? No tā izvelc trīs bultas: jā → uzvara, par lielu, par mazu.
  5. Saglabā shēmu kā shema_1.png.

2. uzdevums: Pilns spēles cikls

Paplašini shēmu, iekļaujot ciklu un mēģinājumu skaitīšanu.

Izpildes soļi:

  1. Pirms cikla pievieno taisnstūri: megi = 0; izveido patiesais skaitlis.
  2. Iekļauj 1. uzdevuma loģiku ciklā.
  3. Pēc katra mēģinājuma pievieno megi += 1.
  4. Pievieno Rombu: Vai megi < 10?. Ja jā, atgriežas pie ievades; ja nē, iet uz beigām (Game Over).
  5. Saglabā kā shema_pilna.png.

3. uzdevums: Sasaiste ar Python funkcijām

Blakus katram shēmas simbolam ieraksti, kura Python funkcija to īsteno.

Izpildes soļi:

  1. Pārkopē shema_pilna.png uz shema_kods.png.
  2. Pie sākuma taisnstūra pieraksti: izveido_skaitli().
  3. Pie ievades paralelograma pieraksti: input().
  4. Pie pārbaudes romba pieraksti: salidzina(minetais, patiesais).
  5. Pie cikla atpakaļ-bultas pieraksti: while megi < 10:.
  6. Augšupielādē abas shēmas savā GitHub krātuvē mapē plani/.

Papildus uzdevums: Mājienu sistēmas paplašinājums

Pievieno shēmai uzlabojumu, kas dod papildu mājienu pēc 5 mēģinājumiem.

Izpildes soļi:

  1. Pievieno Rombu: Vai megi == 5?.
  2. Ja jā — pievieno taisnstūri: Izvadi mājienu — pirmais cipars ir X.
  3. Ja nē — turpini ciklu kā parasti.
  4. Apraksti komentārā shēmā, kā šo loģiku īstenotu Python kodā ar if megi == 5:.

Biežākās kļūdas

  • Bezgalīgs cikls: Aizmirsta megi += 1 rinda — programma nekad neapstājas. Vienmēr pārbaudi, vai kāds mainīgais maina pārtraukšanas nosacījumu.
  • Aklie zari: Romba zars iziet "nekur" — vienmēr katra zara galā jābūt nākamai darbībai vai beigu ovālam.
  • Nedefinēts mainīgais: Programmā tiek lasīts mainīgais (piem., patiesais), kas vēl nav izveidots — definē to pirms cikla.

Koda piemērs: No shēmas uz kodu

import random

def izveido_skaitli():
    return random.randint(1, 100)

def viens_meginajums(patiesais):
    minetais = int(input("Min skaitli (1-100): "))
    if minetais == patiesais:
        return "trāpīts"
    elif minetais < patiesais:
        return "par mazu"
    return "par lielu"

patiesais = izveido_skaitli()
megi = 0
while megi < 10:
    rezultats = viens_meginajums(patiesais)
    print(rezultats)
    megi += 1
    if rezultats == "trāpīts":
        break
