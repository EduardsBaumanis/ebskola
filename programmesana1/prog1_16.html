Palīgmateriāli

1. tēma: Ievads un darba vide. Noslēguma pārbaudes darbs.

Tavs mērķis: Izveidot un publicēt sakārtotu programmēšanas projekta struktūru, kas ietver gan strādājošu kodu, gan dokumentāciju par Tavu darba vidi un ergonomiku. Vērtējums tiks piešķirts, analizējot Tavu iesniegto GitHub krātuves saiti.

Darba uzdevumi

1. solis: Projekta struktūras izveide (Terminālis)

  1. Uz datora darbvirsmas (Desktop) izveido galveno mapi PD1_Vards_Uzvards.
  2. Atver šo mapi programmā VS Code.
  3. VS Code terminālī izmanto komandu mkdir projekts, lai izveidotu apakšmapi kodam.

2. solis: Programmas un dokumentācijas izstrāde

  1. Mapē projekts izveido failu sveiciens.py un ieraksti kodu: print("Mana darba vide ir gatava!").
  2. Galvenajā mapē (PD1_...) izveido teksta failu README.txt. Tajā iekļauj:
    • Projekta nosaukums: (Tava brīva izvēle).
    • Autors: (Tavs Vārds un Uzvārds).
    • Lietošana: Instrukcija, kā terminālī palaist sveiciens.py.
    • Ergonomikas pašvērtējums: Apraksti 3 konkrētus uzstādījumus vai paradumus (piemēram: ekrāna attālums, muguras poza, apgaismojums), ko Tu šodien ievēro, lai saudzētu veselību.

3. solis: Versiju pārvaldība un publicēšana

  1. Inicializē Git krātuvi (git init).
  2. Pievieno visus failus un veic pirmo commit ar ziņojumu: "Projekta struktura un dokumentacija".
  3. Veic nelielu labojumu README.txt failā (piem., pievieno šodienas datumu) un veic otro commit ar ziņojumu: "Papildinata dokumentacija".
  4. Izveido Private krātuvi GitHub vietnē un nosūti datus (git push), izmantojot savu SSH atslēgu.

Snieguma līmeņa apraksts (Vērtēšanas tabula)

Vērtējums tiek noteikts, pārbaudot iesniegto GitHub saiti.

Kritēriji Sācis apgūt (1-3) Turpina apgūt (4-6) Apguvis (7-8) Apguvis padziļināti (9-10)
Struktūra un resursi Faili nav sakārtoti mapēs vai nosaukumi ir kļūdaini. Struktūra ir izveidota, bet faili atrodas neprecīzās vietās. Izveidota precīza mapju struktūra atbilstoši uzdevumam. Nevainojama resursu pārvaldība; izmantoti korekti paplašinājumi (.py, .txt).
Git vēsture Krātuve nav izveidota vai tajā nav saglabātas versijas. Veikts viens saglabāšanas punkts ar nepilnīgu ziņojumu. Vēsturē redzami vismaz divi punkti ar skaidriem aprakstiem. Vēsture precīzi dokumentē darba progresu un loģisku izmaiņu gaitu.
Ergonomikas integrācija README failā nav minēta ergonomika vai veselība. Minēts tikai viens punkts vai apraksts ir pārāk vispārīgs. Dokumentācijā skaidri aprakstīti 3 ergonomikas punkti. Ergonomikas punkti demonstrē dziļu izpratni par veselīgu darba vidi.
GitHub un drošība Projekts nav publicēts vai saite nedarbojas. Projekts ir publisks vai izmantots nedrošs savienojums (HTTPS). Projekts droši publicēts Private krātuvē, izmantojot SSH. Nevainojama sinhronizācija; krātuve ir privāta un pieejama skolotājam.

Pārbaudi sevi pirms iesniegšanas:

  • Vai krātuve ir Private? Pārliecinies par to GitHub iestatījumos.
  • Vai README satur Tavu vārdu? Bez autorības darbu nevarēs ieskaitīt.
  • Vai SSH adrese ir pareiza? Izmanto adresi, kas sākas ar git@github.com....
  • Vai faili ir saglabāti? Neaizmirsti Ctrl+S pirms git add komandas.
