9.2 Atvērtā koda licences un dokumentācija

Tavs šīs stundas izaicinājums: Iepazīt atvērtā koda licences (MIT, GPL, Apache) un iemācīties lasīt bibliotēku dokumentāciju.

SR 3.1.3. Skaidro intelektuālā īpašuma tiesības un licences SR 2.4.11. Standartizētas bibliotēkas

70 min darba sadalījums: 1. uzdevums (~20 min) — atjauno teorijas piemēru vai minimālo prototipu; 2. uzdevums (~25 min) — pielieto to galvenajā uzdevumā; 3. uzdevums (~25 min) — pārbaudi rezultātu, izlabo kļūdas un pieraksti, kas strādā. Papildus/4. līmeņa uzdevums ir paredzēts tikai tad, ja pamatdarbs ir pabeigts.

Priekšzināšanu robeža: uzdevumos izmanto tikai iepriekš apgūto un šīs lapas teorijas/koda piemēros parādīto. Ja parādās jauna komanda vai rīks, vispirms atrod tās paraugu teorijas blokā.

Teorija: Atvērtā koda licences un dokumentācijas lasīšana

Atvērtā koda licence nosaka, kā Tu drīksti izmantot, modificēt un izplatīt cita autora kodu.

LicenceAtļaujPienākumi
MITVisu (komercdarbība, modifikācijas)Saglabāt licences tekstu
Apache 2.0VisuLicences teksts + izmaiņu paziņojums
GPL v3VisuTava versija arī jāpublicē kā GPL ("copyleft")
BSDVisuSaglabāt licences tekstu
Bez licencesNEKO — pēc noklusējuma viss aizliegts—

Kā lasīt dokumentāciju: 1) sākums (Quickstart) → 2) primers ar koda piemēru → 3) API atsauce (visas funkcijas) → 4) FAQ → 5) Issues GitHub.

1. uzdevums: Licences saviem projektiem

Pievieno LICENSE failu visiem saviem projektiem.

Izpildes soļi:

  1. Atver choosealicense.com.
  2. Aizpildi anketu — saproti atšķirību starp MIT un GPL.
  3. Iepriekšējiem projektiem (vismaz 3) pievieno LICENSE failu (MIT vai Apache).
  4. Atjaunini README.md ar licences sekciju.
  5. Push uz GitHub.

2. uzdevums: Bibliotēku licences pārbaude

Pirms lieto cita autora kodu, pārbaudi licenci.

Izpildes soļi:

  1. Atver 5 dažādas Python bibliotēkas PyPI: requests, flask, numpy, pillow, django.
  2. Atrod katras licenci PyPI lapā vai GitHub repo.
  3. Pieraksti tabulā — kura GPL? kura MIT? kura komerciāla?
  4. Padomā: ja Tava lietotne lieto GPL bibliotēku, vai vari to pārdod kā slēgtu kodu?

3. uzdevums: Dokumentācijas analīze

Iemācies ātri orientēties bibliotēkas dokumentācijā.

Izpildes soļi:

  1. Izvēlies bibliotēku, ko vēl nelieto (piem., rich, tabulate, pillow).
  2. Atrod oficiālo dokumentāciju.
  3. Atrod "Quickstart" — kopē pirmo piemēru, palaiž.
  4. Atrod 3 funkcijas, kas šķiet noderīgas — eksperimentē ar tām.
  5. Saglabā piemērus failā biblioteka_demo.py.

Papildus uzdevums: Open Source Contribution

Iemēģini ieguldīt atvērtā kodā.

Izpildes soļi:

  1. Atver kāda projekta GitHub Issues sadaļu (filtrē "good first issue").
  2. Izlasi vienu — vai saproti, kas jādara?
  3. Padomā, kuru rīku tu lietoji, kuru gribētu uzlabot.

Biežākās kļūdas

  • Bez licences = pilnībā aizliegts: Pēc noklusējuma copyright pieder autoram.
  • MIT izmantošana ≠ komerciāla pārdošana: MIT atļauj komercdarbību, GPL prasa atvērtu kodu.
  • Lasi tikai sākumu: Reālā problēma bieži ir API atsauces sadaļā.
