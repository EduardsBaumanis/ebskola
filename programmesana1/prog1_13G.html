1. tēma: Ievads un darba vide. 2. stunda: Darba saglabāšana ar Git.

Tavs šīs stundas izaicinājums: Tu iemācīsies atpazīt versiju pārvaldības sistēmas (Git) lomu, patstāvīgi iestatīt savu digitālo parakstu un saglabāt koda progresu drošos "laika punktos", lai nekad nezaudētu paveikto.

SR 2.4.9. (Versiju pārvaldība), 2.3.11. (Darba efektivitāte)

1. uzdevums (Vienkāršs): Savas identitātes un krātuves izveide

Sagatavot Git sistēmu darbam un izveidot projekta "seifu".

  1. Atver VS Code un dodies uz mapi mana_spele.
  2. Loga apakšā atver Terminal.
  3. Ieraksti komandu: git config --global user.name "Tavs Vārds" (nospied Enter).
  4. Ieraksti komandu: git config --global user.email "tavs@epasts.lv" (nospied Enter).
  5. Ieraksti galveno komandu: git init (tā pasaka datoram, ka šī mape tiks uzraudzīta).

2. uzdevums (Viduvējs): Pirmais saglabāšanas punkts (Commit)

Iemācīties "nofotografēt" savu pašreizējo darbu, lai tam varētu atgriezties vēlāk.

  1. Terminālī ieraksti git status (tu redzēsi sarkanus failus – Git tos vēl nav saglabājis).
  2. Ieraksti git add . (punkts beigās nozīmē "pievienot visu, kas atrodas mapē").
  3. Vēlreiz ieraksti git status (failiem jākļūst zaļiem).
  4. Ieraksti: git commit -m "Mans pirmais saglabāšanas punkts" (teksts pēdiņās ir tavs komentārs par paveikto).

3. uzdevums (Padziļināts): Izmaiņu reģistrēšana un vēsture

Saprast, kā Git pamana izmaiņas un kā tās pārskatīt.

  1. Atver failu apraksts.txt, pieraksti klāt teikumu par savas spēles noteikumiem un saglabā failu (Ctrl+S).
  2. Terminālī ieraksti git status (Git pamanīs, ka fails ir mainīts).
  3. Atkārto saglabāšanas soļus: git add . un tad git commit -m "Papildināti spēles noteikumi".
  4. Ieraksti komandu git log, lai redzētu visus savus saglabātos punktus.
  5. Lai izietu no vēstures saraksta, nospied burtu Q uz tastatūras.

Jaunās vielas kopsavilkums (Kāpēc Git?)

  • Kas ir Git? Tā ir "laika mašīna" tavam kodam, kas atceras katru izmaiņu.
  • Krātuve (Repository): Īpaša, slēpta mape tavā projektā, kurā tiek glabāta visa vēsture.
  • Commit (Saglabāšanas punkts): Tas nav vienkārši "Save". Tā ir koda "fotogrāfija" konkrētā brīdī ar tavu aprakstu.
  • Vairs nekādu kopiju: Tev nav jāveido faili spele_v1, spele_final. Git glabā visas versijas vienā failā.
  • Identitāte: Vārds un e-pasts ir nepieciešami, lai lielos projektos zinātu, kurš un kad veicis izmaiņas.

Analoģija: Git ir kā "Save Point" video spēlē. Ja tu pieļausi kļūdu, kuras dēļ programma vairs nestrādā, tu varēsi viegli atgriezties iepriekšējā punktā.

git init        # Izveido krātuvi

git commit -m   # Saglabā vēsturē ar aprakstu
git log         # Apskata vēsturi
