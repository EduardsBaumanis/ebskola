SR 2.4.8. (Koda komentēšana un dokumentēšana)

1. uzdevums (Vienkāršs): Duelīša "higiēna"

Iemācīties pareizi dokumentēt programmas sākumu un paskaidrot koda soļus.

Izpildes soļi:

  1. VS Code izveido jaunu failu dokumentets_duelis.py.
  2. Faila sākumā pievieno vairāku rindu komentāru ("""), kurā ieraksti spēles nosaukumu, savu vārdu un datumu.
  3. Uzraksti kodu, kas prasa ievadīt divu spēlētāju vārdus.
  4. Virs katras input() rindas pievieno vienas rindas komentāru (#), paskaidrojot, ka šeit tiek reģistrēti cīņas dalībnieki.

2. uzdevums (Viduvējs): Mainīgo "pārvērtības"

Labot sliktu koda stilu un ieviest profesionālu mainīgo nosaukumu došanas kārtību.

Izpildes soļi:

  1. Izveido failu tirs_kods.py.
  2. Iekopē šo "slikto" kodu: 
    
X = int(input("Speks: "))
y_BONUSS = 10
REzultAtS = X + y_BONUSS
print(REzultAtS)
  3. Pārveido visus mainīgo nosaukumus atbilstoši Snake Case (mazie burti, vārdi atdalīti ar _).
  4. Sakārto atstarpes ap operatoriem (=, +) atbilstoši PEP 8 standartam.

3. uzdevums (Padziļināts): Duelīša matemātiskais modelis

Apvienot aprēķinus ar augstiem lasāmības standartiem un paskaidrot "kāpēc" loģiku.

Izpildes soļi:

  1. Izveido failu duelisa_logika.py.
  2. Uzraksti programmu, kas aprēķina reitingu pēc formulas: $Reitings = (Uzvaras \cdot 100) / (Zaudējumi + 1)$.
  3. Izmanto tikai jēgpilnus Snake Case mainīgos (piem., uzvaru_skaits).
  4. Pievieno komentāru, kas paskaidro: "Pieskaitām 1, lai izvairītos no dalīšanas ar nulli."
  5. Izvadi rezultātu, izmantojot f-string.

Lasāmības standarti: Snake Case un PEP 8

  • Komentāri (#): Dators tos ignorē. Tie domāti programmētājam. Raksti par to, kāpēc kods kaut ko dara, nevis acīmredzamo.
  • Snake Case: Python standarts mainīgo nosaukumiem. Visu rakstām ar mazajiem burtiem, vārdus atdalot ar apakšsvītru.
  • PEP 8: Python stila rokasgrāmata. Galvenais noteikums: lieto atstarpes ap vienādības zīmi un matemātiskajiem simboliem (a = 5 + 2).
