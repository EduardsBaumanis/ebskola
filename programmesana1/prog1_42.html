Tēmas Sasniedzamie rezultāti Palīgmateriāli

4. tēma: Funkcijas un Modularitāte. 2. stunda: Scope un return.

Tavs šīs stundas izaicinājums: Izprast mainīgo redzamības (scope) principus, izmantot return komandu datu atgriešanai un izveidot funkcijas, kas spēj apmainīties ar informāciju ar programmas galveno daļu.

SR 2.4.15. (Atgriežamās vērtības un lokālie mainīgie)

Teorija: Svarīgākie noteikumi

  • Return pret Print: print() tikai parāda tekstu ekrānā. return "izmet" vērtību no funkcijas ārā, lai to varētu saglabāt mainīgajā un lietot tālāk.
  • Lokālie mainīgie (Scope): Mainīgie, kas definēti funkcijas iekšpusē, ir "dzīvi" tikai tajā. Ārpusē tie neeksistē.
  • Funkcijas beigas: Tiklīdz dators sasniedz komandu return, funkcijas darbs tiek tūlītēji pārtraukts. Jebkurš kods zem tās netiks izpildīts.

1. uzdevums: Zāļu eliksīrs (return)

Vienkārša uzdevuma ietvaros izveidosi matemātisku aprēķinu funkciju, kas atgriež rezultātu tālākai izmantošanai spēlē.

Izpildes soļi:

  1. VS Code izveido failu eliksirs.py.
  2. Definē funkciju def izrekinat_veselibu(pasreizeja, eliksira_speks):.
  3. Funkcijas iekšienē saskaiti abus skaitļus, piemēram, summa = pasreizeja + eliksira_speks.
  4. Tā vietā, lai izmantotu print(), ieraksti: return summa.
  5. Ārpus funkcijas izsauc to un saglabā rezultātu mainīgajā: jauna_veseliba = izrekinat_veselibu(50, 20).
  6. Izvadi galarezultātu lietotājam: print(f"Pēc eliksīra lietošanas Tev ir {jauna_veseliba} HP!").

2. uzdevums: Lokālais "slazds" (Scope)

Vidējas grūtības uzdevumā analizēsi kļūdas paziņojumu, ko izraisa lokālo mainīgo nepieejamība ārpus funkcijas, un pielietosi return tā labošanai.

Izpildes soļi:

  1. Izveido failu scope_tests.py.
  2. Definē funkciju def pievienot_zeltu():.
  3. Funkcijas iekšienē izveido mainīgo zelts = 100.
  4. Izsauc funkciju un tūlīt pēc tam mēģini izpildīt print(zelts) ārpus funkcijas.
  5. Palaid programmu un novēro NameError kļūdu terminālī.
  6. Labojums: Pārveido funkciju, pievienojot return zelts, un piešķir to mainīgajam ārpusē pirms izvadīšanas.

3. uzdevums: CYOA Izvēles dzinējs

Padziļinātā uzdevumā izstrādāsi interaktīvu izvēļņu sistēmu, kas pieņem lietotāja ievadi un atgriež to spēles loģikas vadīšanai.

Izpildes soļi:

  1. Izveido failu izveles_dzinejs.py.
  2. Definē funkciju def veikt_izveli(jautajums, opcija_a, opcija_b):.
  3. Funkcijas iekšienē izvadi jautājumu, abas opcijas un palūdz lietotājam ievadi ("A" vai "B").
  4. Izmanto return, lai "atdotu" ievadīto burtu spēles galvenajai daļai.
  5. Izsauc funkciju, saglabājot to mainīgajā, piemēram, cels = veikt_izveli("Kurp dosies?", "A - Mežs", "B - Pils").
  6. Ar if pārbaudi: ja atgrieztais rezultāts ir "A", izvadi "Tu dodies uz mežu", ja "B" — "Tu dodies uz pili".

Biežākās kļūdas (un kā tās labot):

  • "Pazudusī" vērtība: Tu izmanto return, bet ārpusē nepiešķir rezultātu mainīgajam. Tad vērtība vienkārši izgaist. Risinājums: Vienmēr piesaisti atgriezto vērtību mainīgajam, piemēram, rezultats = mana_funkcija().
  • Scope kļūda: Mēģinājums piekļūt mainīgajam, kas "pieder" funkcijai, no programmas ārpuses. Risinājums: Atceries, ka funkcijas iekšējie mainīgie ir lokāli; lai tos iegūtu, tie ir jāatgriež ar return.
  • None tips: Ja funkcijā trūkst return, bet Tu mēģini rezultātu saglabāt mainīgajā, tas saņems tukšu vērtību None. Risinājums: Pārbaudi, vai funkcijas loģika beidzas ar atbilstošu atgriežamo datu rindu.

Koda piemērs

def dabu_punktus():
    bonuss = 50 # Lokālais mainīgais
    return bonuss

# Ārpusē 'bonuss' neeksistē, mums ir jāizmanto funkcijas rezultāts
rezultats = dabu_punktus()
print(f"Tu ieguvi {rezultats} punktus!")

# Kods pēc return nekad netiek izpildīts:
def beigas():
    return "Pirmais ziņojums"
    print("Šo rindu dators ignorēs!")
Tu ieguvi 50 punktus!
