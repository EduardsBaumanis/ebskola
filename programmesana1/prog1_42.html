Tēmas Sasniedzamie rezultāti Palīgmateriāli

4. tēma: Funkcijas un Modularitāte. 2. stunda: Scope un return.

Tavs šīs stundas izaicinājums: Tu iemācīsi funkcijām "sarunāties" ar pārējo programmu. Tu apgūsi, kā izmantot return, lai saņemtu aprēķinu rezultātus, un uzzināsi, kāpēc daži mainīgie ir "slepeni" un pieejami tikai funkcijas iekšienē.

SR 2.4.15. (Atgriežamās vērtības un lokālie mainīgie)

1. uzdevums (Vienkāršs): Zāļu eliksīrs (return)

Iemācīties atgriezt vērtību no funkcijas, lai to varētu saglabāt mainīgajā un izmantot tālākos aprēķinos.

Izpildes soļi:

  1. VS Code izveido failu eliksirs.py.
  2. Definē funkciju izrekinat_veselibu(pasreizeja, eliksira_speks):.
  3. Funkcijas iekšienē saskaiti abus skaitļus. Tā vietā, lai izmantotu print(), ieraksti: return summa.
  4. Ārpus funkcijas izsauc to un saglabā rezultātu mainīgajā: jauna_veseliba = izrekinat_veselibu(50, 20).
  5. Izvadi galarezultātu lietotājam: "Pēc eliksīra lietošanas Tev ir 70 HP!".

2. uzdevums (Viduvējs): Lokālais "slazds" (Scope)

Praktiski pārliecināties par mainīgo redzamības zonu (scope) un saprast, kāpēc funkcijas iekšējie mainīgie nav pieejami ārpusē.

Izpildes soļi:

  1. Izveido failu scope_tests.py.
  2. Definē funkciju pievienot_zeltu():.
  3. Funkcijas iekšienē izveido mainīgo zelts = 100 un izvadi to.
  4. Izsauc šo funkciju un pēc tam tūlīt mēģini izpildīt print(zelts) ārpus funkcijas.
  5. Palaid programmu un novēro NameError kļūdu.
  6. Labojums: Pārveido funkciju tā, lai tā izmantotu return zelts, un ārpusē piešķir šo vērtību jaunam mainīgajam.

3. uzdevums (Padziļināts): CYOA Izvēles dzinējs

Izveidot universālu, atkārtoti lietojamu funkciju spēles dialogiem, kas atgriež spēlētāja pieņemto lēmumu.

Izpildes soļi:

  1. Izveido failu izveles_dzinejs.py.
  2. Definē funkciju veikt_izveli(jautajums, opcija_a, opcija_b):.
  3. Funkcijas iekšienē izvadi jautājumu un abas opcijas, tad palūdz lietotājam ievadi ("A" vai "B").
  4. Izmanto return, lai "atdotu" spēlētāja ievadi spēles galvenajai daļai.
  5. Izsauc funkciju mainīgajā celš = veikt_izveli(...) un pēc tam ar if pārbaudi: ja rezultāts ir "A", izvadi "Tu dodies uz mežu", ja "B" — "Tu dodies uz pili".

Jaunā viela: Svarīgākie noteikumi

  • Return pret Print: print() tikai parāda tekstu cilvēkam. return "pabīda" vērtību atpakaļ programmētājam, lai to varētu ielikt mainīgajā.
  • Lokālie mainīgie: Mainīgie, kas definēti funkcijas iekšpusē, ir "dzīvi" tikai tik ilgi, kamēr funkcija strādā. Ārpusē tie neeksistē.
  • Funkcijas beigas: Tiklīdz dators sasniedz komandu return, funkcijas darbs tiek pārtraukts. Jebkurš kods zem return (tajā pašā funkcijā) netiks izpildīts.

Biežākās kļūdas:

1. "Pazudusī" vērtība: Tu izmanto return, bet ārpusē nepiešķir rezultātu mainīgajam (ierakstīts tikai mana_funkcija()).
2. Scope kļūda: Mēģinājums izsaukt funkcijas iekšējo mainīgo ārpus tās (NameError).
3. None tips: Ja funkcija beidzas bez return, bet Tu mēģini to piešķirt mainīgajam, mainīgais saņems vērtību None. 

def dabu_punktus():
    bonuss = 50 # Lokālais mainīgais
    return bonuss

# Šeit bonuss nav redzams!
rezultats = dabu_punktus()
print(f"Tu ieguvi {rezultats} punktus!")

# Kods pēc return nekad nestrādā:
def tests():
    return "Gatavs"
    print("Šo neviens neredzēs")
Tu ieguvi 50 punktus!
