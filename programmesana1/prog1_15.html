1. tēma: Ievads un darba vide. 5. stunda: Kodu koplietošana un dokumentēšana.

Tavs šīs stundas izaicinājums: Tu iemācīsies atpazīt dokumentācijas nozīmi programmēšanā, patstāvīgi izveidot skaidru projekta aprakstu parastā teksta formātā un ievērot sadarbības etiķeti, publicējot darbu GitHub vietnē.

SR 2.4.9. (Kodu koplietošana), 3.1.5. (Intelektuālais īpašums un autorība)

1. uzdevums (Vienkāršs): Projekta vizītkartes izveide

Izveidot galveno informācijas failu, lai ikviens, kurš atver Tavu projektu, uzreiz saprastu, kas tā ir par spēli.

Izpildes soļi:

  1. Atver VS Code un savu projekta mapi mana_spele.
  2. Izveido jaunu failu ar nosaukumu README.txt.
  3. Faila pirmajā rindā ar lielajiem burtiem ieraksti savas spēles nosaukumu.
  4. Zem nosaukuma pieraksti vienu teikumu: "Šī ir mana pirmā izvēles spēle, kas tapusi programmēšanas stundās."
  5. Saglabā failu (Ctrl+S).

2. uzdevums (Viduvējs): Instrukciju un autorības pievienošana

Sagatavot projekta aprakstu tā, lai citi lietotāji zinātu, kā spēli palaist un kurš ir tās autors.

Izpildes soļi:

  1. Atver esošo README.txt failu.
  2. Pievieno sadaļu KĀ SPĒLĒT: un uzskaiti soļus:
    • Uzstādiet Python 3.12 no Microsoft Store.
    • Lejupielādējiet šo mapi savā datorā.
    • Atveriet termināli un ierakstiet: python tests.py
  3. Faila beigās pievieno rindu AUTORS: un norādi savu vārdu un uzvārdu.
  4. Saglabā izmaiņas.

3. uzdevums (Padziļināts): Dokumentācijas publicēšana un pārbaude

Nostiprināt profesionālu darba ciklu, nosūtot dokumentāciju uz serveri un pārliecinoties par tās pieejamību.

Izpildes soļi:

  1. Atver VS Code termināli.
  2. Pārbaudi, vai Git redz jauno failu, ierakstot git status.
  3. Sagatavo failu saglabāšanai: git add README.txt.
  4. Saglabā vēsturē ar skaidru ziņojumu: git commit -m "Pievienota projekta dokumentācija un instrukcija".
  5. Nosūti datus uz GitHub: git push.
  6. Atver savu krātuvi pārlūkā (github.com) un pārliecinies, ka README.txt fails ir redzams failu sarakstā.

Jaunās vielas kopsavilkums

  • README faila loma: Tā ir projekta "lietošanas pamācība". Bez šī faila citiem programmētājiem Tavs kods var būt nesaprotams.
  • Sadarbības etiķete: Programmēšanā ir pieņemts norādīt autoru un sniegt skaidras norādes, kā kodu izmantot.
  • Intelektuālais īpašums: Dokumentācijā norādīts vārds palīdz fiksēt Tavu autorību uz radīto darbu.
  • Skaidrība: Labā praksē commit ziņojumiem jābūt īsiem, bet aprakstošiem (ko tieši Tu izdarīji).

Biežākās kļūdas (un kā tās labot):

  • Fails nav pievienots Git: Ja pēc git push fails neparādās GitHub, iespējams, Tu aizmirsi komandu git add.
  • Neskaitāmi "Update" ziņojumi: Izvairies no commit ziņojumiem kā "Update", "Fix" vai "123". Raksti precīzi: "Pievienots autors".
  • Fails nav saglabāts: Git nevar saglabāt to, kas nav saglabāts Windows failu sistēmā. Pirms git add vienmēr nospied Ctrl+S.
  • Nepareiza atrašanās vieta: Ja terminālis rāda kļūdu, pārliecinies ar komandu dir, ka tiešām atrodies savā projekta mapē.

Zini, kāpēc tas ir svarīgi? Pat ja Tu strādā viens, pēc mēneša Tu pats varētu aizmirst, kāda bija faila tests.py nozīme. README.txt palīdz Tev un Taviem biedriem saprast projekta mērķi.
