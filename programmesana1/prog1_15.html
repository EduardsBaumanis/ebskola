1. tēma: Mini-projekts — darba vide, projekta struktūra, Git un dokumentācija

Stundas sasniedzamais rezultāts

Stundas beigās skolēns izstrādā vienkāršu CLI programmu, izveido sakārtotu projekta mapju struktūru, izmanto Git ar vismaz 3 commit, noformē README ar instrukcijām un piemēro drošības principus (nepublicē sensitīvu informāciju). (SR: 2.4.12; 2.3.9; 2.4.9; 3.1.5)

1. uzdevums — Projekta ideja un prasības

Avots: Wikimedia Commons — Software development lifecycle diagram.

Mērķis: skaidri definēt, ko tu veidosi, lai darbs nav “nejaušs kods”.

A. Izvēlies vienu mini-projekta ideju (CLI)
Izvēlies vienu no variantiem vai izdomā savu (saskaņo ar skolotāju):
• Vienību pārveidotājs (cm↔m, °C↔°F u.c.)
• Budžeta kalkulators (ievade → summa → atskaite)
• Vienkārša viktorīna (3–5 jautājumi, punktu skaits)
• Reakcijas laika spēle (teksts/taimeris, bez GUI)

B. Uzraksti prasības README failā
README sadaļā “Prasības” ieraksti:
1) Programmas mērķi (1 teikums)
2) Ievadi (ko lietotājs ievada?)
3) Izvadi (ko programma izvada?)
4) Vismaz 2 validācijas noteikumus (piem., “nepieņem tukšu ievadi”, “pieņem tikai skaitli”)

Paškontrole: prasībām jābūt tik skaidrām, lai cits skolēns varētu uzrakstīt programmu pēc tava apraksta.

2. uzdevums — Projekta struktūra un Git darba plūsma (3+ commit)

Avots: Wikimedia Commons — Git branch and merge diagram.

Mērķis: veidot projektu pa soļiem ar saprotamu vēsturi.

A. Izveido jaunu projekta mapi
1) Izveido mapi ar nosaukumu: mini_projekts
2) Izveido struktūru (minimums):
• main.py
• README.md
• .gitignore

B. Inicializē Git un izveido 1. commit
Terminālī projekta mapē izpildi: 


git init
git add .
git commit -m "Initial project structure"

C. Izstrādā programmas “skeletu” un izveido 2. commit
1) Failā main.py uzraksti programmas pamata plūsmu (bez pilnas loģikas):
• ievade
• apstrāde
• izvade
2) Pievieno īsu instrukciju README “Kā palaist”.
3) Noformē commit: 


git add main.py README.md
git commit -m "Add program skeleton"

D. Pievieno validāciju un izveido 3. commit
1) Pievieno vismaz 2 ievades pārbaudes (piem., skaitlis, robežas, izvēlne).
2) Noformē commit ar skaidru ziņojumu: 


git add main.py
git commit -m "Add input validation"

Paškontrole: git log --oneline jāparāda vismaz 3 commit ar saprotamiem nosaukumiem.

3. uzdevums — Drošība, licence un README noformēšana

Avots: Wikimedia Commons — Public key encryption (ilustratīvs drošības princips).

Mērķis: nodrošināt, ka projektu var saprast, palaist un tas ir korekts koplietošanai.

A. README minimums
README failā jābūt šādām sadaļām (kopē un aizpildi): 


# Mini-projekts
Autors: [Tavs vārds Uzvārds]

## Apraksts
[1–2 teikumi par programmu]

## Prasības
- Mērķis: ...
- Ievade: ...
- Izvade: ...
- Validācija: ...

## Kā palaist?
python main.py
(Windows: py main.py)

## Licence
MIT

## Drošība
- Ko es nekad nelikšu publiskā repozitorijā: [piemērs]
- Kā es aizsargāju kontu: (spēcīga parole / 2FA)

B. Licence (MIT)
1) Ja skolotājs prasa LICENSE failu: izveido LICENSE un ieliec MIT licences tekstu.
2) Ja pietiek ar norādi README: sadaļā “Licence” ieraksti MIT.

C. Drošības pārbaude
Pārskati savu projektu un pārliecinies, ka tajā nav:
• paroļu, tokenu, e-pastu, personu datu
• failu, kurus nevajag koplietot (piem., __pycache__, .vscode)

Pēc tam izveido vēl vienu commit (ja kaut ko laboji): 


git add .
git commit -m "Finalize README and safety notes"

Paškontrole: README jābūt saprotamam arī cilvēkam, kurš projektu redz pirmo reizi.

Papildus informācija

Mini-projekta mērķis ir parādīt “pilnu darba ciklu”: prasības → struktūra → kods → validācija → dokumentācija → Git vēsture.

Veido commit bieži un ar skaidriem ziņojumiem — tas palīdz gan tev pašam, gan skolotājam saprast izstrādes gaitu.

Drošība sākas ar ieradumiem: nekad neglabā sensitīvu informāciju kodā, un izmanto 2FA tur, kur tas ir iespējams.
