SR 2.4.8. (Koda noformēšanas kultūra)

Teorija: PEP 8 Galvenie Standarti

  • PEP 8 nozīme: Pasaulē atzīts Python "etiķetes noteikumu" standarts. Labi noformēts kods ietaupa laiku gan Tev, gan Taviem komandas biedriem, jo kodu daudz biežāk lasa nekā raksta.
  • Nosaukumi (snake_case): Mainīgos un funkcijas raksta tikai ar mazajiem burtiem, vārdus atdalot ar pasvītrojumu (piemēram, mana_spele, nevis manaSpele).
  • Konstantes: Vērtības, kas programmas laikā nemainās, raksta ar lielajiem burtiem (piemēram, MAKS_HP = 100).
  • Atstarpes: Ap operatoriem (=, +, -, u.c.) vienmēr jāliek atstarpes (piemēram, x = 5 + 10, nevis x=5+10).
  • Struktūra: Starp funkciju definīcijām kodā obligāti jāatstāj tieši divas tukšas rindas.

1. uzdevums: "Briesmīgā koda" glābšana

Manuāli pārrakstīsi un formatēsi slikti strukturētu kodu atbilstoši PEP 8 standartiem.

Izpildes soļi:

  1. VS Code izveido failu haoss.py.
  2. Iekopē vai uzraksti tajā kodu, kurā nav atstarpju ap operatoriem un ir slikti mainīgo nosaukumi (piemēram, x1=int(input()) un def f(a,b):return a+b).
  3. Pārveido mainīgo un funkciju nosaukumus uz snake_case formatējumu.
  4. Saliec atstarpes ap visiem operatoriem.
  5. Nodrošini, ka starp funkcijām ir tieši divas tukšas rindas.
  6. Palaid programmu, lai pārliecinātos, ka koda loģika joprojām strādā nevainojami.

2. uzdevums: Automātiskais stilists

Integrēsi un izmantosi koda formatēšanas paplašinājumu VS Code vidē, lai automatizētu stila kļūdu labošanu.

Izpildes soļi:

  1. Atver VS Code paplašinājumu sadaļu (Extensions) un instalē paplašinājumu Black Formatter.
  2. Izveido jaunu failu auto_stils.py un saraksti tīši nepareizi noformētu kodu vienā garā rindā bez atstarpēm.
  3. Izmanto Windows taustiņu kombināciju Shift + Alt + F (Format Document).
  4. Ja VS Code prasa izvēlēties noklusējuma formatētāju, izvēlies Black Formatter.
  5. Novēro, kā VS Code pats automātiski sakārto atstarpes un atkāpes atbilstoši PEP 8 standartam.

3. uzdevums: CYOA koda revīzija

Veiksi sava spēles projekta koda auditu, pievienojot dokumentāciju un pārbaudot failu struktūru.

Izpildes soļi:

  1. Atver savas topošās spēles mapes visus esošos .py failus.
  2. Pārbaudi failu sākumu: pārliecinies, ka visas import komandas atrodas pašā faila augšā.
  3. Pārskati funkciju un mainīgo nosaukumus – vai tie paskaidro savu mērķi un atbilst snake_case?
  4. Katras funkcijas definīcijas sākumā pievieno dokumentāciju jeb docstring (aprakstu trīskāršās pēdiņās """), paskaidrojot, ko funkcija dara un kādus datus atgriež.
  5. Izmanto Shift + Alt + F katrā failā, lai pabeigtu koda sakārtošanu.

Biežākās kļūdas (un kā tās labot):

  • Tab vs Spaces: Kodā tiek jauktas Tab un atstarpju (Spaces) atkāpes, kas izraisa IndentationError. Risinājums: Vienmēr izmanto 4 atstarpes atkāpei. VS Code parasti automātiski pārvērš Tab par 4 atstarpēm.
  • Lieki komentāri: Tiek rakstīti komentāri, kas paskaidro pašsaprotamas lietas (piemēram, x = 5 # piešķir 5). Risinājums: Raksti komentārus tikai tam, lai paskaidrotu, kāpēc kods dara to, ko tas dara, nevis ko tas dara.
  • Pārāk garas rindas: Koda rinda pārsniedz ekrāna platumu, padarot to grūti lasāmu. Risinājums: Centies neiziet ārpus 79–88 rakstzīmēm vienā rindā. Izmanto Black Formatter, kas garas rindas automātiski sadalīs vairākās.
