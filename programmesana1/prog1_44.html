SR 2.4.8. (Koda noformēšanas kultūra)

Teorija: PEP 8 Galvenie Standarti

PEP 8 (Python Enhancement Proposal 8) ir dokuments, kas apraksta, kā jāizskatās Python kodam. Tas ir kā pareizrakstības vārdnīca programmētājiem.

Elements Standarts Labi Slikti
Nosaukumi snake_case mana_spele manaSpele
Konstantes LIELIE BURTI MAKS_HP = 100 hp_limit = 100
Atstarpes Ap operatoriem x = 5 + 10 x=5+10
Struktūra Starp funkcijām 2 tukšas rindas Salipušas kopā

Kāpēc tas ir svarīgi? Kodu daudz biežāk lasa, nekā raksta. Labi noformēts kods ietaupa laiku gan Tev, kad to labosi pēc mēneša, gan Taviem komandas biedriem.

Praktiskie uzdevumi

1. uzdevums: "Briesmīgā koda" glābšana

  1. VS Code izveido failu haoss.py.
  2. Iekopē tajā kodu, kurā nav atstarpju un ir slikti mainīgo nosaukumi (piem., x1 = input()).
  3. Pārveido nosaukumus uz snake_case (mazie burti ar pasvītrojumu).
  4. Saliec atstarpes ap operatoriem un nodrošini divas tukšas rindas starp funkcijām.
  5. Palaid programmu, lai pārliecinātos, ka tā joprojām strādā nevainojami.

2. uzdevums: Automātiskais stilists

  1. VS Code paplašinājumu veikalā (Extensions) instalē "Black Formatter".
  2. Saraksti tīši nepareizi noformētu kodu vienā garā rindā.
  3. Izmanto taustiņu kombināciju Shift + Alt + F (Format Document).
  4. Novēro, kā VS Code pats sakārto atstarpes un atkāpes atbilstoši standartam.

3. uzdevums: CYOA koda revīzija

  1. Atver savas topošās spēles mapes visus failus.
  2. Pārbaudi: vai visi import ir faila augšā? Vai funkciju nosaukumi ir skaidri?
  3. Katras funkcijas sākumā pievieno docstring (aprakstu trīskāršās pēdiņās """), paskaidrojot, ko funkcija dara.

Biežākās stila kļūdas:

1. Tab vs Spaces: Vienmēr izmanto 4 atstarpes atkāpei. Nejauc tās vienā failā!
2. Lieki komentāri: Neraksti komentārus, kas paskaidro pašsaprotamas lietas (piem., x = 5 # piešķir 5). Raksti komentārus tikai tam, kāpēc kods ir tāds.
3. Pārāk garas rindas: Centies neiziet ārpus 79–88 rakstzīmēm vienā rindā.
