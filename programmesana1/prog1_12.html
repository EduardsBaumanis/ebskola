1. tēma: Komandrinda un pamata drošība — mapes, komandas, PATH, licences

Stundas sasniedzamais rezultāts

Stundas beigās skolēns izmanto komandrindas pamatkomandas (cd, ls/dir, mkdir), izpilda Python programmu no CLI, izskaidro PATH nozīmi un piemēro drošas darba vides principus (paroles/2FA/personas dati), kā arī noformē README ar autoru un licenci. (SR: 2.3.1; 2.3.9; 3.1.5)

1. uzdevums — komandrindas pamati un programmas palaišana no CLI

Attēla avots: code.visualstudio.com (Terminal Basics — integrētais terminālis).

Mērķis: iemācīties pārvietoties pa mapēm un palaist programmu bez IDE pogas “Run”.

A. Atver termināli/komandrindu
Izvēlies vienu variantu:
• VS Code: View → Terminal
• Windows: Start → cmd vai PowerShell
• macOS/Linux: Terminal

B. Aizej uz projekta mapi
1) Atrodi, kur glabājas tavs projekts_01 (no iepriekšējās stundas).
2) Terminālī izmanto cd, lai atvērtu šo mapi.
Piemēri (pielāgo savam ceļam):
• Windows (cmd): cd Documents\projekts_01
• macOS/Linux: cd ~/Documents/projekts_01

C. Pārbaudi mapes saturu
1) Windows (cmd): ievadi dir
2) macOS/Linux: ievadi ls
3) Pārliecinies, ka sarakstā ir main.py un README.md.

D. Izveido jaunu mapi ar komandu mkdir
1) Terminālī ievadi: mkdir tests
2) Pārbaudi ar dir vai ls, ka mape ir izveidota.
3) Ieej mapē: cd tests
4) Atgriezies atpakaļ uz projekta mapi:
• Windows/macOS/Linux: cd ..

E. Palaid Python programmu tikai no CLI
1) Pārliecinies, ka esi mapē projekts_01 (kur atrodas main.py).
2) Ievadi:
• parasti: python main.py
• ja Windows vidē nestrādā: py main.py
3) Terminālī jāparādās “Hello, World!” (vai tavs teksts, ja esi to mainījis).

Paškontrole: ja komandai python main.py parādās kļūda “file not found”, tu visticamāk esi nepareizajā mapē (atkārto B un C).

2. uzdevums — vides mainīgie un PATH: kā sistēma atrod python

Attēla avots: architectryan.com (Add to the PATH on Windows — PATH rediģēšanas logs).

Mērķis: saprast, kāpēc dažreiz “python” terminālī strādā, bet citreiz ne, un kā to pārbaudīt.

A. Izskaidro (1 teikumā): kas ir PATH?
README failā (sadaļā “Problēmas” vai jaunā sadaļā “PATH”) uzraksti vienu teikumu, piemēram:
“PATH ir mapju saraksts, kurās operētājsistēma meklē programmas, kad terminālī ierakstu komandu (piem., python).”

B. Noskaidro, kur atrodas Python
Terminālī izpildi atbilstošo komandu:
• Windows (cmd): where python
• Windows (PowerShell): where.exe python
• macOS/Linux: which python vai which python3
Iekopē rezultātu README (ceļu, ko izvada).

C. Apskati PATH vērtību
Izpildi vienu no šīm komandām (atkarībā no vides):
• Windows (cmd): echo %PATH%
• Windows (PowerShell): $env:Path
• macOS/Linux: echo $PATH

D. (Ja python nestrādā)
1) Uztaisi ekrānšāviņu ar kļūdas paziņojumu (vai nokopē 1–2 rindas).
2) Atver Windows meklētāju un ieraksti environment variables → izvēlies “Edit the system environment variables”.
3) Atver Environment Variables… un atrodi Path (skat. attēlu augstāk).
4) Nerediģē bez skolotāja norādes—šajā uzdevumā pietiek saprast, kur tas atrodas un kāpēc tas ietekmē komandu “python”.

3. uzdevums — droša darba vide + licence un IP + README noformēšana

Attēla avots: docs.github.com (Adding a license to a repository — licenses sagataves izvēle).

Mērķis: noformēt projektu tā, lai tas ir korekts un drošs koplietošanai.

A. Drošības mini-tests (pieraksti README)
README beigās izveido sadaļu “Drošība” un atbildi ar īsiem teikumiem:
1) Paskaidro, kāpēc nedrīkst kodā (un repozitorijā) glabāt paroles vai piekļuves atslēgas.
2) Nosauc vienu piemēru “personas datiem”, kurus nevajadzētu publicēt (piem., personas kods, adrese u.tml.).
3) Nosauc vienu drošības soli kontam: spēcīga parole vai 2FA (divfaktoru autentifikācija).

B. Licence un autorība (pieraksti README)
README augšdaļā (zem virsraksta) pievieno:
• Autors: Tavs vārds/uzvārds
• Apraksts: 1–2 teikumi, ko projekts dara
• Licence: MIT (vai cita, ja skolotājs norāda)

C. README sagatave (kopē un aizpildi) 


# projekts_01
Autors: [Tavs vārds Uzvārds]

## Apraksts
Šis projekts izpilda pirmo Python programmu un demonstrē palaišanu no komandrindas.

## Kā palaist programmu?
- Komandrindā / terminālī: python main.py
  (ja Windows vidē nestrādā: py main.py)

## Licence
MIT

## Drošība
1) Kāpēc paroles nedrīkst glabāt kodā/repozitorijā? — [atbilde]
2) Personas datu piemērs, ko nepublicēt — [piemērs]
3) Konta drošības solis (spēcīga parole vai 2FA) — [atbilde]

Paškontrole: README jābūt autoram, aprakstam un licences nosaukumam. “Drošība” sadaļā jābūt 3 īsām atbildēm.

Papildus informācija

Komandrinda programmētājam ir ikdienas rīks: ar cd tu nonāc pareizajā mapē, ar dir/ls pārbaudi failus, ar mkdir izveido struktūru, un ar python main.py vari palaist programmu jebkurā datorā arī bez IDE.

PATH ir kā “meklēšanas maršruts” operētājsistēmai: ja Python mapes nav PATH sarakstā, komanda python var netikt atrasta. Tāpēc šajā stundā tu mācies ne tikai “ko ierakstīt”, bet arī kā pārbaudīt, kur tieši sistēma atrod Python.

Licence un drošība ir daļa no profesionāla projekta: licence pasaka, ko citi drīkst darīt ar tavu kodu, bet drošības ieradumi pasargā kontus un datus. Vienkāršākais noteikums: ja informāciju nebūtu droši uzrakstīt uz tāfeles klasē, tad to nedrīkst ievietot kodā vai publiskā repozitorijā.
