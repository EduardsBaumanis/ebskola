1. tēma: Ievads un darba vide. 2. stunda: Windows resursu pārvaldība.

Tavs šīs stundas izaicinājums: Tu iemācīsies atpazīt Windows failu sistēmas uzbūvi, izskaidrot atšķirību starp dažādiem failu atrašanās vietu veidiem un patstāvīgi pārvaldīt mapes un failus, izmantojot tikai komandrindu.

SR 2.3.9. (Operētājsistēmas resursu pārvaldība: failu sistēma, ceļi)

1. uzdevums (Vienkāršs): Orientēšanās Windows mapēs

Iemācīties noteikt, kurā mapē Tu pašlaik atrodies, un apskatīt tās saturu bez pelītes klikšķiem.

Izpildes soļi:

  1. Atver programmu VS Code un loga apakšā atver Terminal.
  2. Ieraksti pavēli echo %cd% un nospied Enter. Dators parādīs Tavu pašreizējo darba adresi (piemēram, C:\Lietotāji\Vārds).
  3. Ieraksti pavēli dir un nospied Enter. Tu redzēsi sarakstu ar visām mapēm un failiem, kas atrodas šajā vietā.
  4. Ieraksti pavēli cd .. (ievēro atstarpi un divus punktus) un nospied Enter. Tu pakāpsies vienu līmeni atpakaļ uz sākuma pusi.
  5. Vēlreiz ieraksti dir, lai pārliecinātos, ka esi nonācis citā vietā.

2. uzdevums (Viduvējs): Projekta "skeleta" izveide

Izveidot sakārtotu mapju struktūru topošajai spēlei, neizmantojot peli un logus.

Izpildes soļi:

  1. Terminālī ieraksti cd Desktop, lai nonāktu uz datora darbvirsmas.
  2. Izveido jaunu mapi savam darbam, rakstot mkdir laboratorija2.
  3. Ieej jaunajā mapē ar komandu cd laboratorija2.
  4. Izveido tur trīs apakšmapes dažādiem spēles materiāliem: mkdir kods, mkdir bildes un mkdir skanas.
  5. Ieraksti komandu tree un nospied Enter, lai vizuāli apskatītu izveidoto mapju koku.

3. uzdevums (Padziļināts): Failu manipulācija un relatīvie ceļi

Iemācīties pārvietot failus starp mapēm un palaist kodu, norādot ceļu līdz tam.

Izpildes soļi:

  1. Mapē laboratorija2 izveido tukšu failu, terminālī rakstot type nul > tests.py.
  2. Pārvieto šo failu uz mapi kods, rakstot move tests.py kods\.
  3. Atver VS Code kreisajā pusē failu kods\tests.py un ieraksti tajā kodu: print("Skripts veiksmīgi palaists no citas mapes!"). Saglabā to (Ctrl+S).
  4. Pārliecinies, ka terminālī Tu joprojām atrodies mapē laboratorija2 (to rāda adrese pirms kursora).
  5. Mēģini palaist skriptu, rakstot python kods\tests.py.

Jaunās vielas kopsavilkums

  • Datora resursi: Viss, ko operētājsistēma Tev nodrošina — vieta uz diska (faili), operatīvā atmiņa un jauda.
  • Terminālis: Tiešs teksta saziņas veids ar datoru, kas ļauj veikt darbības ātrāk nekā ar peli.
  • Absolūtais ceļš: Pilna adrese no paša sākuma (piemēram, C:\Lietotaji\Vards\Desktop\fails.py).
  • Relatīvais ceļš: Adrese attiecībā pret vietu, kurā Tu atrodies pašlaik (piemēram, kods\fails.py).

Biežākās kļūdas (un kā tās labot):

  • Kļūda: "The system cannot find the path specified": Tu mēģini ieiet mapē, kuras nav šajā vietā, vai esi pieļāvis pārrakstīšanās kļūdu nosaukumā. Vienmēr ieraksti dir, lai pārliecinātos, kā tieši saucas mape.
  • Aizmirsta atstarpe: Komanda un tās mērķis ir jāatdala ar atstarpi. Raksti cd Desktop, nevis cdDesktop.
  • Slīpsvītru lietošana: Windows sistēmā mapes atdala ar reverso slīpsvītru \ (Backslash). Ja ierakstīsi kods/tests.py, terminālis var nesaprast ceļu.
  • Atstarpes mapju nosaukumos: Ja mapi sauc mana spele, terminālis domās, ka tās ir divas dažādas komandas. Šādos gadījumos lieto pēdiņas: cd "mana spele".
  • Pazaudēta atrašanās vieta: Ja esi "apmaldījies" mapēs, ieraksti cd %userprofile%\Desktop, lai ātri atgrieztos uz darbvirsmas.

Zini, kāpēc tas ir svarīgi? Programmējot spēles, Tavs kods bieži meklēs attēlus vai skaņas citās mapēs. Bez izpratnes par failu ceļiem spēle šos resursus neatradīs un nedarbosies.
