SR 2.4.15. (Programmas sadalīšana moduļos)

1. uzdevums: Instrumentu kaste

Iemācīties pamata mehānismu, kā izsaukt funkciju, kas atrodas citā failā, izmantojot standarta import komandu.

Izpildes soļi:

  1. Izveido jaunu mapi mans_projekts un atver to ar VS Code.
  2. Izveido failu riki.py. Tajā definē funkciju linija(), kas izvada rindu ar 20 domuzīmēm.
  3. Izveido otru failu galvenais.py tajā pašā mapē.
  4. Faila galvenais.py augšā ieraksti: import riki.
  5. Izsauc funkciju, ierakstot: riki.linija().
  6. Novēro, kā Python "aiziet" uz otru failu, lai izpildītu Tavu komandu.

2. uzdevums: Stāsta glabātuve

Apgūt specifisku importēšanas veidu (from...import), kas ļauj ērtāk piekļūt konkrētiem mainīgajiem vai funkcijām.

Izpildes soļi:

  1. Izveido failu stasts.py. Tajā definē funkciju ievads(), kas ar return atgriež garu spēles sākuma tekstu.
  2. Failā spele.py pašā augšā ieraksti: from stasts import ievads.
  3. Izsauc funkciju pa tiešo: print(ievads()).
  4. Pievieno vēl vienu funkciju failā stasts.py un importē arī to, atdalot ar komatu.

3. uzdevums: CYOA Arhitekts

Izveidot pilnvērtīgu trīs failu sistēmu savai topošajai spēlei, nodalot drošību, saturu un galveno dzinēju.

Izpildes soļi:

  1. Izveido failu drosiba.py un ievieto tur savu try/except validācijas funkciju.
  2. Izveido failu pasaule.py un definē tur vismaz divas lokāciju funkcijas (piem., mežs, pils).
  3. Failā main.py importē nepieciešamos rīkus no abiem moduļiem.
  4. Saliec visu kopā: pajautā lietotājam skaitli (validē), un, balstoties uz to, izsauc attiecīgo lokāciju no pasaules faila.

Jaunā viela: Koda arhitektūra un Moduļi

Programmētāji neraksta visu kodu vienā failā. Tieši otrādi – mēs cenšamies kodu sadalīt pēc iespējas mazākos, loģiskos failos jeb moduļos. Tas padara kodu vieglāk lasāmu, testējamu un ļauj vairākiem cilvēkiem strādāt pie viena projekta vienlaicīgi.

1. Kas ir modulis?

Python valodā jebkurš .py fails ir modulis. Kad Tu raksti import random, Tu pieslēdz failu random.py, kas atrodas Python sistēmas mapēs. Tagad Tu vari veidot savas bibliotēkas.

2. Importēšanas stratēģijas

  • import mans_fails — Drošākais veids. Lai izsauktu funkciju, jālieto "uzvārds": mans_fails.start(). Tas novērš situācijas, kur divos failos ir funkcijas ar vienādiem nosaukumiem.
  • from mans_fails import start — Ērtākais veids. Vari saukt funkciju pa tiešo: start().

3. Kur Python meklē moduļus?

Kad Tu veic importu, dators vispirms skatās tajā pašā mapē, kurā atrodas fails, kuru Tu pašlaik darbini. Tāpēc visiem spēles failiem jābūt vienā projektā.

Biežākās kļūdas:

1. ModuleNotFoundError: Kļūda nosaukumā (atceries: Python atšķir lielos/mazos burtus!) vai fails nav tajā pašā mapē.
2. Cikliskais imports: Fails A importē B, bet fails B mēģina importēt A. Programma "iesprūst" bezgalīgā cilpā un avarē.
3. Nosaukumu konflikti: Nekad nesauc savu failu random.py vai string.py. Python vispirms mēģinās importēt Tavu failu, nevis īsto bibliotēku, un programma pārstās darboties. 

# Failā: matematika.py
def saskaitit(a, b):
    return a + b

# Failā: galvenais.py
import matematika

rezultats = matematika.saskaitit(5, 10)
print(f"Summa ir: {rezultats}")
Summa ir: 15
