SR 2.4.15. (Programmas sadalīšana moduļos)

Teorija: Koda arhitektūra un Moduļi

  • Modulis: Python valodā jebkurš .py fails ir modulis. Koda dalīšana moduļos padara to lasāmu, testējamu un ļauj komandai strādāt vienlaicīgi.
  • Meklēšana: Kad veic importu, Python vispirms meklē failus tajā pašā mapē, kur atrodas galvenais darbināmais fails. Visiem spēles failiem jābūt vienā projektā.
  • import fails: Drošākais importēšanas veids. Lai izsauktu funkciju, obligāti jālieto "uzvārds": fails.funkcija(). Tas novērš nosaukumu konfliktus.
  • from fails import funkcija: Ērtākais veids. Ļauj izsaukt konkrētu funkciju pa tiešo, bez faila nosaukuma priekšā: funkcija().

1. uzdevums: Instrumentu kaste

Pielietot pamata mehānismu funkciju izsaukšanai no cita faila, izmantojot standarta import komandu.

Izpildes soļi:

  1. Izveido jaunu mapi mans_projekts un atver to ar VS Code.
  2. Izveido failu riki.py. Tajā definē funkciju def linija():, kas izvada rindu ar 20 domuzīmēm.
  3. Izveido otru failu galvenais.py tajā pašā mapē.
  4. Faila galvenais.py augšā ieraksti: import riki.
  5. Izsauc funkciju, ierakstot: riki.linija().
  6. Palaid galvenais.py un novēro, kā Python izpilda komandu no cita faila.

2. uzdevums: Stāsta glabātuve

Izmantot specifisku importēšanas veidu (from ... import), lai tieši piekļūtu konkrētiem koda blokiem.

Izpildes soļi:

  1. Izveido failu stasts.py. Tajā definē funkciju def ievads():, kas ar return atgriež garu spēles sākuma tekstu.
  2. Failā spele.py pašā augšā ieraksti: from stasts import ievads.
  3. Izsauc funkciju pa tiešo un izvadi to ekrānā: print(ievads()).
  4. Pievieno vēl vienu funkciju failā stasts.py (piemēram, beigas()) un importē arī to, atdalot nosaukumus ar komatu: from stasts import ievads, beigas.

3. uzdevums: CYOA Arhitekts

Projektēt un izveidot trīs failu sistēmu savai spēlei, loģiski nodalot koda funkcionalitāti.

Izpildes soļi:

  1. Izveido failu drosiba.py un ievieto tur savu iepriekš izveidoto try/except lietotāja ievades validācijas funkciju.
  2. Izveido failu pasaule.py un definē tur vismaz divas lokāciju aprakstu funkcijas (piemēram, mežs, pils).
  3. Izveido failu main.py un tajā importē nepieciešamos rīkus no abiem iepriekšējiem moduļiem.
  4. Saliec visu kopā main.py failā: pajautā lietotājam izvēli (validējot ievadi ar funkciju no drosiba.py) un, balstoties uz to, izsauc attiecīgo lokāciju no pasaule.py.

Biežākās kļūdas (un kā tās labot):

  • ModuleNotFoundError: Kļūda nosaukumā (atceries: Python atšķir lielos un mazos burtus) vai fails neatrodas tajā pašā mapē, kur galvenais skripts. Risinājums: Pārbaudi failu nosaukumus un mapes struktūru.
  • Cikliskais imports: Fails A importē B, bet fails B tajā pašā laikā mēģina importēt A. Programma "iesprūst" bezgalīgā cilpā un avarē. Risinājums: Pārplāno koda struktūru; parasti galvenais fails importē citus, bet pakārtotie faili neimportē galveno.
  • Nosaukumu konflikti: Sava faila nosaukšana sistēmas moduļu vārdos (piemēram, random.py vai string.py). Python vispirms mēģinās importēt Tavu failu, nevis īsto bibliotēku. Risinājums: Izmanto unikālus un aprakstošus failu nosaukumus.
