6. tēma: Datu pastāvība un Analīze. 1. stunda: Failu I/O.

Tavs šīs stundas izaicinājums: Piešķirt savai programmai "bezmirstību". Tu iemācīsies izmantot Failu I/O (Input/Output), lai saglabātu informāciju (piemēram, spēles progresu vai rezultātu tabulas) failos, kas neizzūd pēc programmas aizvēršanas. Mēs apskatīsim trīs galvenos formātus: TXT, JSON un CSV.

SR 2.4.14. (Failu lasīšana un rakstīšana)

1. uzdevums (Vienkāršs): Digitālā dienasgrāmata (.txt)

Iemācīties pamata darbus ar vienkāršiem teksta failiem: atvēršanu, ierakstīšanu un nolasīšanu.

Izpildes soļi:

  1. VS Code izveido failu dienasgramata.py.
  2. Pajautā lietotājam: "Kā Tev šodien gāja?".
  3. Izmantojot with open("notes.txt", "w", encoding="utf-8") as f:, ieraksti atbildi failā.
  4. Pievieno koda daļu, kas atver šo pašu failu "r" režīmā un nolasa saturu.
  5. Izvadi nolasīto tekstu terminālī: print(f"Šodienas ieraksts: {saturs}").

2. uzdevums (Viduvējs): "Save Game" dzinējs (.json)

Apgūt JSON formātu, kas ir ideāls Python vārdnīcu struktūras saglabāšanai.

Izpildes soļi:

  1. Izveido failu game_save.py un importē json bibliotēku.
  2. Izveido vārdnīcu speletajs ar datiem: vārds, HP, zelts.
  3. Izmanto json.dump(speletajs, f), lai saglabātu to failā progress.json.
  4. Izveido funkciju, kas nolasa šo failu, palielina zeltu par 50 un saglabā izmaiņas atpakaļ failā.

3. uzdevums (Padziļināts): Spēlētāju saraksts (.csv)

Iemācīties strādāt ar tabulāriem datiem, izmantojot CSV formātu, un veikt to analīzi.

Izpildes soļi:

  1. Importē csv moduli failā rezultati.py.
  2. Izveido programmu, kas ciklā ļauj ievadīt vairāku spēlētāju vārdus un viņu punktus.
  3. Saglabā šos datus failā top_rezultati.csv (katrs spēlētājs savā rindā).
  4. Nolasi failu un aprēķini visu ievadīto spēlētāju vidējo punktu skaitu.

Jaunā viela: Failu apstrādes režīmi

Atverot failu, mums ir jāpasaka Python, ko mēs plānojam darīt. Katram režīmam ir sava loma un riski.

Režīms Nosaukums Ko tas dara? Bīstamība
"r" Read Atver tikai lasīšanai. Kļūda, ja fails neeksistē.
"w" Write Izveido jaunu vai pārraksta esošo. Viss iepriekšējais saturs pazūd!
"a" Append Pieraksta datus faila beigās. Saglabā esošo saturu.

Kāpēc lietot "with open()"?

Šī konstrukcija nodrošina, ka fails tiek automātiski aizvērts pat tad, ja programmā rodas kļūda. Tas novērš datu korupciju un atmiņas noplūdes.

Biežākās kļūdas:

1. encoding="utf-8": Neaizmirsti šo parametru, citādi latviešu burti pārvērtīsies par ķeburiem.
2. Write vs Append: Izmantojot "w", Tu izdzēs visu, kas failā bija iepriekš. Lieto "a", lai papildinātu.
3. Import: Pirms lietot JSON vai CSV funkcijas, faila augšā ir jābūt import json vai import csv. 

import json

dati = {"vards": "Drosmīgais", "hp": 100}

# Saglabāšana
with open("save.json", "w", encoding="utf-8") as f:
    json.dump(dati, f, indent=4)

# Nolasīšana
with open("save.json", "r", encoding="utf-8") as f:
    ieladetie_dati = json.load(f)

print(ieladetie_dati["vards"])
