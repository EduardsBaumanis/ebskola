Tēmas Sasniedzamie rezultāti Palīgmateriāli

5. tēma: Noslēguma darbs. Projekts: "CYOA 2.0: Sistēmu integrācija"

Mērķis: Izveidot dinamisku teksta piedzīvojumu spēli, kurā spēlētāja progress un resursi tiek pārvaldīti ar Python kolekcijām. Šis darbs apvieno visu 5. tēmā apgūto — no sarakstiem un vārdnīcām līdz lietotāju datu analīzei.

Svarīgi: Vērtēšana notiks, analizējot Tavu GitHub repozitoriju. Tiks ņemta vērā gan koda kvalitāte, gan izstrādes gaita (commits).

Tehniskās prasības

Tavam projektam obligāti jāsatur šādi elementi:

  • Varoņa profils (Dictionary): Visi spēlētāja dati (HP, zelts, vārds) glabājas vienā vārdnīcā.
  • Mugursoma (List): Dinamisks saraksts inventāra pārvaldībai (pievienošana/izņemšana).
  • Datu pārlūkošana (For loop): Vismaz divas vietas, kur dati tiek apstrādāti ar ciklu (piem., inventāra izvade).
  • Aptaujas integrācija: Spēlē jābūt vismaz 2 elementiem, kas izvēlēti, balstoties uz 5.5. stundas analīzi.
  • Modularitāte: Projekts ir sadalīts vismaz 3 failos: main.py, logic.py un data.py.

Vērtēšanas kritēriji (SLA)

Kritērijs Sācis apgūt (1-3) Turpina apgūt (4-6) Apguvis (7-8) Padziļināti (9-10)
Datu arhitektūra Lieto atsevišķus mainīgos. Lieto kolekcijas, bet ir kļūdas datu nodošanā. Prasmīgi lieto vārdnīcas un sarakstus. Lieto ligzdotas struktūras un universālas funkcijas.
Sistēmu loģika Inventārs ir statisks. Var pievienot mantas, bet tās neietekmē spēli. Inventārs un resursi tieši ietekmē spēles gaitu. Sarežģīta loģika (piem., tirgošanās, HP aprēķini).
Git & Kvalitāte Viens commits, nav README. Commits ir haotiski. PEP 8 ievērots daļēji. Skaidra commit vēsture. Ievērotas PEP 8 un docstrings. Profesionāls README un nevainojama commit vēsture.

Ieteicamā darba gaita

  1. Sagatavošanās: Izveido mapi, inicializē Git un izveido data.py ar sākuma vārdnīcu.
  2. Sistēmu izveide: Uzraksti funkciju statusa rādīšanai (HUD) un mantu pievienošanai.
  3. Stāsta būvniecība: Pievieno lokācijas logic.py failā, padodot spēlētāja vārdnīcu kā parametru.
  4. Integrācija: Ievieto spēlē lietotāju aptaujas ieteikumus (piem., jaunu ieroci vai tēlu).
  5. Noslēgums: Veic pēdējo PEP 8 revīziju un Push visus labojumus uz GitHub.

def doties_uz_veikalu(speletajs, jauna_prece):
    """Pievieno preci sarakstam un atņem zeltu no vārdnīcas."""
    if speletajs["zelts"] >= 10:
        speletajs["inventars"].append(jauna_prece)
        speletajs["zelts"] -= 10
        print(f"Iegādāts: {jauna_prece}")
    else:
        print("Nepietiek zelta!")
    return speletajs
