1. tēma: Operētājsistēmas resursi un darba efektivitāte

Stundas sasniedzamais rezultāts

Stundas beigās skolēns izskaidro operētājsistēmas lomu, identificē procesus un sistēmas resursus (CPU, RAM), izvērtē nepieciešamos resursus konkrētam uzdevumam, kā arī piemēro ergonomikas principus darba vidē. (SR: 2.3.9; 2.3.11; 3.1.1)

1. uzdevums — Kas ir OS? Procesi un resursi

Avots: Wikimedia Commons — Operating system placement diagram.

Mērķis: saprast, kā operētājsistēma pārvalda datoru.

A. Apskati attēlu
1) Atrodi, kur atrodas lietojumprogrammas.
2) Atrodi, kur atrodas aparatūra (hardware).
3) Paskaidro, kā OS savieno šos divus slāņus.

B. Atrodi procesus savā datorā
1) Windows: Ctrl+Shift+Esc → Task Manager.
2) macOS: Activity Monitor.
3) Atrodi, cik procesi šobrīd darbojas.
4) Noskaidro, kurš process izmanto visvairāk CPU vai RAM.

C. README failā uzraksti:
• Kas ir operētājsistēma?
• Kas ir process?
• Ko nozīmē CPU un RAM izmantošana?

2. uzdevums — Resursu izvēle konkrētam uzdevumam

Avots: Wikimedia Commons — Computer system diagram.

Mērķis: izvērtēt, cik jaudīgs dators nepieciešams dažādiem uzdevumiem.

README failā izveido tabulu ar 3 kolonnām:
Uzdevums | Nepieciešamie resursi | Pamatojums 


1) Teksta apstrāde
2) Mašīnmācīšanās projekts
3) Spēles izstrāde

Norādījumi:
• Teksta apstrādei parasti pietiek ar nelielu RAM un vidēju CPU.
• ML projektam nepieciešams vairāk RAM un jaudīgs CPU (vai GPU).
• Spēles izstrādei svarīgs CPU + grafiskā karte (GPU).

Uzdevums: katram punktam pamato, kāpēc tieši šādi resursi ir vajadzīgi.

3. uzdevums — Ergonomika un sagatavošanās mini-projektam

Avots: Wikimedia Commons — Ergonomic workstation diagram.

Mērķis: organizēt drošu un efektīvu darba vidi un sagatavoties mini-projektam.

A. Ergonomika
1) Pārbaudi monitora augstumu — augšējai malai jābūt acu līmenī.
2) Novērtē, vai plaukstas atrodas taisnā līnijā ar klaviatūru.
3) Uzraksti 2 riskus ilgstošam darbam (redze, mugura).
4) Nosauc vienu ieradumu, kas palīdz (piem., pauze ik pēc 20 minūtēm).

B. Sagatavošanās mini-projektam
README failā izveido sadaļu “Mini-projekta plāns” un definē:
• Projekta mērķis (1–2 teikumi)
• Nepieciešamie resursi (CPU, RAM, programmas)
• Mapju struktūra (piem., src/, data/, README.md)
• Git repo izveide (komanda: git init) 


git init
git add .
git commit -m "Prepare project structure"

Paškontrole:
• Vai projekts ir strukturēts mapēs?
• Vai ir izveidots Git repozitorijs?
• Vai README apraksta mērķi un resursus?

Papildus informācija

Operētājsistēma pārvalda aparatūru un nodrošina, ka programmas var droši darboties vienlaicīgi.

CPU veic aprēķinus, RAM īslaicīgi glabā aktīvos datus, bet SSD uzglabā failus ilgtermiņā.

Ergonomiska darba vieta un regulāras pauzes palīdz saglabāt produktivitāti un veselību ilgtermiņā.
