Teorija: Izņēmumu apstrāde (Exceptions Handling)

Programmēšanā negaidītas kļūdas sauc par Exceptions (Izņēmumiem). Tā vietā, lai ļautu programmai "uzkārties", mēs šīs kļūdas pārtveram un apstrādājam.

  • Try block (Mēģinājuma bloks): Koda daļa, kurā varētu notikt kļūda (piemēram, datu konvertācija). Dators "mēģina" šo kodu izpildīt.
  • Except block (Izņēmuma bloks): Drošības tīkls, kas aktivizējas TIKAI tad, ja try blokā rodas kļūda.
  • ValueError (Vērtības kļūda): Specifisks izņēmuma tips, kas rodas, kad mēģinām pārvērst neatbilstošu String (Simbolu virkni) par matemātisku skaitli.
# Īss kļūdu apstrādes paraugs
try:
    vecums = int(input("Cik Tev gadu? "))
    print(f"Tev ir {vecums} gadi.")
except ValueError:
    print("Kļūda! Lūdzu, ievadi tikai ciparus.")

Biežākās kļūdas (un kā tās labot):

  • Bare except (Tukšs izņēmums): Nekad neizmanto atslēgvārdu except: bez norādīta kļūdas tipa, jo tas noķers pilnīgi visas kļūdas, pat tās, kas saistītas ar paša koda sintakses problēmām vai programmas piespiedu apturēšanu (Ctrl+C). 
    # Slikti:
except:
    print("Notika kļūda.")

# Labi:
except ValueError:
    print("Datu tipa kļūda.")
  • Bloated try block (Pārpildīts mēģinājuma bloks): Iekš try liec tikai tās rindiņas, kas reāli var izraisīt norādīto kļūdu. 
    # Slikti:
try:
    vards = input("Vārds: ") # Šeit ValueError nevar notikt
    vecums = int(input("Vecums: "))
except ValueError:
    print("Kļūda!")

# Labi:
vards = input("Vārds: ")
try:
    vecums = int(input("Vecums: "))
except ValueError:
    print("Kļūda!")
  • Missing break (Aizmirsta pārtraukšana): Ja, ievietojot try/except bezgalīgajā ciklā, aizmirsīsi pielikt break veiksmīgas izpildes gadījumā, cikls turpināsies mūžīgi.
