3. tēma: Noslēguma darbs. Projekts: "Gudrais Akmens-Šķēres-Papīrīts".

Tavs mērķis: Izveidot pilnvērtīgu, kļūdu izturīgu spēli, kurā cilvēks var spēkoties ar datoru vairāku raundu garumā. Vērtējums tiks piešķirts, analizējot Tavu GitHub krātuvi — kodu, versiju vēsturi un dokumentāciju.

Darba uzdevumi

1. solis: Spēles loģikas kodols

  1. Izmanto import random, lai dators veiktu nejaušu izvēli no saraksta.
  2. Realizē tiesneša loģiku, izmantojot if/elif/else un loģiskos operatorus (and/or).
  3. Ievieto visu spēli while True: ciklā, lai to varētu spēlēt atkārtoti.

2. solis: Izturība un Validācija

  1. Nodrošini, ka programma "nenolūst", ja lietotājs ievada tukšumu vai nepareizu vārdu (izmanto continue).
  2. Ja spēlē tiek izmantotas likmes vai punkti, lieto try/except bloku skaitliskās ievades drošībai.
  3. Izveido komandu "iziet", kas ar break korekti aptur ciklu un atvadās no spēlētāja.

3. solis: Publicēšana un Stils

  1. Ievēro PEP 8 standartus (Snake Case mainīgie, atstarpes ap operatoriem).
  2. Veic vismaz 3 jēgpilnus commit savā Git vēsturē (piem., "Pievienota validācija").
  3. Papildini README.md failu ar īsu spēles aprakstu un instrukciju.
