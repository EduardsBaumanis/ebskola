1. tēma: Git pamati — repozitorijs, commit, branch, merge un .gitignore

Stundas sasniedzamais rezultāts

Stundas beigās skolēns izskaidro repozitorija, commit un branch jēdzienus, izveido Git repozitoriju, veic vairākus commit, izveido un apvieno branch, kā arī izmanto .gitignore failu. (SR: 2.4.9)

1. uzdevums — repo, commit, branch (vizualizācija)

Avots: Wikimedia Commons — Git branch and merge diagram.

Apskati diagrammu un atrodi:
• commit punktus,
• branch atzarojumu,
• merge brīdi.

README failā uzraksti saviem vārdiem:
• Kas ir repozitorijs?
• Ko dara commit?
• Kāpēc branch ir noderīgs?

2. uzdevums — git init, add, commit

Avots: Wikimedia Commons — Git data transport / repository diagram.

Atver termināli projekta mapē un izpildi: 


git init
git status
git add .
git commit -m "Initial commit"

Maini failā main.py tekstu un izveido vēl vienu commit: 


git add main.py
git commit -m "Update output text"
git log --oneline

Paškontrole: git log jāparāda vismaz 2 commit.

3. uzdevums — branch, merge un .gitignore

Avots: Wikimedia Commons — klikšķini, lai atvērtu pilnā izmērā. 


git checkout -b experiments
git add README.md
git commit -m "Edit README in branch"
git checkout main
git merge experiments

Refleksija:
• Ko dara commit?

Papildus informācija

Commit saglabā konkrētu projekta stāvokli un veido izmaiņu vēsturi.

Branch ļauj eksperimentēt, nenojaucot galveno versiju.

.gitignore novērš lieku failu iekļaušanu repozitorijā.
