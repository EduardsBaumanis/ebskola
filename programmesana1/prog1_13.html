1. tēma: Ievads un darba vide. 3. stunda: Git uzstādīšana un pirmie saglabāšanas punkti.

Tavs šīs stundas izaicinājums: Tu iemācīsies patstāvīgi uzstādīt Git vadības sistēmu savā datorā, iestatīt savu digitālo parakstu un izveidot pirmo lokālo krātuvi, lai nodrošinātu sava darba saglabāšanu vēsturē.

SR 2.4.9. (Versiju pārvaldības lietošana lokāli: krātuves izveide, koda versiju saglabāšana)

Teorija: Kas ir Git un kāpēc tas ir nepieciešams?

  • Versiju vadības sistēma (Git): Profesionāla "laika mašīna", kas atceras katru koda izmaiņu un ļauj pie tām atgriezties, ja kaut kas noiet greizi.
  • Krātuve (Repository): Slēpta mape tavam projektam (.git), kurā dators glabā visu izmaiņu vēsturi.
  • Sagatavošanas zona (Staging Area): Vieta, kur Tu sakārto failus pirms to galīgās saglabāšanas vēsturē.
  • Saglabāšanas punkts (Commit): Fiksēts vēstures brīdis ar Tavu parakstu un aprakstu. Tas ir kā "Check Point" video spēlē.

1. uzdevums: Git uzstādīšana un pārbaude

Sagatavot datoru darbam ar versiju pārvaldības sistēmu Windows vidē.

Izpildes soļi:

  1. Dodies uz vietni git-scm.com un lejuplādē Windows versiju.
  2. Veic instalāciju, visos soļos spiežot "Next" (atstājot standarta iestatījumus).
  3. Atver VS Code termināli (Ctrl+Shift+`).
  4. Ieraksti komandu: git --version.
  5. Ja terminālis izvada versijas numuru (piem., git version 2.4x.x), instalācija ir veiksmīga.

2. uzdevums: Identitāte un krātuves inicializācija

Pateikt datoram, kas veic izmaiņas, un izveidot pirmo krātuvi projekta mapē.

Izpildes soļi:

  1. Iestati savu vārdu: git config --global user.name "Tavs Vārds".
  2. Iestati e-pastu: git config --global user.email "tavs@epasts.lv".
  3. VS Code atver savu mapi mana_spele (File -> Open Folder).
  4. Terminālī ieraksti komandu: git init.
  5. Pārbaudi stāvokli: git status. Failiem jābūt sarkaniem, jo tie vēl nav pievienoti krātuvei.

3. uzdevums: Pirmā saglabāšanas punkta (Commit) izveide

Apgūt pilnu ciklu, lai fiksētu pašreizējo progresu vēsturē.

Izpildes soļi:

  1. Sagatavo visus failus: git add . (punkts nozīmē "visu").
  2. Ieraksti git status – failu nosaukumiem jākļūst zaļiem.
  3. Izveido vēstures punktu: git commit -m "Projekta struktura un pirmais skripts".
  4. Atver apraksts.txt, pieraksti jaunu teikumu un saglabā (Ctrl+S).
  5. Atkārto: git add . un git commit -m "Papildināts apraksts".
  6. Apskati vēsturi: git log --oneline. Nospied Q, lai izietu no saraksta.

Biežākās kļūdas (un kā tās labot):

  • "git is not recognized...": Restartē VS Code programmu, lai Windows pamanītu jauno Git instalāciju.
  • Aizmirsts punkts: Komandā git add . punkts ir obligāts, lai pievienotu visas izmaiņas.
  • Sarkani faili pēc commit: Pārliecinies, vai esi saglabājis pašu failu (Ctrl+S). Git neredz to, kas nav ierakstīts diskā.
  • Iestrēgšana vēsturē: Ja terminālis neļauj rakstīt pēc git log, nospied Q.

Git pamata komandu kopsavilkums

git init          # Inicializē (izveido) krātuvi
git add .         # Sagatavo visas izmaiņas saglabāšanai
git commit -m ""  # Izveido saglabāšanas punktu ar ziņojumu
git status        # Parāda pašreizējo krātuves stāvokli
git log --oneline # Parāda īsu izmaiņu vēsturi
[main (root-commit) 5b1d2e4] Projekta struktura
2 files changed, 5 insertions(+)
create mode 100644 tests.py

Atceries: Kļūdas terminālī ir normāla programmēšanas sastāvdaļa. Lasi ziņojumus — Git bieži vien pats priekšā pasaka pareizo komandu!
