1. tēma: Ievads un darba vide. 3. stunda: Git uzstādīšana un pirmie saglabāšanas punkti.

Tavs šīs stundas izaicinājums: Tu iemācīsies patstāvīgi uzstādīt Git vadības sistēmu savā datorā, iestatīt savu digitālo parakstu un izveidot pirmo lokālo krātuvi, lai nodrošinātu sava darba saglabāšanu vēsturē.

SR 2.4.9. (Versiju pārvaldības lietošana lokāli: krātuves izveide, koda versiju saglabāšana)

1. uzdevums (Vienkāršs): Git lejuplāde un uzstādīšana

Sagatavot datoru darbam ar versiju pārvaldības sistēmu, uzstādot Git programmatūru Windows vidē.

Izpildes soļi:

  1. Atver pārlūkprogrammu un dodies uz vietni git-scm.com.
  2. Sadaļā "Downloads" izvēlies Windows versiju.
  3. Lejuplādē "64-bit Git for Windows Setup".
  4. Atver lejuplādēto failu un veic instalāciju, visos soļos spiežot "Next" (atstājot ražotāja ieteiktos standarta iestatījumus).
  5. Kad instalācija pabeigta, atver VS Code programmu.
  6. Atver termināli (Ctrl+Shift+`) un ieraksti git --version. Ja redzi versijas numuru, Git ir gatavs darbam.

2. uzdevums (Viduvējs): Identitātes iestatīšana un krātuves izveide

Pateikt datoram, kas veic izmaiņas, un izveidot pirmo "seifu" (krātuvi) sava projekta mapē.

Izpildes soļi:

  1. VS Code terminālī ieraksti savu vārdu un uzvārdu: git config --global user.name "Tavs Vārds" (spied Enter).
  2. Ieraksti savu e-pastu: git config --global user.email "tavs@epasts.lv" (spied Enter).
  3. Atver savu iepriekš izveidoto mapi mana_spele (File -> Open Folder).
  4. Terminālī ieraksti galveno komandu: git init.
  5. Ieraksti git status, lai redzētu, ka Git ir veiksmīgi "pamanījis" Tavus failus (tie pašlaik būs sarkanā krāsā).

3. uzdevums (Padziļināts): Pirmā saglabāšanas punkta (Commit) izveide

Iemācīties pilnu ciklu, lai fiksētu pašreizējo darbu vēsturē.

Izpildes soļi:

  1. Terminālī ieraksti git add . (neaizmirsti punktu beigās – tas nozīmē "pievienot visu").
  2. Vēlreiz ieraksti git status – failu nosaukumiem jākļūst zaļiem (tie ir sagatavoti saglabāšanai).
  3. Izveido vēstures punktu: git commit -m "Projekta sakums un pirmais skripts".
  4. Atver failu apraksts.txt, pieraksti jaunu teikumu par spēles mērķi un saglabā (Ctrl+S).
  5. Atkārto procesu: git add . un tad git commit -m "Papildinats speles apraksts".
  6. Ieraksti git log --oneline, lai apskatītu savu pirmo vēstures sarakstu. (Nospied Q, lai izietu no saraksta).

Jaunās vielas kopsavilkums

  • Kas ir Git: Tā ir "laika mašīna", kas atceras katru koda izmaiņu un ļauj pie tām atgriezties.
  • Krātuve (Repository): Slēpta mape tavam projektam, kurā Git glabā visus vēstures punktus.
  • Git Init: Komanda, kas pasaka datoram, ka šī mape turpmāk tiks uzraudzīta.
  • Commit (Saglabāšanas punkts): Darba fiksēšana vēsturē ar īsu aprakstu. Tas ir kā "Check Point" video spēlē.
  • Identitāte: Vārds un e-pasts ir nepieciešami, lai vēsturē būtu redzams, kurš veicis labojumus.

Biežākās kļūdas (un kā tās labot):

  • Kļūda: "git is not recognized...": Ja pēc instalācijas terminālis neatpazīst komandu git, vienkārši aizver un vēlreiz atver VS Code programmu. Windows sistēmai ir "jāierauga" jaunā programma.
  • Aizmirsts punkts pie "git add": Komanda git add . obligāti prasa punktu beigās. Punkts pasaka Git: "pievieno visu, kas atrodas šajā mapē".
  • Git status rāda sarkanu pēc commit: Pārliecinies, ka esi saglabājis failu (Ctrl+S). Git nevar saglabāt vēsturē izmaiņas, kas vēl atrodas tikai VS Code atmiņā, bet nav ierakstītas diskā.
  • Iestrēgšana vēstures skatā: Pēc komandas git log dators reizēm gaida Tavu nākamo soli. Vienkārši nospied burtu Q uz tastatūras, lai atgrieztos pie komandu rakstīšanas.
  • Kļūda pie identitātes: Ja Tavā vārdā ir atstarpes (piemēram, "Jānis Bērziņš"), obligāti lieto pēdiņas: git config --global user.name "Jānis Bērziņš".

Atceries: Kļūdas terminālī ir normāla programmēšanas sastāvdaļa. Lasi, ko dators Tev raksta — parasti tur jau ir pasniegts risinājums!
